本次任务目标：

优化地图模块，让微信小程序接入腾讯地图 API，实现类似地图 App 中“地点搜索、点击地图公共地点、查看地点详情、规划路线、导航过去”的基础能力。

腾讯地图 Key 已提供，开发时请按照项目现有配置方式使用。如果项目已有环境变量、配置文件或统一常量管理方式，优先按项目规范接入，不要在多个页面或组件里重复硬编码。

本次地图模块需要实现以下核心需求：

1. 地图页面展示腾讯地图底图上的公共 POI 点。
   用户在地图上看到的公共地点，例如教学楼、宿舍、食堂、超市、校门等，应当可以被点击。
2. 用户点击地图上的公共 POI 后，前端获取该 POI 的名称和坐标。
   由于小程序地图组件点击公共 POI 时可能只能拿到名称和经纬度，所以需要结合后端接口调用腾讯位置服务，对该 POI 做二次匹配，拿到更完整的标准 POI 信息。
3. 用户点击公共 POI 后，需要展示地点详情。
   详情内容包括地点名称、地址、分类、距离等信息。展示形式可以是在地图页的动态内容展示区域内展示，风格要和现有地图模块 UI 保持一致。
4. 地图模块支持地点搜索。
   用户可以搜索校园范围内的公共 POI，例如搜索某某楼、某某宿舍、某某超市。搜索结果应当限制在学校附近或校园范围内，避免返回大量无关城市级结果。
5. 用户点击搜索结果后，地图移动到该地点。
   需要在地图上高亮该 POI，并展示地点详情卡片。搜索结果列表中应展示名称、地址、分类、距离等基础信息。
6. 管理员创建任务点时，可以自动关联附近公共 POI。
   当管理员在地图上选择一个自定义任务点坐标后，系统需要根据该坐标调用腾讯位置服务，查找附近的公共 POI，并推荐最近或最相关的地点给管理员确认，然后将最近或最相关的POI填入选点页的位置补充说明中。同时选点页位置补充说明和喂食点名称都不能够有默认值，必须为空，而且必须要填写。
7. 任务点需要同时保存“项目自定义坐标”和“关联 POI 信息”。
   自定义坐标是任务点真实位置，不能被 POI 坐标覆盖。关联 POI 只是辅助展示和搜索的信息。任务点应支持保存关联 POI 的 ID、名称、地址、分类、经纬度、距离、匹配方式等信息。
8. 管理员可以确认、切换或取消关联 POI。
   创建任务点时，如果系统推荐了附近 POI，需要展示类似“检测到附近地点：某某楼，距离 37 米，是否关联？”的交互。管理员可以选择关联、换一个，或者不关联。
9. 用户进入任务详情页后，可以查看任务点位置。
   如果任务点有关联 POI，则可以展示关联 POI 名称和地址；如果没有关联 POI，则展示任务点本身的名称和说明。
10. 用户可以从当前位置到任务点进行路线预览。
    使用腾讯地图路线规划能力，优先实现校园场景下的步行路线规划。路线需要绘制在小程序 map 组件上，并展示距离和预计时间。
11. 用户可以导航到任务点。
    导航目的地优先使用任务点自定义坐标，而不是关联 POI 坐标。地点名称可以优先展示关联 POI 名称。可以使用微信小程序 `wx.openLocation` 调起导航能力。
