# 前端缺省页组件设计

## 背景

页面原型中有多种缺省状态，包括数据为空、记录为空、搜索结果为空、网络加载异常、权限不足、404、图片加载失败、位置为空和消息为空。这些页面结构相近，适合做成统一组件，减少后续页面重复实现。

## 范围

本切片实现可复用缺省页能力，并替换当前四个一级 tab 的占位内容。

已纳入配置的常用状态：

- `under_development`：功能建设中
- `empty_data`：数据为空
- `empty_records`：记录为空
- `empty_search`：搜索结果为空
- `network_error`：网络加载异常
- `permission_denied`：权限不足
- `not_found`：404
- `empty_location`：位置为空
- `image_error`：图片加载失败
- `empty_messages`：消息为空

当前四个一级 tab 页面先统一使用 `under_development`，后续业务页面再按实际场景切换到对应状态。

## 组件设计

- `src/components/default-state.ts` 保存状态配置、插画资源、标题、说明和默认按钮。
- `src/components/DefaultState.vue` 负责渲染背景、标题、白色圆角卡片、插画、说明和操作按钮。
- 背景复用 `frontend/素材/加载页素材/背景.jpg`，避免再引入一套新背景资源。
- 插画使用 `frontend/素材/svg/缺省页` 中的 SVG，不使用外部替代素材。
- 页面可以通过 props 覆盖标题、说明和按钮文案；按钮事件由页面决定具体动作。
