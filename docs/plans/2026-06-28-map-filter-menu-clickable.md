## 根因

原筛选菜单是普通 `view/button`，并放在包含小程序原生 `map` 的 `.map-viewport` 内。地图发生缩放手势后，原生地图层可能重建或优先接管触摸事件，导致菜单视觉上仍在原位置，但点击事件被地图层吞掉。

## 方案

1. 将筛选菜单移出 `.map-viewport`，放在地图和抽屉之间的独立 `cover-view` 浮层中，避免被原生 `map` 层遮挡事件。
2. 使用 `cover-view/cover-image` 渲染小程序地图安全浮层。
3. 使用 `filter-menu.wxs` 驱动菜单宽度、展开透明度和箭头旋转动画。
4. 使用 `drawer.wxs` 同步移动浮层位置，保证抽屉拖动或地图区域变化后视觉位置与点击区域一致。
5. 使用更新后的地图点 SVG 素材，并用 `箭头.svg` 替代 CSS 边框箭头。

## 验证

- 新增回归测试覆盖：
  - 菜单位于原生地图安全 `cover-view` 浮层。
  - 筛选项使用命名 SVG 素材。
  - 箭头动画由 WXS 控制，收起态向下、展开态向上。
- 跑前端测试、类型检查和 H5/小程序构建。

## 不做

- 不改地图 API。
- 不改数据库表。
- 不重做筛选业务逻辑。
