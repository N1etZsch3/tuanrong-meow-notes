# 管理员人员管理与地标点设计

## 背景

- 当前管理员入口已有“添加账户”“发布喂食任务”“新建物资点”。
- 鉴权模块已有管理员创建、列表、重置密码、状态、角色基础接口，但缺少人员详情编辑、部门筛选和喵喵号排序。
- 地图模块已经支持 `landmark` 点位类型，地标应作为长期 `map_points` 数据保存，不新增业务表。
- 地图首页已能渲染 `landmark` marker，并已使用 `frontend/素材/png/地图点/地标.png`。
- 任务列表和猫咪列表当前是整体滚动，搜索和筛选会随列表滚走；本次按需求统一为顶部固定、底部内容滚动。

## 设计选择

### 人员管理

- 新增管理员入口“人员管理”，移除独立“添加账户”入口。
- 人员管理页包含固定顶部搜索和筛选区，下方成员列表独立滚动。
- 筛选项：
  - 身份：全部、管理员、成员、暑期志愿者。
  - 部门：全部、生存保障部、活动部、宣传部、秘书部、养护部。
  - 排序：喵喵号正序、喵喵号倒序。
- 成员卡片显示头像、昵称、喵喵号、身份标签、部门标签和进入箭头。
- 身份标签颜色：
  - 管理员：绿色。
  - 暑期志愿者：橙色。
  - 成员：蓝色。
- 列表浮动按钮“新增成员”进入现有新增账户页，并同步页面文案。
- 成员详情复用个人资料编辑页的表单体验：
  - 喵喵号只读。
  - 非管理员成员可编辑昵称、真实姓名、部门、年级、联系方式、身份和状态。
  - 管理员账号详情只读，不允许修改管理员信息。
  - 管理员账号不显示重置密码按钮；成员和暑期志愿者显示重置密码按钮。

### 地标点

- 不新增地标表，使用 `map_points.point_type = "landmark"` 与 `map_point_photos`。
- 新增管理员入口“新建地标点”。
- 新建/编辑页复用物资点表单布局与选点交互，但只保留：
  - 地标名称。
  - 地标位置。
  - 照片。
  - 地标说明，保存到 `description`，并同步为 `route_instruction`，用于路线说明和导航页展示。
- 选点页复用物资点选点逻辑，marker 使用 `地标.png`。
- 地标详情页复用物资点详情页的视觉结构，展示照片、名称、说明、地址、附近地标、路线说明和导航按钮。
- 地图 marker 继续走现有 `landmark` 点位逻辑，地图概要详情跳转改为 `/pages/landmarks/detail?landmark_id=<map_point_id>`。
- 物资点详情页“公共地点”文案改为“附近地标”。

## 后端边界

- 人员管理扩展现有 `/api/v1/admin/users`：
  - `GET /admin/users` 增加 `department`、`sort_by=meow_no`、`sort_order=asc|desc`。
  - `GET /admin/users/{user_id}` 获取详情。
  - `PATCH /admin/users/{user_id}` 更新非管理员账号信息。
  - `PATCH /admin/users/{user_id}/reset-password` 兼容文档路径，保留现有 `/password` 路径。
  - 重置密码和编辑均禁止目标账号为管理员。
- 地标新增 API：
  - `POST /admin/landmarks` 创建地标点。
  - `GET /admin/landmarks/{landmark_id}` 获取管理员可编辑详情。
  - `PATCH /admin/landmarks/{landmark_id}` 编辑地标点。
  - `DELETE /admin/landmarks/{landmark_id}` 软删除并隐藏 marker。
  - `GET /landmarks/{landmark_id}` 获取成员可见详情。

## 验证

- 后端测试覆盖：
  - 人员列表部门筛选和喵喵号排序。
  - 管理员可查看并编辑成员，不能编辑管理员。
  - 管理员可重置成员密码，不能重置管理员密码。
  - 管理员可创建、查看、编辑、软删除地标，成员可从地图看到地标 marker 和详情跳转。
- 前端验证：
  - `npm run type-check`。
  - 相关页面构建通过。
  - 人员、任务、猫咪列表搜索/筛选区固定，底部内容独立滚动。
