# 管理员人员管理与地标点实施计划

## 目标

完成管理员人员管理和地标点插入功能，并统一列表页顶部固定、内容区滚动的交互。

## 步骤

1. 后端测试先行
   - 为管理员用户列表增加部门筛选、喵喵号排序测试。
   - 为管理员成员详情编辑和管理员保护规则增加测试。
   - 为地标点创建、地图展示、详情跳转、编辑、删除增加测试。

2. 人员管理后端
   - 扩展管理员用户查询参数。
   - 增加详情与更新 schema。
   - 增加用户详情、资料更新、管理员保护和重置密码保护逻辑。
   - 补充 `/reset-password` 路由别名。

3. 地标后端
   - 新增地标 schema、service 和 router。
   - 复用 `map_points`、`map_point_photos`、`admin_operation_logs`。
   - 更新地图 summary 详情路径。

4. 人员管理前端
   - 扩展 `admin-users` API 类型与方法。
   - 新增人员管理列表页。
   - 新增管理员成员详情页。
   - 调整管理员入口与新增成员页文案。

5. 地标前端
   - 新增地标 API 类型与方法。
   - 新增地标新建/编辑、选点、详情页。
   - 更新页面路由。
   - 更新物资点详情“附近地标”文案。

6. 列表滚动统一
   - 调整任务列表、猫咪列表为顶部筛选固定、列表区域滚动。
   - 保持原有下拉刷新、加载更多和状态展示不变。

7. 进度与验证
   - 更新 `docs/开发进度.md`。
   - 运行后端相关测试。
   - 运行前端类型检查和可用构建/测试命令。
