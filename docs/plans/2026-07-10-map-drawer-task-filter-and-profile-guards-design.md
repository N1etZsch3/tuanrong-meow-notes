# 地图抽屉、任务筛选与资料保护设计

## 状态

已于 2026-07-10 与用户确认；本轮不处理定位授权。

## 目标与范围

1. 地图页使用“喵图”、底部导航使用“喵图 / 喵的”。
2. 抽屉最大展开位置按窗口高度比例计算，顶部更高且各屏幕比例一致；始终为悬浮底部导航保留可视区域。
3. 地图下拉将完成/未完成投喂任务合并为“任务”，移除选项说明。选中任务时仅请求一次全部任务 Marker；抽屉标题默认“未完成任务”，点击后通过单选菜单选择全部、已完成或未完成。状态切换只筛选内存中的同一批 Marker，地图 Marker、列表和点位数共享结果。
4. 任务详情删除独立“完成照片”卡片，保留任务动态记录详情里的照片。
5. 个人资料页在昵称、部门、联系方式或头像相对上次成功保存的快照发生变化后，返回时要求确认放弃修改。
6. 密码登录不再预先展示微信绑定确认。前端先提交未确认绑定的微信 code；已绑定且匹配的账号直接登录；仅后端明确返回“需要确认绑定”时才展示确认，再用新的 code 重试绑定。已绑定但微信不匹配继续显示原错误，不能误弹确认框。
7. 验证头像上传后保存的 URL 能通过另一账号可读取的成员资料接口返回。排查发现资料保存的是裸 COS URL，而当前私有读策略下跨账号加载可返回 403；新增回归测试，并将三处头像上传入口改为保存文件内容路由 URL。

## 技术方案

- 前端任务筛选维持现有 `/api/v1/map/points` 契约：任务外层过滤传 `point_types=task`、`business_types=feeding`，不扩展地图接口参数。缓存返回 Marker 后，由纯函数按 `status_key` 构造全部、已完成和未完成视图；状态切换不得触发地图请求。
- 抽屉 WXS 用 `windowHeight` 的比例计算最大顶部留白与高度，展开态保留底部 Tab 的安全间距。地图视口、筛选浮层和抽屉仍由同一 progress 同步更新。
- 未保存资料以保存快照和当前表单/头像 URL 比较；确认放弃后再调用返回。上传中的头像不等同于已保存的个人资料。
- 新增鉴权错误码 `WECHAT_BINDING_CONFIRMATION_REQUIRED`，仅用于“密码、验证码和微信 code 均有效，但账号尚未绑定且未确认”的状态；同步前端错误识别及鉴权接口文档。不新增数据库迁移。
- 头像上传仍产生持久 `user_avatar` 资产，但资料字段保存 `buildFileAssetContentUrl(asset_id, "avatar_profile")` 的后端内容路由，避免小程序跨账号直接请求私有 COS URL；展示层同时把可识别的旧 COS 资产 URL 转换为该内容路由。

## 非目标

- 不新增地图筛选接口参数、数据库字段或迁移。
- 不改变任务动态中的完成记录和照片查看能力。
