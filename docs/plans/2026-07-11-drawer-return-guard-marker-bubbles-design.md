# 抽屉动画、返回确认与 Marker 气泡设计

## 目标

在不改定位权限和地图接口契约的前提下，改善地图抽屉拖拽、筛选下拉、资料编辑返回保护与 Marker 标题气泡体验。

## 范围

1. 抽屉继续由 WXS 驱动，不在 `touchmove` 跨线程调用 Vue；降低拖动进度灵敏度、合并无变化的样式写入，并放慢释放后的抽屉回弹。原生地图继续维持预铺尺寸与无 resize 回弹，避免重新出现瓦片重载和中心漂移。
2. 地图筛选按钮展开时保持自身宽度，选项以相同宽度从正下方纵向展开；WXS 只控制向下的透明度、位移和箭头旋转，不再把按钮或命中层向右加宽。
3. 个人资料与管理员成员编辑页采用同一套“返回守卫”：微信小程序通过 `page-container` 的 `beforeleave` 消费右滑/返回；脏表单显示确认框，确认后只放行一次真实返回，取消后重新武装守卫。非微信平台保留当前自定义返回路径作为回退。
4. Marker 标题气泡改由“缩放门槛 + 当前地图视口内数量”共同决定：
   - 外部 POI 始终无标题气泡；选中业务 Marker 只显示自己的气泡。
   - 地图拖动/缩放中、筛选数据未就绪、无标记状态、缩放低于 `18` 时，不显示未选中 Marker 气泡。
   - 地图稳定后读取原生地图可见 region；在缓存 Marker 中计算当前视口的业务 Marker 数量。数量 `<= 6` 时显示视口内可显示的气泡；数量 `>= 8` 时收起，`7` 时保持上一次状态，防止边界闪烁。
   - 筛选切换后不再依赖“之前是否选中过点”的临时布尔状态；等新筛选数据与稳定 region 共同完成后重新判断。
   - 为实现缩放浮现，最多 6 个气泡使用地图安全 `cover-view` 覆盖层，依据 `toScreenLocation` 定位，在地图 `regionchange` 结束后淡入缩放。原生 Marker callout 不承担未选中气泡动画；它只保留选中 Marker 的即时标题兜底。

## 返回守卫细节

- 当前 `onBackPress` 不适用于微信小程序，`enableAlertBeforeUnload` 也不拦截右滑；`page-container` 是微信端唯一覆盖右滑、系统返回和 `navigateBack` 的合适入口。
- 守卫容器本身不可见、无业务遮罩，每页只实例化一个。`beforeleave` 首先消费默认退出：
  - 表单未变更时设置一次性放行标记，再调用 `uni.navigateBack()`。
  - 表单已变更时显示统一的“修改尚未保存，是否放弃？”；确认时设置放行标记并返回，取消时重新显示守卫。
- 管理员快照包括头像、昵称、真实姓名、部门、年级、联系方式、角色和状态；成员资料加载和保存成功都会更新快照。

## 测试与验收

- 先为抽屉 WXS 节流/回弹、筛选固定宽度、两页返回守卫及 Marker 可见区域规则新增失败测试。
- 纯函数测试覆盖缩放、视口数量、滞回、选中态和筛选切换；页面源码测试覆盖 WXS 和 `page-container` 接入。
- 前端全量 Vitest、类型检查和微信小程序构建必须通过。
- 微信开发者工具/真机需要验收：右滑取消/确认返回、管理员页同样保护、抽屉连续快慢拖拽、筛选只向下展开、Marker 在密集/稀疏/拖动后视口的气泡行为和缩放入场。

## 非目标

- 不修改位置授权、定位申请、地图后端筛选参数或数据库结构。
- 不恢复旧的图片型 `customCallout` 扇出实现。
