# 点位选中时抽屉单向展开设计

## 目标

地图动态内容区域处于折叠档时，用户选中业务 Marker 或解析成功的地图 POI 后，抽屉自动展开到图示详情高度；清除 Marker/POI 选中态时保持抽屉当前位置，不自动收缩。

## 当前状态

- 抽屉档位只保存在 `drawer.wxs` 的模块变量 `currentProgress` 中：`0` 为折叠、`1` 为普通、`2` 为约 `654rpx` 的详情高度、`3` 为最大展开。
- Vue 页面只接收 WXS 的拖拽中状态，不掌握或持续同步抽屉进度；这样可以避免拖拽期间跨逻辑层通信。
- Marker/POI 选中只更新 `selectedSummary`，没有向 WXS 发送自动展开请求。
- 清除选中态只清理详情和选中 Marker，本来就没有抽屉收缩逻辑。

## 方案

1. 页面新增递增的“详情抽屉可见请求”令牌，并通过 WXS change-prop 绑定到 `drawer.wxs`。令牌只表达一次“确保详情档可见”，不把实时抽屉进度同步回 Vue。
2. 监听 `selectedSummary`：每次写入有效详情时递增令牌，覆盖业务 Marker、搜索结果、待处理导航和解析成功的腾讯 POI。是否需要移动仍由 WXS 根据真实档位决定。
   - 详情变为空时不修改令牌，因此清除选中态不会触发抽屉回退。
3. WXS 收到新令牌后检查自己的真实 `currentProgress`：
   - 仅当 `currentProgress === 0` 时设置为 `2`，使用现有 `applyFinalProgress` 动画展开到约 `654rpx`。
   - 当前处于 `1`、`2`、`3` 或用户拖拽中时不改变抽屉位置。
4. 程序展开继续调用现有 `setDrawerResizeInProgress` 开始/结束通知，避免抽屉尺寸变化期间触发无效地图 region 同步。
5. 不修改地图中心移动、缩放、Marker 气泡、筛选或详情数据请求逻辑。

## 边界行为

- 点位详情接口失败或过期请求被取消时不会展开，因为没有产生有效 `selectedSummary`。
- 已经打开详情时切换其他点位，只更新内容和地图中心，不重复改变抽屉档位。
- 用户清除选中态后抽屉保持第二档或其手动调整后的档位；是否收缩完全由后续手势决定。
- 用户清除后手动折叠，再次选择点位时仍会收到新的令牌并展开。

## 测试与验收

- 页面源码测试覆盖只有有效 `selectedSummary` 才递增请求，清除函数不发送收缩指令。
- WXS 测试覆盖只从 `currentProgress === 0` 跳到 `2`，其他进度保持不变，并复用现有最终进度动画和 resize 保护。
- 回归运行地图页 Vitest、前端全量测试、类型检查和微信小程序构建。
- 微信开发者工具或真机检查：折叠后点击业务 Marker、POI；在普通/详情/最大档点击点位；点击地图空白清除选中；清除后重新选点；确认抽屉只在折叠状态单向展开且不会自动回退。

## 非目标

- 不修改当前已在实机验证有平滑过渡的地图聚焦动画。
- 不改变用户手动拖拽、吸附档位、筛选、搜索和点位详情内容。
