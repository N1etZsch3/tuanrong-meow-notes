# 地图平滑聚焦动画设计（修订）

## 目标

修复地图页点击业务 Marker、地图 POI 和“我的位置”后的视口移动：目标点需要以可见且丝滑的动画移动到视口中心，地图不应瞬移或明显闪烁，同时保留用户已经放大的查看尺度。

## 根因修正

上一版删除逻辑层补间，改为单次调用 `MapContext.moveToLocation({ longitude, latitude })`。该 API 只负责移动地图中心，没有 `duration`、`easing` 或动画完成回调，也未承诺地图中心平移动画；当前微信地图组件实际表现为瞬移。因此不能把它作为平滑聚焦方案。

旧补间同样不适合直接恢复：它在 320ms 内固定更新 12 步，理论频率仅约 37.5 FPS，并使用开头速度较快的 ease-out。每步还会触发 Vue/uni-app 到原生地图的数据同步以及地图 `regionchange` 后的气泡刷新，动画过短、步数不足和附带工作共同造成卡顿。

## 修订方案

1. 聚焦动画改为基于真实时间的受控中心补间，不再使用固定步数，也不再调用 `moveToLocation` 完成聚焦。
2. 动画时长按起终点距离自适应：
   - 最短 `420ms`，适合相邻点位。
   - 随距离线性增加，约 `1200m` 达到最长 `720ms`。
   - 小于中心同步容差的移动直接结束，不产生无意义动画。
3. 以约 `16ms` 为目标申请下一帧：运行环境存在 `requestAnimationFrame` 时使用它，否则使用单个 `setTimeout` 回退。每帧按 `Date.now()` 的真实经过时间计算进度；掉帧时直接追上正确进度，不积累固定步数队列。
4. 缓动使用 `easeInOutCubic`，降低起步和收尾速度，避免旧 ease-out 在第一段产生突跳。
5. Marker、解析成功的 POI 和“我的位置”继续复用同一聚焦入口：
   - 当前缩放低于 `18` 时只提升到 `18`，达到或超过 `18` 时保持原倍率。
   - 如需提升缩放，先只写入一次缩放，并从下一帧开始平移，避免同一轮同步同时重绘缩放和中心。
6. 新聚焦请求会取消旧帧任务，并以屏幕当前受控中心作为新起点；用户主动拖动地图会立即取消程序动画，拖动后的原生中心继续同步回来。

## 闪烁控制

- 动画期间保持 `<map>` 节点、尺寸、key、Marker 数组和筛选数据不变，只更新中心经纬度，不卸载或重建地图。
- 动画开始时只清空一次普通 Marker 气泡；程序产生的中间 `regionchange` 不重复调用 `getRegion`、`toScreenLocation` 或重新挂载气泡。
- 最后一帧只写入精确目标坐标，不再补一次 `moveToLocation`，避免结尾二次跳转。
- 动画结束并等待地图稳定后只刷新一次视口气泡；若期间发生新动画、用户拖动或页面离开，旧刷新任务失效。
- WXS 继续只负责抽屉和筛选的视图层动画。原生 `<map>` 的经纬度不是 WXS 可修改的元素样式，CSS 动画对小程序地图也无效，因此不把地图摄像机补间放入 WXS。

## 备选方案说明

- **逐帧调用 `MapContext.moveToLocation`：** 可以绕开 Vue 响应式写入，但会连续向原生地图发送未提供动画语义的异步命令，存在排队、乱序和闪动风险，不采用。
- **WebView 地图 SDK：** 可使用地图厂商的 `panTo`，但需要重写 Marker、POI、气泡、点击和定位交互，远超本次修复范围。

## 测试与验收

- 纯函数测试覆盖距离到 `420–720ms` 时长的换算、`easeInOutCubic` 边界和中点、时间进度插值。
- 页面源码测试覆盖 Marker、POI 和“我的位置”共用同一补间；不再使用 `moveToLocation`；下一帧可取消；拖动会取消动画；动画期间抑制气泡重复刷新，结束后只恢复一次。
- 运行地图页 Vitest、前端全量测试、TypeScript 类型检查和微信小程序构建。
- 微信开发者工具或真机必须检查：近距离/远距离、低于/高于 18 倍率、快速连续点击、动画中拖动、远距离瓦片加载以及气泡恢复，重点观察是否瞬移、抖动、白闪或结尾二次跳转。

## 非目标

- 不修改位置权限申请、定位服务、地图数据接口、筛选参数或 Marker 气泡显示规则。
- 不更换地图引擎，不改变地图点击后的业务详情、导航或任务状态行为。
