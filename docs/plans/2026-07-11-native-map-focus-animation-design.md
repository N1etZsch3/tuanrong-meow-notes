# 地图原生聚焦动画设计

## 目标

修复地图页点击业务 Marker、地图 POI 和“我的位置”后，地图移动到目标点作为视口中心时的卡顿问题，同时保留用户已经放大的查看尺度。

## 当前问题

当前平滑回中由逻辑层在 320ms 内分 12 次修改地图 `longitude` 和 `latitude`。理论更新频率约为 37.5 FPS，而且每一步都要经过 Vue 响应式更新、uni-app 数据同步和原生地图属性更新。缩放与中心点同时变化时，跨线程逐帧更新更容易产生停顿。

## 方案

1. 删除 JS 定时器驱动的经纬度补间动画，平滑聚焦统一调用原生 `MapContext.moveToLocation({ longitude, latitude })`。地图位移动画由原生地图组件执行，页面只发出一次移动指令。
2. Marker、解析成功的 POI 和“我的位置”统一走同一个视口聚焦入口：
   - 当前缩放级别低于 `18` 时，将目标缩放级别提升到 `18`。
   - 当前缩放级别达到或超过 `18` 时保持原级别，不强制缩小。
   - 随后调用原生地图移动，将目标经纬度置于视口中心。
3. 使用递增的移动序号标识当前聚焦请求。快速连续点击多个目标时，过期回调不得覆盖最后一次目标的中心状态。
4. 原生移动成功后只同步一次受控中心坐标；原生能力不可用或调用失败时，降级为一次性写入受控中心坐标，不恢复逐帧补间。
5. 用户主动拖动地图时使待处理的聚焦回调失效，避免拖动结束后被旧目标拉回。
6. 保留现有 `regionchange` 生命周期：地图移动期间收起普通 Marker 气泡，地图稳定后按缩放级别和视口内 Marker 数量重新计算气泡。

## 备选方案说明

- `requestAnimationFrame` 或提高 JS 补间步数只能增加逻辑层到原生层的数据同步次数，无法消除跨层逐帧更新，因此不采用。
- WXS 无法可靠调用 `MapContext` 或直接驱动原生地图经纬度属性，不适合承担地图回中动画。
- `includePoints` 会同时改变视野范围，单点场景也有兼容性差异，不符合“目标点居中并按既定规则保留缩放”的交互。

## 测试与验收

- 纯函数测试覆盖：当前缩放低于 `18` 返回 `18`；当前缩放高于 `18` 保持原值。
- 页面测试覆盖：Marker、POI 和“我的位置”复用原生聚焦路径；旧动画步数、动画定时器和逐帧中心更新已移除；失败回退只写入一次中心。
- 运行地图页相关 Vitest、前端全量测试、TypeScript 类型检查和微信小程序构建。
- 微信开发者工具或真机手工检查：低倍率和高倍率下分别点击 Marker、POI、“我的位置”；快速连续点击不同 Marker；动画期间主动拖动地图；确认最终中心、缩放级别和气泡恢复均正确。

## 非目标

- 不修改位置权限申请、定位服务、地图数据接口、筛选参数或 Marker 气泡显示策略。
- 不改变地图点击后的业务详情、导航或任务状态行为。
