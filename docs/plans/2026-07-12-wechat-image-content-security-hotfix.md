# 微信图片内容安全审核热修计划

## 目标

- 版本：1.1.2，分支 `hotfix/1.1.2-content-security`，基于 `main`。
- 修复微信审核指出的头像 UGC 图片没有内容安全机制问题。
- 仅用户头像由后端调用微信 `mediaCheckAsync` 2.0，先审后发；普通业务图片保持原有上传完成即可使用的流程。
- 头像一次提交后可离开页面，微信回调通过时服务器自动替换；拒绝时保留旧头像。

## 实现

1. `file_assets` 增加审核状态、微信 trace、建议、标签、错误码和时间字段；历史图片迁移为 `legacy`。
2. `user_profiles` 增加最近头像审核资产、状态和更新时间，防止旧回调覆盖新头像。
3. 用户头像上传前前端调用 `uni.login`，后端用 code 换取本次上传者 OpenID；普通图片不获取或提交 `wechat_code`，AppSecret 和 access token 只在服务端使用。
4. COS 变体上传后使用预签名 URL 提交微信；待审响应不返回可发布 URL。
5. 微信消息服务器回调使用 HTTPS 明文模式，校验 SHA-1 签名并按 trace 幂等更新。
6. `pass` 开放资产并自动应用最新头像；`risky/review` 均拒绝；接口错误关闭失败。
7. 资料和管理员接口拒绝客户端写入新的任意 `avatar_url`，堵住旁路。
8. 非头像业务上传由公共前端封装直接返回后端生成的可用 URL，不进行审核轮询。
9. 业务写入接口再次校验图片资产的存在性、用途和调用者权限；任务、地标、物资和药品接口仍拒绝任意客户端 URL。
10. 历史图片和无需微信审核的普通图片使用 `legacy` 状态，可按用途和权限绑定；药品旧协议的 URL 会反查服务器资产并保存规范 URL。

## 部署前配置

- 生产环境设置 `CATMAP_WECHAT_CONTENT_SECURITY_MODE=enforced`。
- 配置随机 `CATMAP_WECHAT_CONTENT_SECURITY_CALLBACK_TOKEN`，不得提交或输出。
- 微信公众平台消息服务器 URL：`https://trmx.fun/api/v1/wechat/content-security/events`。
- 消息加密方式选择明文模式，token 与生产环境一致。
- 在小程序后台 UGC 场景声明中记录用户头像使用平台内容安全 API 先审后发。

## 验证与回滚

- 自动化覆盖：v2 请求参数、XML/JSON 回调、头像待审不可访问、通过自动换头像、拒绝保留旧头像、回调签名/幂等、普通图片立即可用和 URL 旁路。
- 真机覆盖：正常头像、微信官方风险测试头像、页面离开后自动生效、审核失败一次性提示，以及任务/地标/物资/药品图片立即上传使用。
- 部署前备份数据库和环境；迁移可删除新增列回滚。
- 若微信服务异常，保持 `enforced` 并阻止新图片发布，不允许自动放行；紧急回滚应恢复上一版代码和数据库备份。
