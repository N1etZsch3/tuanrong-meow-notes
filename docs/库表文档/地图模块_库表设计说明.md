# 校园猫协地图任务系统｜地图模块库表设计说明

> 文档版本：V1.0  
> 模块名称：地图模块  
> 适用阶段：MVP / 第一版数据库设计  
> 推荐数据库：PostgreSQL + PostGIS  
> 后端建议：FastAPI + SQLAlchemy + Alembic  
> 设计目标：支撑地图首页的看点、搜点、选点、预览、导航、底部动态卡片、管理员长按插点、点位编辑与拖拽修改位置等能力。

---

# 1. 模块定位

地图模块是系统首页，也是校园猫协内部协作的空间工作台。

地图模块数据库设计只负责：

1. 校园范围配置。
2. 校园区域划分。
3. 地图点位的空间位置。
4. Marker 图标和显示规则。
5. 点位照片、路线图、入口图。
6. 点位搜索所需的基础文本。
7. 点位可见性、状态和管理员编辑记录。
8. 与任务、猫咪、物资模块的空间关联。

地图模块不负责保存完整业务详情：

| 业务内容 | 所属模块 | 对应表 |
|---|---|---|
| 任务标题、状态、人数、截止时间、接取记录 | 任务模块 | `tasks`、`task_participants` |
| 猫咪名称、健康状态、绝育状态、观察记录 | 猫咪库模块 | `cats`、`cat_map_points`、`cat_observation_records` |
| 物资种类、库存、管理人、使用说明 | 物资模块 | `supply_points`、`supply_point_items` |
| 地图经纬度、位置说明、路线说明、Marker 展示 | 地图模块 | `map_points`、`map_point_photos`、`map_marker_configs` |

核心原则：

```text
map_points 只保存空间点位和地图展示信息。
tasks / cats / supply_points 保存各自业务详情。
地图页需要的卡片信息由后端接口实时聚合返回。
```

# 3. 通用约定

## 3.1 主键

推荐所有业务表统一使用：

```sql
id UUID PRIMARY KEY DEFAULT gen_random_uuid()
```

需要启用：

```sql
CREATE EXTENSION IF NOT EXISTS pgcrypto;
CREATE EXTENSION IF NOT EXISTS postgis;
```

## 3.2 时间字段

核心业务表统一包含：

| 字段名 | 类型 | 说明 |
|---|---|---|
| `created_at` | timestamptz | 创建时间 |
| `updated_at` | timestamptz | 更新时间 |
| `deleted_at` | timestamptz | 软删除时间，可为空 |

照片表、日志表等只追加不修改的表可以不设置 `updated_at`。

## 3.3 空间字段

经纬度字段保留普通数值，方便前端直接读取：

```sql
lng numeric(10,7)
lat numeric(10,7)
```

空间查询字段使用 PostGIS：

```sql
geom geography(Point, 4326)
boundary geography(Polygon, 4326)
```

说明：

1. `lng` / `lat` 负责接口返回和前端展示。
2. `geom` 负责距离计算、范围查询、附近点位查询。
3. 更新经纬度时必须同步更新 `geom`。

# 4. campuses 校园配置表

## 4.1 表名

```text
campuses
```

## 4.2 作用

保存校园地图默认配置，用于地图初始化、限制地图范围、计算点位是否属于校园范围。

第一版系统只服务一个学校，但仍建议保留 `campuses` 表，后续可以支持多校区或新校区。

## 4.3 字段设计

| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | uuid | PK | 校园 ID |
| `name` | varchar(128) | not null | 校园名称 |
| `code` | varchar(64) | unique, nullable | 校园编码，例如 `hbnu_main` |
| `center_lng` | numeric(10,7) | not null | 默认中心经度 |
| `center_lat` | numeric(10,7) | not null | 默认中心纬度 |
| `center_point` | geography(Point,4326) | nullable | 默认中心空间点 |
| `boundary` | geography(Polygon,4326) | nullable | 校园边界 |
| `default_zoom` | int | default 16 | 默认缩放级别 |
| `min_zoom` | int | nullable | 最小缩放级别 |
| `max_zoom` | int | nullable | 最大缩放级别 |
| `map_provider` | varchar(32) | default 'amap' | 地图供应商，第一版为高德地图 |
| `is_active` | boolean | default true | 是否启用 |
| `created_at` | timestamptz | not null | 创建时间 |
| `updated_at` | timestamptz | not null | 更新时间 |

## 4.4 主要索引

```sql
CREATE INDEX idx_campuses_active
ON campuses (is_active);

CREATE INDEX idx_campuses_boundary
ON campuses
USING GIST (boundary);
```

## 4.5 使用场景

1. 地图首页初始化。
2. 默认定位到校园中心。
3. 限制地图拖动范围。
4. 查询校园范围内业务点位。
5. 管理员新增点位时校验坐标是否在校园范围内。

# 6. map_points 地图点位主表

## 6.1 表名

```text
map_points
```

## 6.2 作用

保存所有地图点位的空间信息和地图展示信息。

统一承载：

1. 任务点。
2. 猫咪点。
3. 物资点。
4. 固定喂食点。
5. 临时观察点。
6. 地标点。
7. 路线提示点，后期扩展。

## 6.3 设计原则

`map_points` 只保存地图层面的信息：

```text
点位在哪里
点位叫什么
点位怎么走
点位是否可见
Marker 怎么展示
缩放到什么级别显示标题或预览卡片
```

不在 `map_points` 中保存完整任务详情、猫咪健康信息或物资库存。

## 6.4 字段设计

| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | uuid | PK | 点位 ID |
| `campus_id` | uuid | FK -> campuses.id | 所属校园 |
| `area_id` | uuid | FK -> campus_areas.id, nullable | 所属区域 |
| `point_type` | varchar(32) | not null | 点位类型 |
| `point_scope` | varchar(32) | not null | 长期点位或临时点位 |
| `name` | varchar(128) | not null | 点位名称 |
| `subtitle` | varchar(255) | nullable | 副标题 |
| `description` | text | nullable | 点位说明 |
| `location_name` | varchar(128) | nullable | 位置名称，例如体育馆西侧 |
| `location_detail` | text | nullable | 详细位置描述 |
| `lng` | numeric(10,7) | not null | 经度 |
| `lat` | numeric(10,7) | not null | 纬度 |
| `geom` | geography(Point,4326) | not null | PostGIS 地理点 |
| `amap_poi_id` | varchar(128) | nullable | 高德 POI ID |
| `amap_address` | varchar(255) | nullable | 高德逆地理地址 |
| `route_instruction` | text | nullable | 校园内部路线说明 |
| `landmark_hint` | text | nullable | 附近地标提示 |
| `entrance_hint` | text | nullable | 入口说明 |
| `icon_key` | varchar(64) | FK -> map_marker_configs.marker_key, nullable | Marker 图标 key |
| `cover_photo_id` | uuid | FK -> map_point_photos.id, nullable | 点位封面图 ID |
| `display_level` | int | default 0 | 显示优先级，数值越高越优先展示 |
| `label_min_zoom` | int | default 17 | 地图缩放到多少级后显示标题 |
| `preview_enabled` | boolean | default false | 是否允许地图贴点预览卡片 |
| `preview_min_zoom` | int | default 17 | 地图缩放到多少级后显示预览卡片 |
| `visibility` | varchar(32) | default 'public' | 可见状态 |
| `status` | varchar(32) | default 'active' | 点位状态 |
| `created_by` | uuid | FK -> users.id, nullable | 创建人 |
| `updated_by` | uuid | FK -> users.id, nullable | 更新人 |
| `created_at` | timestamptz | not null | 创建时间 |
| `updated_at` | timestamptz | not null | 更新时间 |
| `deleted_at` | timestamptz | nullable | 软删除时间 |

## 6.5 point_type 枚举

| 值 | 说明 |
|---|---|
| `task` | 任务点 |
| `cat` | 猫咪点 |
| `supply` | 物资点 |
| `feeding` | 固定喂食点 |
| `observation` | 临时观察点 |
| `landmark` | 地标点 |
| `route_hint` | 路线提示点，后期扩展 |

说明：

```text
紧急任务不单独作为 point_type。
紧急任务仍然是 point_type = 'task'，通过 tasks.task_type = 'emergency' 区分。
```

## 6.6 point_scope 枚举

| 值 | 说明 |
|---|---|
| `long_term` | 长期点位，例如常驻猫咪点、物资点、固定喂食点 |
| `temporary` | 临时点位，例如日常任务、紧急任务、临时观察点 |

## 6.7 visibility 枚举

| 值 | 说明 |
|---|---|
| `public` | 普通成员可见 |
| `admin_only` | 仅管理员可见 |
| `hidden` | 隐藏，不在普通地图展示 |

## 6.8 status 枚举

| 值 | 说明 |
|---|---|
| `active` | 正常 |
| `inactive` | 暂停使用 |
| `archived` | 归档 |
| `deleted` | 软删除 |

## 6.9 展示规则说明

### 6.9.1 地图缩小时

```text
只显示 Marker 图标。
不显示点位标题。
不显示图片预览卡片。
必要时按 display_level 选择高优先级点位展示。
```

### 6.9.2 地图放大后

当地图缩放级别满足：

```text
current_zoom >= label_min_zoom
```

可以显示点位标题。

当满足：

```text
preview_enabled = true
AND current_zoom >= preview_min_zoom
```

可以显示贴点预览卡片。

### 6.9.3 优先级建议

| 点位类型 | display_level 建议 |
|---|---:|
| 紧急任务点 | 100 |
| 进行中任务点 | 80 |
| 可接取日常任务点 | 60 |
| 常驻猫咪点 | 40 |
| 物资点 | 30 |
| 地标点 | 10 |

## 6.10 主要索引

```sql
CREATE INDEX idx_map_points_geom
ON map_points
USING GIST (geom);

CREATE INDEX idx_map_points_campus_type_status
ON map_points (campus_id, point_type, status);

CREATE INDEX idx_map_points_area
ON map_points (area_id);

CREATE INDEX idx_map_points_visible
ON map_points (visibility, status);

CREATE INDEX idx_map_points_active_public
ON map_points (campus_id, point_type)
WHERE status = 'active' AND visibility = 'public';

CREATE INDEX idx_map_points_display
ON map_points (campus_id, display_level DESC, preview_enabled, preview_min_zoom);

CREATE INDEX idx_map_points_created_by
ON map_points (created_by);
```

## 6.11 搜索索引

推荐启用 `pg_trgm`：

```sql
CREATE EXTENSION IF NOT EXISTS pg_trgm;

CREATE INDEX idx_map_points_name_trgm
ON map_points USING gin (name gin_trgm_ops);

CREATE INDEX idx_map_points_location_name_trgm
ON map_points USING gin (location_name gin_trgm_ops);
```

## 6.12 使用场景

1. 地图首页 Marker 查询。
2. 点击 Marker 展示底部点位简略卡片。
3. 搜索点位、地标、地址、位置说明。
4. 导航到点位。
5. 管理员长按地图插点。
6. 管理员拖拽修改点位位置。
7. 管理员隐藏、恢复、归档点位。

# 8. map_marker_configs Marker 配置表

## 8.1 表名

```text
map_marker_configs
```

## 8.2 作用

统一管理地图 Marker 的图标、颜色、默认展示层级和缩放预览规则。

它解决的问题是：

1. 不同点位类型使用不同图标。
2. 紧急任务点显示优先级更高。
3. 缩放到不同等级时，是否显示标题和预览卡片。
4. 前端可以根据配置动态渲染 Marker。

## 8.3 字段设计

| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | uuid | PK | 配置 ID |
| `marker_key` | varchar(64) | unique, not null | 图标 key |
| `point_type` | varchar(32) | not null | 点位类型 |
| `business_type` | varchar(32) | nullable | 业务细分类型 |
| `label` | varchar(64) | not null | 显示名称 |
| `icon_url` | varchar(512) | nullable | 图标地址 |
| `icon_svg` | text | nullable | SVG 内容 |
| `color` | varchar(32) | nullable | 颜色 |
| `z_index` | int | default 0 | 地图层级 |
| `default_visible` | boolean | default true | 默认是否展示 |
| `default_label_min_zoom` | int | default 17 | 默认标题显示缩放阈值 |
| `default_preview_enabled` | boolean | default false | 默认是否启用预览卡片 |
| `default_preview_min_zoom` | int | default 17 | 默认预览卡片缩放阈值 |
| `icon_width` | int | nullable | 图标宽度，px |
| `icon_height` | int | nullable | 图标高度，px |
| `anchor_x` | int | nullable | 图标锚点 x |
| `anchor_y` | int | nullable | 图标锚点 y |
| `sort_order` | int | default 0 | 排序 |
| `created_at` | timestamptz | not null | 创建时间 |
| `updated_at` | timestamptz | not null | 更新时间 |

## 8.4 示例配置

| marker_key | point_type | business_type | label | z_index | default_preview_enabled | default_preview_min_zoom |
|---|---|---|---|---:|---|---:|

# 11. v_map_search_items 地图搜索视图

## 11.1 作用

地图搜索需要匹配：

1. 点位名称。
2. 地标。
3. 校园区域。
4. 地址和位置说明。
5. 路线说明。
6. 任务标题。
7. 猫咪名称和别名。
8. 物资点名称。
9. 物资名称。

如果直接在 service 层写多张表搜索，后期维护会比较乱。

建议建立搜索视图：

```text
v_map_search_items
```

第一版可以用普通视图；数据量变大后可以改为物化视图。

## 11.2 视图字段建议

| 字段名 | 类型 | 说明 |
|---|---|---|
| `result_type` | varchar | 结果类型，如 map_point、task、cat、supply、area |
| `map_point_id` | uuid | 对应地图点位 ID，可为空 |
| `business_id` | uuid | 对应业务对象 ID |
| `point_type` | varchar | 点位类型 |
| `business_type` | varchar | 业务细分类型 |
| `title` | varchar | 搜索结果标题 |
| `subtitle` | text | 搜索结果副标题 |
| `search_text` | text | 搜索匹配文本 |
| `lng` | numeric | 经度 |
| `lat` | numeric | 纬度 |
| `geom` | geography(Point,4326) | 空间点 |
| `display_level` | int | 展示优先级 |

## 11.3 视图示例 SQL

```sql
CREATE OR REPLACE VIEW v_map_search_items AS
SELECT
  'map_point'::varchar AS result_type,
  mp.id AS map_point_id,
  mp.id AS business_id,
  mp.point_type,
  NULL::varchar AS business_type,
  mp.name AS title,
  mp.location_name AS subtitle,
  concat_ws(' ',
    mp.name,
    mp.subtitle,
    mp.description,
    mp.location_name,
    mp.location_detail,
    mp.route_instruction,
    mp.landmark_hint,
    mp.entrance_hint,
    mp.amap_address
  ) AS search_text,
  mp.lng,
  mp.lat,
  mp.geom,
  mp.display_level
FROM map_points mp
WHERE mp.status = 'active'
  AND mp.visibility = 'public'
  AND mp.deleted_at IS NULL

UNION ALL

SELECT
  'task'::varchar AS result_type,
  t.map_point_id,
  t.id AS business_id,
  'task'::varchar AS point_type,
  t.task_type AS business_type,
  t.title AS title,
  mp.location_name AS subtitle,
  concat_ws(' ',
    t.title,
    t.description,
    t.task_type,
    t.status,
    mp.name,
    mp.location_name,
    mp.location_detail,
    mp.route_instruction
  ) AS search_text,
  mp.lng,
  mp.lat,
  mp.geom,
  mp.display_level
FROM tasks t
JOIN map_points mp ON mp.id = t.map_point_id
WHERE t.is_public = true
  AND t.deleted_at IS NULL
  AND mp.deleted_at IS NULL

UNION ALL

SELECT
  'cat'::varchar AS result_type,
  cmp.map_point_id,
  c.id AS business_id,
  'cat'::varchar AS point_type,
  cmp.relation_type AS business_type,
  c.name AS title,
  mp.name AS subtitle,
  concat_ws(' ',
    c.name,
    c.coat_color,
    c.personality_description,
    mp.name,
    mp.location_name,
    mp.location_detail,
    mp.route_instruction
  ) AS search_text,
  mp.lng,
  mp.lat,
  mp.geom,
  mp.display_level
FROM cat_map_points cmp
JOIN cats c ON c.id = cmp.cat_id
JOIN map_points mp ON mp.id = cmp.map_point_id
WHERE c.deleted_at IS NULL
  AND cmp.deleted_at IS NULL
  AND mp.deleted_at IS NULL

UNION ALL

SELECT
  'supply'::varchar AS result_type,
  sp.map_point_id,
  sp.id AS business_id,
  'supply'::varchar AS point_type,
  NULL::varchar AS business_type,
  sp.name AS title,
  sp.summary AS subtitle,
  concat_ws(' ',
    sp.name,
    sp.summary,
    sp.remark,
    mp.name,
    mp.location_name,
    mp.location_detail,
    mp.route_instruction
  ) AS search_text,
  mp.lng,
  mp.lat,
  mp.geom,
  mp.display_level
FROM supply_points sp
JOIN map_points mp ON mp.id = sp.map_point_id
WHERE sp.is_public = true
  AND sp.deleted_at IS NULL
  AND mp.deleted_at IS NULL

UNION ALL

SELECT
  'area'::varchar AS result_type,
  NULL::uuid AS map_point_id,
  ca.id AS business_id,
  'landmark'::varchar AS point_type,
  ca.area_type AS business_type,
  ca.name AS title,
  ca.description AS subtitle,
  concat_ws(' ', ca.name, ca.area_type, ca.description) AS search_text,
  ca.center_lng AS lng,
  ca.center_lat AS lat,
  ca.center_point AS geom,
  5 AS display_level
FROM campus_areas ca
WHERE ca.is_visible = true;
```

## 11.4 搜索实现建议

MVP 可以使用：

```sql
SELECT *
FROM v_map_search_items
WHERE search_text ILIKE '%' || :keyword || '%'
ORDER BY display_level DESC
LIMIT 20;
```

如果使用 `pg_trgm`，可以升级为相似度搜索。

# 13. 管理员地图操作的数据流程

## 13.1 管理员长按插点

### 流程

```text
管理员长按地图
  ↓
前端获得 lng / lat
  ↓
弹出快速插点菜单
  ↓
选择添加任务点 / 猫咪点 / 物资点 / 地标点
  ↓
填写点位表单
  ↓
创建 map_points
  ↓
根据类型创建或跳转对应业务表单
  ↓
写入 admin_operation_logs
```

### 添加任务点事务

```text
BEGIN
  INSERT map_points(point_type='task', point_scope='temporary')
  INSERT tasks(map_point_id=...)
  INSERT task_activity_logs(...)
  INSERT notifications(...)
  INSERT admin_operation_logs(...)
COMMIT
```

### 添加猫咪点事务

```text
BEGIN
  INSERT map_points(point_type='cat', point_scope='long_term')
  INSERT cats 或选择已有 cats
  INSERT cat_map_points(cat_id, map_point_id, relation_type='resident')
  INSERT admin_operation_logs(...)
COMMIT
```

### 添加物资点事务

```text
BEGIN
  INSERT map_points(point_type='supply', point_scope='long_term')
  INSERT supply_points(map_point_id=...)
  INSERT supply_point_items(...)
  INSERT admin_operation_logs(...)
COMMIT
```

### 添加地标点事务

```text
BEGIN
  INSERT map_points(point_type='landmark', point_scope='long_term')
  INSERT admin_operation_logs(...)
COMMIT
```

## 13.2 管理员拖拽修改点位

### 流程

```text
管理员点击 Marker
  ↓
点击编辑点位
  ↓
Marker 进入可拖拽状态
  ↓
拖拽到新位置
  ↓
前端显示新经纬度
  ↓
管理员点击保存
  ↓
后端更新 map_points.lng / lat / geom
  ↓
写入 admin_operation_logs
  ↓
地图刷新点位位置
```

### 更新 SQL 示例

```sql
UPDATE map_points
SET
  lng = :lng,
  lat = :lat,
  geom = ST_SetSRID(ST_MakePoint(:lng, :lat), 4326)::geography,
  updated_by = :admin_id,
  updated_at = now()
WHERE id = :point_id
  AND deleted_at IS NULL;
```

### 操作日志 before_data 示例

```json
{
  "lng": 114.1234567,
  "lat": 30.1234567,
  "geom": "POINT(114.1234567 30.1234567)"
}
```

### 操作日志 after_data 示例

```json
{
  "lng": 114.1234999,
  "lat": 30.1234999,
  "geom": "POINT(114.1234999 30.1234999)"
}
```

## 13.3 管理员隐藏或恢复点位

隐藏点位不物理删除。

```sql
UPDATE map_points
SET
  visibility = 'hidden',
  updated_by = :admin_id,
  updated_at = now()
WHERE id = :point_id;
```

恢复点位：

```sql
UPDATE map_points
SET
  visibility = 'public',
  updated_by = :admin_id,
  updated_at = now()
WHERE id = :point_id;
```

归档点位：

```sql
UPDATE map_points
SET
  status = 'archived',
  updated_by = :admin_id,
  updated_at = now()
WHERE id = :point_id;
```

# 17. 设计自检

## 17.1 是否重复保存业务数据？

没有。

`map_points` 只保存空间位置和地图展示信息。

任务详情、猫咪详情、物资详情仍然在各自业务表中。

## 17.2 是否能支持地图缩放预览？

可以。

相关字段：

```text
display_level
label_min_zoom
preview_enabled
preview_min_zoom
cover_photo_id
```

## 17.3 是否能支持搜索？

可以。

MVP 可以通过：

```text
v_map_search_items + ILIKE
```

后期可以升级为：

```text
pg_trgm 相似度搜索
全文搜索
搜索缓存
```

## 17.4 是否能支持导航？

可以。

相关字段：

```text
lng
lat
geom
amap_poi_id
amap_address
route_instruction
landmark_hint
entrance_hint
```

相关图片：

```text
map_point_photos.photo_type = route / entrance / scene
```

## 17.5 是否能支持管理员长按插点？

可以。

核心写入：

```text
map_points
map_point_photos
admin_operation_logs
```

再根据点位类型挂接：

```text
tasks
cats + cat_map_points
supply_points + supply_point_items
```

## 17.6 是否能支持管理员拖拽修改位置？

可以。

修改字段：

```text

map_points.lat
map_points.geom
```

并写入：

```text
admin_operation_logs.before_data
after_data
```

## 17.7 是否能支持底部动态内容区？

可以。

底部动态内容区不需要单独建表。

它根据当前状态实时查询：

| 状态 | 查询来源 |
|---|---|
| 最新任务 | `tasks` + `map_points` |
| 我的进行中任务 | `task_participants` + `tasks` + `map_points` |
| 搜索结果 | `v_map_search_items` |
| 点位简略卡片 | `map_points` + 对应业务表 |

# 20. 2026-06-30 腾讯地图 POI 关联字段补充

`map_points` 新增腾讯 POI 关联字段，用于保存管理员选点时关联到的公共 POI 信息。

| 字段名 | 类型 | 说明 |
|---|---|---|
| `tencent_poi_id` | varchar(128) | 腾讯 POI ID |
| `tencent_poi_name` | varchar(128) | 腾讯 POI 名称 |
| `tencent_poi_address` | varchar(255) | 腾讯 POI 地址 |
| `tencent_poi_category` | varchar(128) | 腾讯 POI 分类 |
| `tencent_poi_lng` | numeric(10,7) | 腾讯 POI 经度 |
| `tencent_poi_lat` | numeric(10,7) | 腾讯 POI 纬度 |
| `tencent_poi_distance_meters` | int | 自定义任务点坐标到 POI 的距离 |
| `tencent_poi_match_method` | varchar(32) | 匹配来源，如 `search`、`poi_tap`、`nearby`、`admin_selected` |

设计规则：

1. `map_points.lng` / `map_points.lat` / `map_points.geom` 仍然是真实业务点位坐标。
2. 腾讯 POI 字段只用于展示、搜索、选点辅助和位置说明，不替代真实点位坐标。
3. 管理员可以关联、切换或取消腾讯 POI；取消时上述字段置空。
4. `campuses.map_provider` 在当前实现中更新为 `tencent`。

迁移：`20260630_0010_add_tencent_map_poi_fields.py`。
