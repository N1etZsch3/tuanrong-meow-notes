# 校园猫协地图任务系统｜猫咪库模块库表设计文档

> 版本：V1.1 管理员录入流程调整版  
> 模块名称：猫咪库模块  
> 数据库：PostgreSQL + PostGIS  
> 后端建议：FastAPI + SQLAlchemy + Alembic  
> 本次调整重点：删除独立亲人程度字段；新增 `story`、`resident_area_text`；档案状态加入 `waiting_adoption`；性格改为标签制；每只猫最多 5 张照片；V1 草稿暂存由前端本地实现，不强制新增草稿表。

---

# 1. 模块数据定位

猫咪库模块保存猫咪相关业务数据，包括：

1. 猫咪基础档案。
2. 猫咪照片。
3. 猫咪别名。
4. 性格标签。
5. 常驻区域文字描述。
6. 地图点位关联。
7. 观察记录。
8. 健康记录。
9. 用户收藏。
10. 与任务模块、地图模块的关联信息。

地图坐标和路线说明仍由地图模块的 `map_points`、`map_point_photos` 负责；猫咪库只保存猫咪业务档案和猫咪与地图点位的关联关系。

---

# 2. 设计原则

## 2.1 `cats` 保存当前档案摘要

`cats` 主表保存猫咪当前最常用的展示信息，例如名称、主头像、花色、性别、绝育状态、健康状态、档案状态、性格标签、常驻区域文字描述、猫咪故事和注意事项。

列表页和详情页优先读取 `cats` 主表，避免每次都聚合大量历史记录。

## 2.2 性格标签替代亲人程度

V1 不再使用独立 `friendliness_level` 字段。

亲人、怕人、胆小、警惕、有攻击性等统一进入：

```text
cats.personality_tags
```

每个标签最少 1 个字，最多 4 个字，支持管理员自定义。

## 2.3 常驻区域文字优先

V1 新增：

```text
cats.resident_area_text
```

用于保存管理员录入的自然语言常驻区域，例如：

```text
学生宿舍区东侧草坪
教学楼 A 后方台阶
图书馆后门
```

`primary_area_id` 可保留为结构化区域字段，但 V1 页面优先展示 `resident_area_text`。

## 2.4 地图点位可选

猫咪可以没有精确地图点位。

当管理员在新增猫咪第 2 步跳过点位绑定时：

```text
cats.primary_map_point_id = null
```

管理员端可以提示“暂未绑定地图点位”。成员端仍展示 `resident_area_text`。

## 2.5 照片使用统一文件能力

猫咪照片由统一文件上传模块处理压缩、COS 上传、缩略图生成。猫咪库只保存文件 URL、缩略图 URL、类型和排序信息。

每只猫 V1 最多 5 张有效照片，至少 1 张主头像照片。

## 2.6 草稿机制不强制落库

V1 支持管理员录入猫咪草稿暂存，但优先由前端实现。

数据库层面不新增 `cat_drafts` 表。

后续如果需要多端恢复、多人协作、服务端草稿管理，可以再扩展服务端草稿表。

# 4. `cats` 猫咪档案主表

## 4.1 表名

```text
cats
```

## 4.2 作用

保存猫咪基础档案和当前状态摘要。

## 4.3 字段设计

| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | uuid | PK | 猫咪 ID |
| `name` | varchar(64) | not null | 猫咪名称 |
| `avatar_url` | varchar(512) | nullable | 主头像展示 URL，列表页优先使用 |
| `avatar_thumbnail_url` | varchar(512) | nullable | 主头像缩略图 URL，建议列表页使用 |
| `coat_color` | varchar(64) | not null | 花色，如橘猫、狸花、奶牛、黑猫 |
| `sex` | varchar(32) | not null, default 'unknown' | 性别 |
| `neuter_status` | varchar(32) | not null, default 'unknown' | 绝育状态 |
| `health_status` | varchar(32) | not null, default 'unknown' | 当前健康状态 |
| `status` | varchar(32) | not null, default 'active' | 档案状态 |
| `resident_area_text` | varchar(128) | not null | 常驻区域文字描述 |
| `primary_area_id` | uuid | FK -> campus_areas.id, nullable | 结构化常驻区域，V1 可为空 |
| `primary_map_point_id` | uuid | FK -> map_points.id, nullable | 主要常驻地图点位，可为空 |
| `last_seen_at` | timestamptz | nullable | 最近出现时间 |
| `last_seen_map_point_id` | uuid | FK -> map_points.id, nullable | 最近出现点位 |
| `personality_tags` | text[] | not null, default '{}' | 性格标签，1-4 字/个 |
| `story` | text | nullable | 猫咪故事，300 字内 |
| `feeding_notes` | text | nullable | 喂食注意事项 |
| `capture_notes` | text | nullable | 抓捕注意事项 |
| `medical_notes` | text | nullable | 医疗注意事项 |
| `remark` | text | nullable | 管理员备注 |
| `created_by` | uuid | FK -> users.id, nullable | 创建人 |
| `updated_by` | uuid | FK -> users.id, nullable | 更新人 |
| `created_at` | timestamptz | not null | 创建时间 |
| `updated_at` | timestamptz | not null | 更新时间 |
| `deleted_at` | timestamptz | nullable | 软删除时间 |

## 4.4 枚举设计

### `sex`

| 值 | 说明 |
|---|---|
| `unknown` | 未知 |
| `male` | 公 |
| `female` | 母 |

### `neuter_status`

| 值 | 说明 |
|---|---|
| `unknown` | 未知 |
| `not_neutered` | 未绝育 |
| `neutered` | 已绝育 |
| `scheduled` | 已预约 |
| `not_suitable` | 暂不适合 |

### `health_status`

| 值 | 说明 |
|---|---|
| `unknown` | 未知 |
| `healthy` | 健康 |
| `watching` | 需观察 |
| `abnormal` | 异常 |
| `injured` | 受伤 |
| `sick` | 疑似生病 |
| `in_treatment` | 治疗中 |
| `recovered` | 已恢复 |

### `status`

| 值 | 说明 |
|---|---|
| `active` | 正常在校 |
| `missing` | 暂时失踪 |
| `waiting_adoption` | 待领养 |
| `adopted` | 已领养 |
| `deceased` | 已死亡 |
| `transferred` | 转移照护 |
| `archived` | 归档 |

## 4.5 索引

```sql
CREATE INDEX idx_cats_name
ON cats (name);

CREATE INDEX idx_cats_status
ON cats (status);

CREATE INDEX idx_cats_health_status
ON cats (health_status);

CREATE INDEX idx_cats_neuter_status
ON cats (neuter_status);

CREATE INDEX idx_cats_primary_area
ON cats (primary_area_id);

CREATE INDEX idx_cats_primary_map_point
ON cats (primary_map_point_id);

CREATE INDEX idx_cats_last_seen
ON cats (last_seen_at DESC);

CREATE INDEX idx_cats_created_by
ON cats (created_by);

CREATE INDEX idx_cats_active
ON cats (status, deleted_at)
WHERE deleted_at IS NULL;
```

如启用 `pg_trgm`，建议：

```sql
CREATE EXTENSION IF NOT EXISTS pg_trgm;

CREATE INDEX idx_cats_name_trgm
ON cats USING gin (name gin_trgm_ops);

CREATE INDEX idx_cats_resident_area_trgm
ON cats USING gin (resident_area_text gin_trgm_ops);
```

## 4.6 业务规则

1. `name` 必填，不强制全局唯一。
2. `coat_color` 必填，可以使用预设值或自定义值。
3. `resident_area_text` V1 必填。
4. `personality_tags` 至少 1 个，最多建议 8 个。
5. `personality_tags` 中每个标签 1-4 字。
6. `story` 最多 300 字。
7. 新增猫咪时至少 1 张照片，最多 5 张照片。
8. 新增猫咪时必须选择 1 张主头像。
9. `primary_map_point_id` 可为空。
10. 不再使用 `friendliness_level` 字段。

cat_photos

## 5.2 作用

## 5.3 字段设计

| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | uuid | PK | 照片 ID |
| `cat_id` | uuid | FK -> cats.id, not null | 猫咪 ID |
| `photo_type` | varchar(32) | not null | 照片类型 |
| `file_url` | varchar(512) | not null | 原图或展示图 URL |
| `thumbnail_url` | varchar(512) | nullable | 缩略图 URL |
| `is_avatar` | boolean | not null, default false | 是否为主头像 |
| `caption` | varchar(255) | nullable | 图片说明 |
| `sort_order` | int | default 0 | 排序 |
| `uploaded_by` | uuid | FK -> users.id, nullable | 上传人 |
| `created_at` | timestamptz | not null | 创建时间 |
| `deleted_at` | timestamptz | nullable | 软删除时间 |

## 5.4 `photo_type` 枚举

| 值 | 说明 |
|---|---|
| `avatar` | 主头像 |
| `profile` | 档案照 |
| `recent` | 近照 |
| `health` | 健康异常照 |
| `neuter` | 绝育相关照 |
| `observation` | 观察记录来源照 |
| `other` | 其他 |

## 5.5 索引与约束

```sql
CREATE INDEX idx_cat_photos_cat
ON cat_photos (cat_id, sort_order);

CREATE INDEX idx_cat_photos_type
ON cat_photos (cat_id, photo_type);

CREATE UNIQUE INDEX uq_cat_avatar_photo
ON cat_photos (cat_id)
WHERE is_avatar = true AND deleted_at IS NULL;
```

## 5.6 业务规则

1. 每只猫最多 5 张未删除照片。
2. 每只猫必须有且只能有 1 张主头像。
3. `cats.avatar_url` 与主头像照片保持同步。
4. 列表页优先使用 `cats.avatar_thumbnail_url`。
5. 删除主头像前必须先设置新的主头像。

# 7. `cat_map_points` 猫咪点位关联表

## 7.1 表名

```text
cat_map_points
```

## 7.2 作用

保存猫咪与地图点位之间的关系。

一只猫可以有多个点位：常驻点、高频出现点、喂食点、临时观察点、最后出现点。

## 7.3 字段设计

| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | uuid | PK | 关系 ID |
| `cat_id` | uuid | FK -> cats.id, not null | 猫咪 ID |
| `map_point_id` | uuid | FK -> map_points.id, not null | 地图点位 ID |
| `relation_type` | varchar(32) | not null | 点位关系类型 |
| `is_primary` | boolean | default false | 是否主要点位 |
| `confidence_level` | varchar(32) | default 'confirmed' | 可信程度 |
| `description` | text | nullable | 关系说明 |
| `created_by` | uuid | FK -> users.id, nullable | 创建人 |
| `created_at` | timestamptz | not null | 创建时间 |
| `updated_at` | timestamptz | not null | 更新时间 |
| `deleted_at` | timestamptz | nullable | 软删除时间 |

## 7.4 `relation_type` 枚举

| 值 | 说明 |
|---|---|
| `home_area` | 常驻点 |
| `frequent_area` | 高频出现点 |
| `feeding_area` | 喂食点 |
| `temporary_area` | 临时观察点 |
| `lost_area` | 最后出现点 |

## 7.5 `confidence_level` 枚举

| 值 | 说明 |
|---|---|
| `confirmed` | 已确认 |
| `likely` | 大概率 |
| `possible` | 可能 |

## 7.6 索引

```sql
CREATE INDEX idx_cat_map_points_cat
ON cat_map_points (cat_id);

CREATE INDEX idx_cat_map_points_point
ON cat_map_points (map_point_id);

CREATE INDEX idx_cat_map_points_relation
ON cat_map_points (relation_type);

CREATE UNIQUE INDEX uq_cat_primary_map_point
ON cat_map_points (cat_id)
WHERE is_primary = true AND deleted_at IS NULL;
```

## 7.7 业务规则

1. 管理员新增猫咪第 2 步可跳过，不写入本表。
2. 如果第 2 步创建猫咪地图点，则同步写入本表。
3. 主要常驻点通常 `relation_type = home_area` 且 `is_primary = true`。
4. `cats.primary_map_point_id` 与主要点位保持同步。

| `cat_id` | uuid | FK -> cats.id, not null | 猫咪 ID |
| `created_at` | timestamptz | not null | 收藏时间 |
| `deleted_at` | timestamptz | nullable | 取消收藏时间 |

## 10.4 索引

```sql
CREATE INDEX idx_cat_favorites_user
ON cat_favorites (user_id, created_at DESC);

CREATE INDEX idx_cat_favorites_cat
ON cat_favorites (cat_id);

CREATE UNIQUE INDEX uq_cat_favorites_user_cat
ON cat_favorites (user_id, cat_id)
WHERE deleted_at IS NULL;
```

# 11. 前端本地草稿结构，不建表

V1 管理员新增猫咪支持草稿暂存，但不新增数据库表。

建议前端本地草稿 key：

```text
cat_create_draft_v1
```

草稿结构：

```json
{
  "version": 1,
  "current_step": 1,
  "updated_at": "2026-06-27T12:00:00+08:00",
  "basic_info": {},
  "photos": [],
  "map_point": null,
  "health_info": {}
}
```

如果后期需要服务端草稿，可新增 `cat_drafts`：

| 字段名 | 类型 | 说明 |
|---|---|---|
| `id` | uuid | 草稿 ID |
| `created_by` | uuid | 管理员 ID |
| `draft_data` | jsonb | 草稿内容 |
| `current_step` | int | 当前步骤 |
| `status` | varchar(32) | draft / submitted / discarded |
| `created_at` | timestamptz | 创建时间 |
| `updated_at` | timestamptz | 更新时间 |
| `expired_at` | timestamptz | 过期时间 |

后期扩展表不进入 V1 建表清单。

# 12. 与其他模块的关联

## 12.1 与地图模块

管理员新增猫咪第 2 步如果插入地图点：

1. 创建 `map_points`，`point_type = cat`，`point_scope = long_term`。
2. 创建 `cat_map_points`，`relation_type = home_area`。
3. 更新 `cats.primary_map_point_id`。
4. 地图页可展示猫咪点位。

## 12.2 与任务模块

任务可以关联猫咪，用于：

1. 喂食任务。
2. 观察任务。
3. 救助任务。
4. 送医任务。
5. 抓捕 / 绝育任务。

猫咪详情页展示相关任务记录。

## 12.3 与个人中心模块

个人中心可以通过 `cat_favorites` 展示用户收藏猫咪，也可以通过 `cat_observation_records` 展示我的观察记录。

## 12.4 与管理员模块

管理员新增、编辑、归档猫咪时应写入 `admin_operation_logs`。
