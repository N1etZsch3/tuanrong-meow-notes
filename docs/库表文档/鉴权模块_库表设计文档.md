# 校园猫协地图任务系统｜鉴权模块库表设计文档

> 版本：V1 鉴权库表设计稿  
> 适用范围：内部成员登录、默认密码改密、验证码、JWT 登录态、管理员创建/重置成员账号  
> 数据库：PostgreSQL  
> 后端建议：FastAPI + SQLAlchemy + Alembic  

---

## 1. 模块定位

鉴权模块负责系统内部成员的登录身份确认和基础账号安全。

V1 只支持：

```text
学号 + 密码 + 字母验证码 + JWT
```

V1 不支持：

```text
成员自主注册
手机号登录
邮箱登录
第三方登录
找回密码
复杂 refresh token 体系
```

账号来源统一为：

```text
管理员创建成员账号 / 管理员批量导入成员账号
```

成员第一次登录时通常使用默认密码，因此系统必须支持首次登录强制修改密码。

---

## 2. 鉴权相关表总览

| 表名 | 是否 V1 必做 | 作用 |
|---|---:|---|
| `users` | 是 | 登录账号、密码哈希、角色、状态、Token 版本 |
| `auth_captchas` | 是 | 登录字母验证码短期记录 |
| `user_profiles` | 是 | 当前用户展示资料，登录后返回昵称、头像等 |
| `admin_operation_logs` | 建议做 | 记录管理员创建账号、重置密码、禁用账号等操作 |

V1 暂不建立：

| 表名 | V1 不做原因 |
|---|---|
| `auth_sessions` | access token + token_version 已够用 |
| `refresh_tokens` | MVP 可以让 access token 过期后重新登录 |
| `password_reset_tokens` | 不做找回密码，由管理员重置 |
| `oauth_accounts` | 不做第三方登录 |

## 3. users 用户账号表

### 3.1 表名

```text
users
```

### 3.2 作用

保存成员登录账号、密码哈希、角色、账号状态和 JWT 失效控制字段。

`student_no` 是 V1 唯一登录账号。

### 3.3 字段设计

| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | uuid | PK | 用户 ID |
| student_no | varchar(64) | unique, not null | 学号，V1 登录账号 |
| password_hash | varchar(255) | not null | 密码哈希，不保存明文密码 |
| role | varchar(32) | not null, default 'member' | 用户角色 |
| status | varchar(32) | not null, default 'active' | 用户状态 |
| must_change_password | boolean | not null, default true | 是否必须修改初始密码；管理员创建账号时默认 true |
| password_updated_at | timestamptz | nullable | 密码最近更新时间 |
| last_login_at | timestamptz | nullable | 最近登录时间 |
| login_failed_count | int | not null, default 0 | 连续登录失败次数 |
| locked_until | timestamptz | nullable | 登录锁定截止时间 |
| token_version | int | not null, default 1 | Token 版本号，修改密码或管理员重置密码后自增 |
| created_at | timestamptz | not null | 创建时间 |
| updated_at | timestamptz | not null | 更新时间 |
| deleted_at | timestamptz | nullable | 软删除时间 |

### 3.4 role 枚举

| 值 | 说明 |
|---|---|
| member | 普通成员 |
| admin | 管理员 |
| super_admin | 超级管理员，后期扩展 |

V1 可以只使用：

```text
member
admin
```

### 3.5 status 枚举

| 值 | 说明 | 是否允许登录 |
|---|---|---:|
| active | 正常 | 是 |
| inactive | 未激活 | 否 |
| blocked | 禁用 | 否 |
| left | 已退出猫协 | 否 |
| deleted | 已删除 | 否 |

### 3.6 token_version 设计

JWT 是无状态的，如果只签发 token，不额外处理，用户修改密码或管理员重置密码后，旧 token 仍可能在过期前继续访问系统。

V1 使用 `token_version` 解决这个问题：

```text
登录成功
  ↓
JWT payload 写入 users.token_version
  ↓
访问接口时解析 JWT
  ↓
查询 users 当前 token_version
  ↓
两者一致：允许访问
两者不一致：拒绝访问，要求重新登录
```

触发 `token_version + 1` 的场景：

1. 成员修改自己的密码。
2. 管理员重置成员密码。
3. 管理员禁用账号时可选择自增。
4. 后期如果增加“强制下线”功能，也可以自增。

### 3.7 主要索引

```sql
CREATE UNIQUE INDEX uq_users_student_no
ON users (student_no);

CREATE INDEX idx_users_role
ON users (role);

CREATE INDEX idx_users_status
ON users (status);

CREATE INDEX idx_users_locked_until
ON users (locked_until)
WHERE locked_until IS NOT NULL;
```

### 3.8 建表示例

```sql
CREATE TABLE users (
    id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    student_no varchar(64) NOT NULL UNIQUE,
    password_hash varchar(255) NOT NULL,
    role varchar(32) NOT NULL DEFAULT 'member',
    status varchar(32) NOT NULL DEFAULT 'active',
    must_change_password boolean NOT NULL DEFAULT true,
    password_updated_at timestamptz NULL,
    last_login_at timestamptz NULL,
    login_failed_count int NOT NULL DEFAULT 0,
    locked_until timestamptz NULL,
    token_version int NOT NULL DEFAULT 1,
    created_at timestamptz NOT NULL DEFAULT now(),
    updated_at timestamptz NOT NULL DEFAULT now(),
    deleted_at timestamptz NULL
);
```

## 4. auth_captchas 登录验证码表

### 4.1 表名

```text
auth_captchas
```

### 4.2 作用

保存登录页字母验证码的短期校验记录。

V1 不强制接入 Redis，可以先用数据库落表；后期如果引入 Redis，再把验证码迁移到 Redis。

### 4.3 字段设计

| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | uuid | PK | 验证码 ID，前端登录时提交 |
| code_hash | varchar(255) | not null | 验证码哈希，不保存明文验证码 |
| scene | varchar(32) | not null, default 'login' | 使用场景 |
| expires_at | timestamptz | not null | 过期时间 |
| used_at | timestamptz | nullable | 使用时间，防止重复使用 |
| client_ip | varchar(64) | nullable | 请求 IP |
| user_agent | text | nullable | 请求设备信息 |
| created_at | timestamptz | not null | 创建时间 |

### 4.4 scene 枚举

| 值 | 说明 |
|---|---|
| login | 登录验证码 |

### 4.5 验证码规则

| 项目 | 建议 |
|---|---|
| 字符类型 | 大写英文字母 + 数字，排除 O/0、I/1 等易混字符 |
| 长度 | 4 位 |
| 有效期 | 5 分钟 |
| 是否区分大小写 | 不区分，后端统一转大写 |
| 是否允许重复使用 | 不允许，登录成功或校验失败达到策略阈值后标记 used_at |
| 存储方式 | 只保存 hash，不保存明文 |

### 4.6 主要索引

```sql
CREATE INDEX idx_auth_captchas_expires_at
ON auth_captchas (expires_at);

CREATE INDEX idx_auth_captchas_scene_created
ON auth_captchas (scene, created_at DESC);

CREATE INDEX idx_auth_captchas_unused
ON auth_captchas (scene, expires_at)
WHERE used_at IS NULL;
```

### 4.7 建表示例

```sql
CREATE TABLE auth_captchas (
    id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
    code_hash varchar(255) NOT NULL,
    scene varchar(32) NOT NULL DEFAULT 'login',
    expires_at timestamptz NOT NULL,
    used_at timestamptz NULL,
    client_ip varchar(64) NULL,
    user_agent text NULL,
    created_at timestamptz NOT NULL DEFAULT now()
);
```

## 5. user_profiles 用户资料表

### 5.1 表名

### 5.2 作用

## 6. admin_operation_logs 管理员操作日志表

### 6.1 表名

```text
admin_operation_logs
```

### 6.2 作用

记录管理员对成员账号的敏感操作，便于追踪误操作和安全问题。

鉴权相关操作建议记录：

1. 创建成员账号。
2. 批量导入成员账号。
3. 重置成员密码。
4. 禁用成员账号。
5. 恢复成员账号。
6. 调整成员角色。

### 6.3 建议字段

| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | uuid | PK | 日志 ID |
| admin_id | uuid | FK -> users.id | 操作管理员 |
| operation_type | varchar(64) | not null | 操作类型 |
| target_type | varchar(64) | not null | 目标类型，如 user |
| target_id | uuid | nullable | 目标 ID |
| summary | varchar(255) | nullable | 操作摘要 |
| before_data | jsonb | nullable | 操作前数据摘要 |
| after_data | jsonb | nullable | 操作后数据摘要 |
| client_ip | varchar(64) | nullable | 请求 IP |
| user_agent | text | nullable | 请求设备信息 |
| created_at | timestamptz | not null | 创建时间 |

### 6.4 operation_type 建议值

| 值 | 说明 |
|---|---|
| user_create | 创建成员账号 |
| user_import | 批量导入成员账号 |
| user_reset_password | 重置成员密码 |
| user_update_status | 修改账号状态 |
| user_update_role | 修改账号角色 |

## 7. 关键业务写入关系

### 7.1 管理员创建成员账号

涉及表：

```text
users
user_profiles
admin_operation_logs
```

流程：

```text
1. 管理员填写学号、初始密码、昵称、真实姓名等资料
2. 校验 users.student_no 是否已存在
3. 对初始密码进行哈希处理
4. 写入 users
   - role = member
   - status = active
   - must_change_password = true
   - token_version = 1
5. 写入 user_profiles
6. 写入 admin_operation_logs
```

### 7.2 成员登录

涉及表：

```text
auth_captchas
users
user_profiles
```

流程：

```text
1. 校验验证码是否存在、未过期、未使用
2. 校验验证码内容
3. 根据 student_no / meow_no 查询 users
4. 校验账号状态 status 是否为 active
5. 校验账号是否处于 locked_until 锁定期
6. 校验密码 hash
7. 登录失败：login_failed_count + 1，达到阈值后写入 locked_until
8. 如果用户 `must_change_password = true`，校验协议勾选项必须为 true
9. 登录成功：更新 last_login_at，清空 login_failed_count 和 locked_until
10. 标记验证码 used_at
11. 签发 JWT，payload 携带 user_id、student_no、role、token_version
12. 返回 user + profile + must_change_password
```

### 7.3 成员修改自己的密码

涉及表：

```text
users
```

流程：

```text
1. 校验旧密码
2. 校验新密码强度
3. 校验新密码不能与旧密码相同
4. 更新 password_hash
5. 更新 password_updated_at
6. must_change_password = false
7. token_version + 1
8. 要求前端清空 token 并重新登录
```

### 7.4 管理员重置成员密码

涉及表：

```text
users
admin_operation_logs
```

流程：

```text
1. 校验当前用户为 admin
2. 查询目标用户
3. 更新 password_hash
4. must_change_password = true
5. password_updated_at = now()
6. login_failed_count = 0
7. locked_until = null
8. token_version + 1
9. 写入 admin_operation_logs
```

## 8. 安全约束

1. 永远不保存明文密码。
2. 永远不在接口中返回 `password_hash`。
3. 验证码只保存 hash，不保存明文。
4. 登录失败提示统一使用“学号或密码错误”，避免泄露账号是否存在。
5. 管理员重置密码后强制用户改密。
6. 用户修改密码或管理员重置密码后，`token_version` 必须自增。
7. `blocked`、`left`、`deleted` 状态用户禁止登录。
8. 初始账号默认 `must_change_password = true`。
9. 当 `must_change_password = true` 时，只允许访问 `/auth/me`、`/auth/password`、`/auth/logout`。
10. 管理员账号不应使用统一默认密码长期运行。

---

## 9. MVP 结论

鉴权模块 V1 必须落地的库表为：

```text
users
auth_captchas
user_profiles
```

建议同步落地：

```text
admin_operation_logs
```

本模块不需要为了 MVP 单独增加会话表或 refresh token 表。通过 `JWT + users.token_version` 可以满足第一版登录、改密、重置密码、旧 token 失效的核心需求。
