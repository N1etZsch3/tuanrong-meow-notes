## 第二版目标（计划 1.1.0）

- 版本口径：`1.0.1` 只用于 1.0.0 线上补丁，例如修复域名、HTTPS、TLS、请求配置和线上 bug；第二版功能主线使用 `1.1.0`。
- 线上可用性目标：微信小程序正式请求必须使用已备案域名和 HTTPS；生产 API 域名切换为 `trmx.fun`，不再继续使用旧域名或裸 IP。上线前需确认 iOS 可接受的 TLS 证书链、证书域名匹配、Nginx/反向代理配置和小程序后台 request 合法域名配置。
- 功能目标：新增药品管理能力，后续设计需明确药品基础信息、存放点、库存/余量、使用记录、关联猫咪、关联任务或喵记记录，以及与物资点、地标点、任务记录的边界。
- 信息架构目标：将当前“任务”页重构为“喵记页”，把任务、地标、物资、药品等记录整合为更适合猫协日常沉淀的记录入口；具体路由、底部 Tab 文案、筛选维度和兼容旧任务接口的方式需先补设计文档。
- 稳定性目标：持续修复 1.0.0 上线后发现的 bug，保持 `main` 只承载线上稳定版本或热修发布，普通第二版开发从 `dev` 拉分支。
- 当前外部阻塞：`trmx.fun` 域名备案审核尚未结束；备案、HTTPS 证书和小程序合法域名配置完成前，1.0.1 或 1.1.0 都不能视为真正可线上使用。
