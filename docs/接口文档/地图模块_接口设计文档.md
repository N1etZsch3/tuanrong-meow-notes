# 校园猫协地图任务系统｜地图模块接口设计文档

> 文档版本：V1.0  
> 模块名称：地图模块  
> 适用阶段：MVP / 第一版接口设计  
> 基于文档：地图模块详细功能说明、地图模块库表设计说明、接口设计规范文档  
> 后端建议：FastAPI + Pydantic + SQLAlchemy 2.0  
> 前端建议：uni-app + Vue 3 + TypeScript + Pinia  
> 数据库建议：PostgreSQL + PostGIS  

---

# 1. 文档定位

本文档用于定义「校园猫协地图任务系统」中地图模块的 HTTP API。

地图模块是系统首页，也是成员查找点位、搜索校园猫协相关位置、查看任务 / 猫咪 / 物资点、导航，以及管理员在地图上进行点位操作的核心入口。

第一版接口目标：

1. 支持地图首页初始化。
2. 支持加载校园范围内的 Marker 点位。
3. 支持按类型、状态、缩放等级、地图视野范围筛选点位。
4. 支持搜索任务点、猫咪点、物资点、地标、地址文本。
5. 支持点击 Marker 或搜索结果后获取底部点位简略信息卡片。
6. 支持从点位卡片发起导航。
7. 支持地图页底部动态内容区展示。
8. 支持管理员长按地图创建点位。
9. 支持管理员编辑点位基础信息。
10. 支持管理员拖拽修改点位经纬度。
11. 支持管理员隐藏、恢复、归档点位。
12. 支持管理员上传和删除点位照片。

# 2. 设计边界

## 2.1 地图模块负责

地图模块接口主要处理：

| 能力 | 说明 |
|---|---|
| 地图初始化 | 返回校园中心点、边界、缩放等级、区域、Marker 配置 |
| 点位展示 | 返回地图 Marker 所需的轻量数据 |
| 点位搜索 | 聚合搜索任务、猫咪、物资、地标、区域、地址文本 |
| 点位卡片 | 返回底部简略信息卡片 |
| 导航信息 | 返回高德导航目标点、路线说明、现场图、入口图 |
| 地图底部动态内容 | 根据用户任务状态和搜索状态返回首页底部内容 |
| 管理员点位操作 | 创建点位、编辑点位、拖拽点位、隐藏恢复点位、上传照片 |

## 2.2 地图模块不负责

以下动作不放在地图模块里实现，只在地图卡片中返回跳转信息或动作入口：

| 业务动作 | 所属模块 | 建议接口 |
|---|---|---|
| 接取任务 | 任务模块 | `POST /api/v1/tasks/{task_id}/join` |
| 放弃任务 | 任务模块 | `POST /api/v1/tasks/{task_id}/abandon` |
| 完成打卡 | 任务模块 | `POST /api/v1/tasks/{task_id}/checkins` |
| 查看任务详情 | 任务模块 | `GET /api/v1/tasks/{task_id}` |
| 查看猫咪档案 | 猫咪库模块 | `GET /api/v1/cats/{cat_id}` |
| 添加观察记录 | 猫咪库模块 | `POST /api/v1/cats/{cat_id}/observations` |
| 查看物资详情 | 物资模块 / 管理员模块 | `GET /api/v1/supply-points/{supply_point_id}` |

核心原则：

```text
地图模块负责“这个点在哪里、如何展示、如何搜索、如何导航”。
任务 / 猫咪 / 物资模块负责“这个点对应的业务内容是什么”。
```

# 3. 接口通用规范

## 3.1 Base URL

```text
/api/v1
```

开发环境示例：

```text
http://localhost:8000/api/v1
```

## 3.2 Content-Type

普通 JSON 接口：

```http
Content-Type: application/json
```

文件上传接口：

```http
Content-Type: multipart/form-data
```

## 3.3 鉴权方式

需要登录的接口统一使用：

```http
Authorization: Bearer <access_token>
```

地图模块接口均需要登录。

原因：本系统是校园猫协内部系统，不面向公开用户开放。

## 3.4 字段命名

接口字段统一使用 `snake_case`。

示例：

```json
{
  "point_type": "task",
  "preview_enabled": true,
  "preview_min_zoom": 17,
  "created_at": "2026-06-23T10:00:00+08:00"
}
```

## 3.5 时间格式

所有时间统一使用 ISO 8601 / RFC3339 字符串。

示例：

```json
{
  "created_at": "2026-06-23T10:00:00+08:00",
  "deadline_at": "2026-06-23T18:00:00+08:00"
}
```

## 3.6 经纬度格式

经纬度字段统一使用：

| 字段 | 类型 | 说明 |
|---|---|---|
| `lng` | number | 经度 |
| `lat` | number | 纬度 |

示例：

```json
{
  "lng": 114.1234567,
  "lat": 30.1234567
}
```

前端不要把经纬度合并成字符串。

# 4. 统一响应结构

所有接口统一返回：

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| `code` | int | 业务错误码，成功为 `0` |
| `message` | string | 响应信息 |
| `data` | any | 实际响应数据 |
| `trace_id` | string | 请求追踪 ID |

成功示例：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": "0d65df4f-10c0-45e4-a48d-b04e0af7d3d2"
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

失败示例：

```json
{
  "code": 60003,
  "message": "点位不在校园范围内",
  "data": null,
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

| 60009 | 400 | 地图缩放等级不合法 |
| 60010 | 400 | 地图视野范围参数不完整 |
| 60011 | 404 | 校园配置不存在 |

# 6. 枚举定义

## 6.1 point_type 点位类型

| 值 | 说明 | 对应业务 |
|---|---|---|
| `task` | 任务点 | `tasks` |
| `cat` | 猫咪点 | `cats`、`cat_map_points` |
| `supply` | 物资点 | `supply_points` |
| `feeding` | 固定喂食点 | 后期扩展，也可以关联猫咪或任务 |
| `observation` | 临时观察点 | 后期扩展，可关联观察记录 |
| `landmark` | 地标点 | 通常只存在 `map_points` |

说明：

```text
紧急任务不单独作为 point_type。
紧急任务属于 task 点位下的 task_type = emergency。
```

## 6.2 point_scope 点位生命周期

| 值 | 说明 |
|---|---|
| `long_term` | 长期点位，如猫咪常驻点、物资点、地标 |
| `temporary` | 临时点位，如某次临时任务点、临时观察点 |

## 6.3 visibility 可见性

| 值 | 说明 |
|---|---|
| `public` | 普通成员可见 |
| `admin_only` | 仅管理员可见 |
| `hidden` | 隐藏，不在普通地图展示 |

## 6.4 status 点位状态

| 值 | 说明 |
|---|---|
| `active` | 正常 |
| `inactive` | 暂停使用 |
| `archived` | 已归档 |
| `deleted` | 已删除 / 软删除 |

## 6.5 photo_type 照片类型

| 值 | 说明 |
|---|---|
| `cover` | 封面图，用于点位卡片和地图预览 |
| `scene` | 现场图 |
| `route` | 路线图 |
| `entrance` | 入口图 |
| `reference` | 参考图 |

## 6.6 bottom_content_type 底部内容类型

| 值 | 说明 |
|---|---|
| `latest_tasks` | 最新任务 |
| `my_tasks` | 我的进行中任务 |
| `search_results` | 搜索结果 |
| `point_summary` | 点位简略信息 |
| `empty` | 无内容 |

## 6.7 action_key 卡片动作类型

| 值 | 说明 | 所属模块 |
|---|---|---|
| `navigate` | 导航 | 地图模块 |
| `view_detail` | 查看详情 | 任务 / 猫咪 / 物资模块 |
| `join_task` | 接取任务 | 任务模块 |
| `abandon_task` | 放弃任务 | 任务模块 |
| `complete_checkin` | 完成打卡 | 任务模块 |
| `view_checkins` | 查看完成记录 | 任务模块 |
| `view_cat_profile` | 查看猫咪档案 | 猫咪库模块 |
| `add_observation` | 添加观察记录 | 猫咪库模块 |
| `view_nearby_points` | 查看附近点位 | 地图模块 |

---

## 9.1 获取地图初始化配置

### 基本信息

- 方法：GET
- 路径：`/api/v1/map/init`
- 权限：需要登录
- Tags：Map
- 说明：进入地图首页时调用，返回校园配置、区域列表、Marker 配置和默认筛选项。

### 请求参数

Query 参数：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| `campus_id` | uuid | 否 | 当前默认校园 | 校园 ID，MVP 通常只有一个校园 |

### 请求示例

```http
GET /api/v1/map/init
Authorization: Bearer <access_token>
```

### 响应字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `campus` | CampusDTO | 校园地图配置 |
| `areas` | CampusAreaDTO[] | 校园区域列表 |
| `marker_configs` | MarkerConfigDTO[] | Marker 图标和展示规则 |
| `default_filters` | object | 默认筛选条件 |
| `ui_config` | object | 地图页 UI 默认配置 |

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "campus": {
      "campus_id": "3f39d2c2-1c5e-4d26-89a8-d5f8a63e4f7a",
```

## 9.2 获取地图 Marker 点位列表

### 基本信息

- 方法：GET
- 路径：`/api/v1/map/points`
- 权限：需要登录
- Tags：Map
- 说明：获取当前地图视野范围内需要展示的 Marker 点位列表。

### 请求参数

Query 参数：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| `campus_id` | uuid | 否 | 默认校园 | 校园 ID |
| `point_types` | string | 否 | 全部 | 点位类型，多个用英文逗号分隔，如 `task,cat,supply` |
| `business_types` | string | 否 | 全部 | 业务细分类型，如 `emergency,daily,resident,food` |
| `area_id` | uuid | 否 | null | 校园区域 ID |
| `status` | string | 否 | `active` | 点位状态 |
| `visibility` | string | 否 | `public` | 可见性，管理员可传 `admin_only` |
| `zoom` | int | 否 | null | 当前地图缩放等级，用于判断是否返回标题和预览 |
| `min_lng` | number | 否 | null | 当前视野最小经度 |
| `min_lat` | number | 否 | null | 当前视野最小纬度 |
| `max_lng` | number | 否 | null | 当前视野最大经度 |
| `max_lat` | number | 否 | null | 当前视野最大纬度 |
| `user_lng` | number | 否 | null | 用户当前位置经度，用于返回距离 |
| `user_lat` | number | 否 | null | 用户当前位置纬度，用于返回距离 |
| `include_preview` | boolean | 否 | true | 是否返回封面图和预览字段 |
| `only_available_tasks` | boolean | 否 | false | 是否只看未满员可接取任务 |
| `only_my_tasks` | boolean | 否 | false | 是否只看我的进行中任务 |
| `limit` | int | 否 | 500 | 最大返回数量 |

### 请求示例

```http
GET /api/v1/map/points?point_types=task,cat,supply&zoom=17&min_lng=114.1200000&min_lat=30.1200000&max_lng=114.1300000&max_lat=30.1300000&user_lng=114.1250000&user_lat=30.1250000
```

| `items` | MapPointMarkerDTO[] | Marker 点位列表 |
| `total` | int | 返回数量 |
| `map_strategy` | object | 前端展示策略 |

### 业务规则

1. 点位不存在返回 `60012`。
2. 普通成员不能查看 `admin_only`、`hidden`、`archived` 点位。
3. 卡片按钮由后端根据业务状态返回，前端不要自行硬编码任务按钮状态。
4. 任务点按钮逻辑：
   - 未接取且未满员：返回 `join_task` 可用。
   - 已接取且任务进行中：返回 `complete_checkin`、`abandon_task` 可用。
   - 未接取但已满员：返回 `join_task` 不可用，`disabled_reason = 人数已满`。
   - 任务已完成：返回 `view_checkins`。
   - 任务已取消：所有参与动作不可用。
5. 猫咪点返回 `view_cat_profile` 和 `add_observation`。
6. 物资点返回 `view_detail` 和 `navigate`。

| 60017 | 无权查看该点位 |
| 40101 | 未登录 |

## 10.5 管理员拖拽更新点位位置

### 基本信息

- 方法：PATCH
- 路径：`/api/v1/admin/map/points/{point_id}/location`
- 权限：管理员
- Tags：Admin Map
- 说明：管理员在地图编辑状态下拖拽 Marker 后，保存新的经纬度。

### 路径参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `point_id` | uuid | 是 | 点位 ID |

### 请求体字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `lng` | number | 是 | 新经度 |
| `lat` | number | 是 | 新纬度 |
| `amap_poi_id` | string | 否 | 新高德 POI ID |
| `amap_address` | string | 否 | 新高德地址 |
| `location_name` | string | 否 | 新位置名称 |
| `location_detail` | string | 否 | 新详细位置说明 |
| `route_instruction` | string | 否 | 新路线说明 |
| `reason` | string | 否 | 修改原因 |

### 请求示例

```json
{
  "lng": 114.1240000,
  "lat": 30.1240000,
  "amap_address": "湖北省黄石市...",
  "location_name": "北门右侧草丛",
  "location_detail": "修正到更靠近实际草丛的位置",
  "route_instruction": "从北门进入后沿右侧小路走约 35 米。",
  "reason": "原点位偏移，现场重新校准"
}
```

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "point_id": "6bf9e8aa-9ce1-4eb2-a1c3-ef3e086b4c59",
    "lng": 114.1240000,
    "lat": 30.1240000,
    "updated_at": "2026-06-23T10:40:00+08:00"
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 写入表

```text
map_points
admin_operation_logs
```

### 业务规则

1. 新经纬度必须合法。
2. 新点位必须在校园边界范围内。
3. 更新时同步更新 `map_points.geom`。
4. 修改前经纬度和修改后经纬度必须写入 `admin_operation_logs`。
5. 已归档点位不能拖拽修改。
6. 关联任务、猫咪、物资业务对象不需要同步更新坐标，因为坐标只存在 `map_points`。

### 错误码

| code | 说明 |
|---:|---|
| 60002 | 经纬度格式错误 |
| 60003 | 点位不在校园范围内 |
| 60012 | 点位不存在 |
| 60014 | 点位已归档，不能编辑 |
| 40302 | 权限不足 |

# 12. 前端调用流程建议

## 12.1 用户进入地图页

```text
1. 调用 GET /map/init
2. 初始化高德地图中心点、缩放等级、校园边界、Marker 图标配置
3. 调用 GET /map/points 加载当前视野 Marker
4. 调用 GET /map/bottom-content?mode=auto 加载底部动态内容
```

## 12.2 用户拖动或缩放地图

```text
1. 前端监听地图视野范围和 zoom 变化
2. 调用 GET /map/points，传入 min_lng、min_lat、max_lng、max_lat、zoom
3. 后端返回当前视野内 Marker
4. 前端根据 label_min_zoom、preview_min_zoom 控制标题和预览卡片展示
```

## 12.3 用户点击搜索栏并输入关键词

```text
1. 搜索栏进入输入状态
2. 输入内容防抖 300ms
3. 调用 GET /map/search?keyword=xxx
4. 底部动态内容区展示搜索结果
```

## 12.4 用户点击搜索结果

```text
1. 前端取 map_point_id
2. 地图移动到该点位
3. 高亮对应 Marker
4. 调用 GET /map/points/{point_id}/summary
5. 底部动态内容区展示点位卡片
```

## 12.5 用户点击 Marker

```text
1. 前端记录 selected_point_id
2. 地图视角移动到该点位附近
3. 调用 GET /map/points/{point_id}/summary
4. 底部动态内容区展示点位卡片
```

## 12.6 用户点击导航

```text
1. 调用 GET /map/points/{point_id}/navigation
2. 前端展示路线说明、入口说明、现场图
3. 前端根据 open_url 或 web_url 打开高德导航
```

## 12.7 管理员长按插点

```text
1. 前端进入管理员地图操作模式
2. 管理员长按地图获取 lng、lat
3. 弹出点位类型选择和表单
4. 提交 POST /admin/map/points
5. 后端写入 map_points 和 admin_operation_logs
6. 如果同时创建业务对象，则同步写入 tasks / cats / supply_points 等表
7. 前端刷新 GET /map/points 或局部添加 Marker
```

## 12.8 管理员拖拽修改点位

```text
1. 管理员进入点位编辑状态
2. 前端允许拖拽 Marker
3. 拖拽结束后展示确认按钮
4. 保存时调用 PATCH /admin/map/points/{point_id}/location
5. 后端校验校园边界并更新 geom
6. 后端写入 admin_operation_logs
7. 前端刷新点位位置
```

# 17. 最终结论

地图模块 V1 接口可以按以下核心组合落地：

```text
成员端：
/map/init
/map/points
/map/search
/map/points/{point_id}/summary
/map/points/{point_id}/navigation
/map/bottom-content

管理员端：
/admin/map/points
/admin/map/points/{point_id}
/admin/map/points/{point_id}/location
/admin/map/points/{point_id}/visibility
/admin/map/points/{point_id}/photos
```

这套接口可以支撑第一版地图首页的完整闭环：

```text
进入地图
加载校园配置
加载 Marker
搜索点位
点击 Marker 或搜索结果
查看底部卡片
发起导航
管理员长按插点
管理员编辑点位
管理员拖拽修改位置
```

同时，任务接取、任务打卡、猫咪观察、物资详情继续交给对应模块处理，地图模块只返回动作入口和页面路径，避免地图模块变成业务大杂烩。
