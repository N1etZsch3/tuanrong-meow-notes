# 校园猫协地图任务系统｜文件上传模块接口文档

> 版本：V1.0  
> 模块名称：文件上传模块 / 图片上传模块  
> Base URL：`/api/v1`  
> 适用范围：用户头像、猫咪头像、猫咪照片、猫咪健康照片、地图点位照片、路线图、入口图、任务打卡照片、观察记录照片  
> 存储服务：腾讯云 COS  
> 字段命名：统一使用 `snake_case`  
> 响应结构：统一使用 `{ code, message, data, trace_id }`  
> 基于文档：图片上传模块库表优化设计文档 V1.1 多缩略图版、COS 图片上传功能流程设计文档、接口设计规范文档

---

# 1. 文档定位

本文档定义「校园猫协地图任务系统」中统一文件上传模块的 HTTP API。

文件上传模块是一个跨模块基础能力，不属于猫咪库、地图、任务或个人中心中的任意一个单独业务模块。

它负责把用户上传的图片统一处理为系统可用的图片资产：

```text
用户上传原始图片
        ↓
后端校验真实图片类型、大小、像素
        ↓
修正 EXIF 方向、去除 EXIF 元数据
        ↓
根据 usage_type 选择图片处理预设
        ↓
生成多个图片变体，例如 thumb_sm / thumb_md / preview_lg / display
        ↓
上传到腾讯云 COS
        ↓
写入 file_assets 和 file_asset_variants
        ↓
返回 ImageAssetDTO 给业务模块或前端
```

业务模块再决定图片的具体业务含义，例如：

| 业务动作 | 所属模块 | 文件模块是否负责 |
|---|---|---:|
| 设置用户头像 | 个人中心模块 | 否 |
| 设置猫咪头像 | 猫咪库模块 | 否 |
| 添加猫咪照片 | 猫咪库模块 | 否 |
| 设置点位封面图 | 地图模块 | 否 |
| 上传任务打卡照片 | 任务模块 | 否 |
| 生成图片多尺寸变体 | 文件模块 | 是 |
| 上传 COS | 文件模块 | 是 |
| 返回图片 URL | 文件模块 | 是 |

# 2. 设计边界

## 2.1 文件上传模块负责

文件上传模块负责：

1. 接收图片文件。
2. 校验图片是否为空。
3. 校验文件大小。
4. 校验图片格式和真实 MIME。
5. 校验图片像素数量。
6. 修正图片 EXIF 方向。
7. 删除 EXIF 元数据。
8. 根据用途选择处理预设。
9. 生成多个图片变体。
10. 上传图片变体到腾讯云 COS。
11. 写入 `file_assets`。
12. 写入 `file_asset_variants`。
13. 返回统一图片资产 DTO。
14. 支持后续按场景选择合适图片变体。
15. 支持逻辑删除图片资产。

## 2.2 文件上传模块不负责

文件上传模块不负责：

1. 判断某张图片是否应该成为猫咪头像。
2. 判断某张图片是否应该成为地图点位封面。
3. 判断任务打卡图片是否有效。
4. 判断普通成员是否有权修改某只猫咪档案。
5. 判断普通成员是否有权删除某张业务照片。
6. 保存复杂上传日志。
7. 物理清理 COS 历史对象。

这些逻辑由对应业务模块处理。

## 2.3 V1 不做的能力

V1 暂不做：

| 能力 | V1 处理方式 |
|---|---|
| 客户端直传 COS | 不做，统一上传到后端，由后端处理后上传 COS |
| STS 临时密钥上传 | 后期扩展 |
| 文件上传日志表 | 不做 |
| 文件物理删除任务表 | 不做 |
| 文件权限独立表 | 不做，复用业务权限 |
| 视频上传 | 不做 |
| 音频上传 | 不做 |
| PDF / Word 上传 | 不做 |
| 原始大图长期保存 | 不做，V1 只保存处理后的图片变体 |

# 3. 通用规范

## 3.1 Base URL

```text
/api/v1
```

文件模块统一路径前缀：

```text
/api/v1/files
```

## 3.2 鉴权方式

除非特别说明，文件模块接口均需要登录。

```http
Authorization: Bearer <access_token>
```

## 3.3 Content-Type

上传接口使用：

```http
Content-Type: multipart/form-data
```

普通查询、删除、绑定接口使用：

```http
Content-Type: application/json
```

## 3.4 统一响应结构

所有接口统一返回：

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

## 3.5 首次登录改密限制

当用户：

```text
must_change_password = true
```

不允许访问图片上传接口。

返回：

```json
{
  "code": 40301,
  "message": "请先修改初始密码",
  "data": null,
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

## 3.6 资料初始化限制

当用户：

```text
profile_completed = false
```

只允许上传：

```text
user_avatar
```

用于首次登录后的头像初始化。

如果用户资料尚未初始化，却上传猫咪照片、点位照片、任务打卡照片等业务图片，返回：

```json
{
  "code": 63006,
  "message": "请先完成个人资料初始化",
  "data": null,
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

# 4. 数据模型

## 4.1 ImageVariantDTO

表示一张图片的某个尺寸版本。

```json
{
  "variant_key": "thumb_md",
  "url": "https://cdn.example.com/catmap/dev/cats/asset_id/thumb_md.jpg",
  "width": 320,
  "height": 240,
  "size_bytes": 48200,
  "mime_type": "image/jpeg"
}
```

字段说明：

| 字段 | 类型 | 说明 |
|---|---|---|
| `variant_key` | string | 图片变体标识 |
| `url` | string | 图片访问 URL，优先返回 CDN 域名，未配置 CDN 时返回 COS URL |
| `width` | int | 图片宽度 |
| `height` | int | 图片高度 |
| `size_bytes` | int | 图片大小，单位字节 |
| `mime_type` | string | MIME 类型，V1 默认 `image/jpeg` |

# 5. 枚举定义

## 5.1 usage_type 图片用途

| 值 | 说明 | 默认处理预设 |
|---|---|---|
| `user_avatar` | 用户头像 | `avatar_v1` |
| `cat_avatar` | 猫咪头像 | `avatar_v1` |
| `cat_photo` | 猫咪普通照片 | `normal_photo_v1` |
| `cat_health_photo` | 猫咪健康 / 受伤 / 异常照片 | `high_detail_photo_v1` |
| `map_point_cover` | 地图点位封面图 | `normal_photo_v1` |
| `map_point_scene` | 地图点位现场图 | `normal_photo_v1` |
| `map_point_route` | 地图路线图 / 入口图 | `route_photo_v1` |
| `task_checkin_photo` | 任务完成打卡照片 | `normal_photo_v1` |
| `observation_photo` | 猫咪观察记录照片 | `normal_photo_v1` |

---

## 5.2 owner_type 归属类型

| 值 | 说明 |
|---|---|
| `user` | 用户 |
| `cat` | 猫咪 |
| `map_point` | 地图点位 |
| `task` | 任务 |
| `task_checkin` | 任务打卡记录 |
| `observation` | 猫咪观察记录 |
| `temporary` | 临时图片，尚未绑定业务对象 |

说明：

```text
如果图片上传时业务对象还没有创建，可以先传 owner_type = temporary。
业务对象创建成功后，再由业务接口或文件绑定接口回填 owner_type / owner_id。
```

---

## 5.3 process_preset 图片处理预设

| 值 | 说明 |
|---|---|
| `avatar_v1` | 头像处理预设，生成 128、256、512 方图 |
| `normal_photo_v1` | 普通照片预设，生成 160、320、720、1280 长边图 |
| `route_photo_v1` | 路线图预设，生成 320、720、1600 长边图 |
| `high_detail_photo_v1` | 高细节照片预设，生成 320、720、1600 长边图 |

---

## 5.4 variant_key 图片变体

### 头像类

| variant_key | 尺寸 | 用途 |
|---|---:|---|
| `avatar_sm` | 128×128 | 头像列表、任务参与人头像 |
| `avatar_md` | 256×256 | 个人中心头像、资料卡片 |
| `avatar_lg` | 512×512 | 头像展示图 |

| `preview_lg` | 长边 720px | 照片墙预览、详情轻量预览 |
| `display` | 长边 1280px | 普通详情页展示 |

### 路线图 / 高细节图

| variant_key | 尺寸 | 用途 |
|---|---:|---|
| `thumb_md` | 长边 320px | 列表缩略图 |
| `preview_lg` | 长边 720px | 预览图 |
| `route_display` | 长边 1600px | 路线图详情展示 |
| `display` | 长边 1600px | 健康异常图详情展示 |

---

## 5.5 process_status 处理状态

| 值 | 说明 |
|---|---|
| `processing` | 图片处理中 |
| `completed` | 处理完成 |
| `failed` | 处理失败 |
| `deleted` | 已逻辑删除 |

V1 中上传接口同步处理图片，成功返回时通常已经是：


## 5.6 visibility 可见性

V1 推荐默认：


# 6. 图片处理预设

## 6.1 avatar_v1

用于：

```text
user_avatar
cat_avatar
```

处理规则：

1. 按图片中心裁剪为正方形。
2. 生成 128×128、256×256、512×512 三个版本。
3. 转为 JPEG。
4. 去除 EXIF。

输出：

| variant_key | 尺寸 | resize_mode | quality | 用途 |
|---|---:|---|---:|---|
| `avatar_sm` | 128×128 | cover | 82 | 头像列表 |
| `avatar_md` | 256×256 | cover | 82 | 个人中心头像 |
| `avatar_lg` | 512×512 | cover | 85 | 头像展示 |

默认：

```text
default_variant_key = avatar_lg
default_thumb_variant_key = avatar_md
```

---

## 6.2 normal_photo_v1

用于：

```text
cat_photo
map_point_cover
map_point_scene
task_checkin_photo
observation_photo
```

处理规则：

1. 等比缩放，不裁剪。
2. 生成 160、320、720、1280 长边图。
3. 转为 JPEG。
4. 去除 EXIF。

输出：

| variant_key | 尺寸 | resize_mode | quality | 用途 |
|---|---:|---|---:|---|
| `thumb_sm` | 长边 160px | fit | 78 | 地图 Marker 小卡片 |
| `thumb_md` | 长边 320px | fit | 80 | 列表、底部卡片 |
| `preview_lg` | 长边 720px | fit | 82 | 详情预览、照片墙 |
| `display` | 长边 1280px | fit | 82 | 详情展示 |

默认：

```text
default_variant_key = display
default_thumb_variant_key = thumb_md
```

---

## 6.3 route_photo_v1

用于：

```text
map_point_route
map_point_scene 中需要保留路线细节的图片
```

输出：

| variant_key | 尺寸 | resize_mode | quality | 用途 |

| `thumb_md` | 长边 320px | fit | 80 | 路线图列表缩略图 |

## 9.2 上传单张图片

### 基本信息

```http
POST /api/v1/files/images
Authorization: Bearer <access_token>
Content-Type: multipart/form-data
```

| 项目 | 内容 |
|---|---|
| 权限 | 登录用户 |
| Tags | Files |
| 说明 | 上传一张图片，后端压缩处理、生成多个变体、上传 COS，并写入文件资产表 |

### FormData 参数

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `file` | file | 是 | 图片文件本体 |
| `usage_type` | string | 是 | 图片用途，如 `user_avatar`、`cat_photo` |
| `owner_type` | string | 否 | 业务归属类型，不传时默认为 `temporary` |
| `owner_id` | uuid | 否 | 业务归属 ID，可为空 |
| `visibility` | string | 否 | 可见性，默认 `internal` |
| `caption` | string | 否 | 图片说明，文件模块只记录摘要，业务表可另存 |

### 请求示例

```http
POST /api/v1/files/images
Authorization: Bearer <access_token>
Content-Type: multipart/form-data

file=@xiaojv.jpg
usage_type=cat_photo
owner_type=cat
owner_id=9a3f6d8e-8aa7-4b69-8501-4c2b3619aa3d
visibility=internal
caption=小橘在体育馆后草坪
```

### 后端处理流程

```text
1. 校验登录态和用户状态。
2. 校验 must_change_password。
3. 校验 profile_completed；未初始化时只允许 user_avatar。
4. 校验 file 是否存在。
5. 校验 usage_type 是否支持。
6. 校验 owner_type 是否支持。
7. 校验文件大小。
8. 读取图片文件头，校验真实 MIME。
9. 解析图片，读取宽高和像素数量。
10. 修正 EXIF 方向。
11. 去除 EXIF 元数据。
12. 根据 usage_type 选择 process_preset。
13. 生成多个图片变体。
14. 按 COS 对象 Key 规则上传每个变体。
15. 在事务中写入 file_assets。
16. 在事务中写入 file_asset_variants。
17. 回填 file_assets.default_url 和 default_thumb_url。
18. 返回 ImageAssetDTO。
```

### 业务规则

1. 上传接口只创建文件资产，不一定写入猫咪、任务、地图等业务表。
2. 如果上传时传入合法 `owner_type` 和 `owner_id`，可以直接绑定文件归属。
3. 如果业务对象尚未创建，可以传 `owner_type = temporary`。
4. V1 不保存用户原始大图，只保存处理后的变体和原始文件摘要。
5. V1 返回完整 `variants`，业务列表接口不必全部返回。
6. 如果 COS 上传成功但数据库写入失败，后端应尝试删除本次已上传 COS 对象。
7. 如果部分变体生成失败，本次上传整体失败，不写入业务表。

### 错误码

| code | 说明 |
|---:|---|
| 65001 | 文件不能为空 |
| 65002 | 文件大小超过限制 |
| 65003 | 文件类型不支持 |
| 65004 | 图片内容损坏或无法解析 |
| 65005 | 图片尺寸超过限制 |
| 65006 | 图片用途类型不支持 |
| 65007 | 图片处理失败 |
| 65008 | 上传 COS 失败 |
| 65014 | 文件元数据写入失败 |
| 65015 | 图片归属类型不支持 |
| 65018 | 图片处理预设不支持 |
| 40101 | 未登录 |
| 40102 | Token 无效或已过期 |
| 40301 | 请先修改初始密码 |
| 63006 | 请先完成个人资料初始化 |

4. 如果希望部分成功部分失败，后期可扩展 `partial_success = true` 参数，V1 不做。
5. 任务打卡、观察记录等业务接口也可以自己接收多图，再内部调用 FileService；本接口适合前端先上传图片再提交业务表单。

## 9.5 按场景或变体获取图片 URL

| `task_checkin_list` | `thumb_md` | 打卡记录列表 |
| `task_checkin_detail` | `display` | 打卡详情 |
| `route_photo_preview` | `preview_lg` | 路线图预览 |
| `route_photo_detail` | `route_display` | 路线图详情 |
| `health_photo_detail` | `display` | 健康异常图片详情 |

### 响应示例


```text
1. scene 对应 variant_key

```


```http
PATCH /api/v1/files/assets/{asset_id}/owner
Authorization: Bearer <access_token>
Content-Type: application/json
```

| 项目 | 内容 |
|---|---|
| 权限 | 登录用户；需要有目标业务对象操作权限 |
| Tags | Files |
| 说明 | 将临时图片绑定到具体业务对象，或更新图片归属 |

### 路径参数

| 参数 | 类型 | 说明 |
|---|---|---|
| `asset_id` | uuid | 文件资产 ID |

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `owner_type` | string | 是 | 目标归属类型 |
| `owner_id` | uuid | 是 | 目标归属 ID |
| `usage_type` | string | 否 | 可选，必要时同步修正用途 |

### 请求示例

```json
{
  "owner_type": "cat",
  "owner_id": "9a3f6d8e-8aa7-4b69-8501-4c2b3619aa3d",
  "usage_type": "cat_photo"
}
```

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "asset_id": "0d65df4f-10c0-45e4-a48d-b04e0af7d3d2",
    "owner_type": "cat",
    "owner_id": "9a3f6d8e-8aa7-4b69-8501-4c2b3619aa3d",
    "usage_type": "cat_photo"
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 业务规则

1. 主要用于先上传图片、后创建业务对象的表单流程。
2. 已绑定图片原则上不能随意改绑到无关业务对象。
3. 如果当前图片已经绑定到其他业务对象，且不是当前用户刚创建的临时图片，返回 `65017`。
4. 目标业务对象的权限校验由对应业务模块提供服务方法。
5. 只更新 `file_assets.owner_type`、`file_assets.owner_id`、必要时更新 `usage_type`。
6. 不移动 COS 对象路径；V1 允许 object_key 中仍保留 temporary 路径，后期如需可异步迁移。

### 错误码

| code | 说明 |
|---:|---|
| 65009 | 文件资产不存在 |
| 65010 | 无权访问该文件 |
| 65012 | 文件已被删除 |
| 65015 | 图片归属类型不支持 |
| 65017 | 图片已经绑定到其他业务对象 |
| 65020 | 临时图片已过期 |
| 40302 | 权限不足 |

## 9.8 逻辑删除图片资产

### 基本信息

```http
DELETE /api/v1/files/assets/{asset_id}
Authorization: Bearer <access_token>
```

| 项目 | 内容 |
|---|---|
| 权限 | 上传者或管理员；业务图片还需业务模块授权 |
| Tags | Files |
| 说明 | 逻辑删除图片资产和对应图片变体记录 |

### 路径参数

| 参数 | 类型 | 说明 |
|---|---|---|
| `asset_id` | uuid | 文件资产 ID |

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "asset_id": "0d65df4f-10c0-45e4-a48d-b04e0af7d3d2",
    "deleted": true,
    "deleted_at": "2026-06-27T15:30:00+08:00"
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 后端处理

```text
1. 校验文件是否存在。
2. 校验是否已删除。
3. 校验当前用户是否有权限删除。
4. 更新 file_assets.process_status = deleted。
5. 更新 file_assets.deleted_at = now()。
6. 更新 file_asset_variants.deleted_at = now()。
7. 不立即删除 COS 对象。
```

### 业务规则

1. V1 只做逻辑删除，不做 COS 物理删除。
2. 如果图片已经被业务表引用，推荐由业务模块先解除关联或软删除业务照片记录。
3. 文件模块可以拒绝直接删除已绑定业务对象的图片，也可以允许管理员删除。推荐 V1：普通成员不能直接删除已绑定业务对象图片，必须走业务删除接口。
4. 头像替换时，旧头像可以标记删除，也可以保留为历史头像。V1 推荐头像替换后旧头像逻辑删除。

### 错误码

| code | 说明 |
|---:|---|
| 65009 | 文件资产不存在 |
| 65010 | 无权访问该文件 |
| 65012 | 文件已被删除 |
| 40302 | 权限不足 |

# 10. 业务模块接入方式

## 10.1 用户头像

推荐流程：

```text
1. 前端调用 POST /files/images，usage_type = user_avatar。

3. 前端调用 PATCH /profile/me 更新头像。
4. 个人中心模块写入 user_profiles.avatar_asset_id。
5. 同步冗余 user_profiles.avatar_url = avatar_lg.url。
6. 同步冗余 user_profiles.avatar_thumb_url = avatar_md.url。
```



# 12. COS 对象 Key 规则

V1 推荐对象 Key：

```text
catmap/{env}/{owner_type}/{owner_id_or_temp}/{asset_id}/{variant_key}.jpg
```

示例：

```text
catmap/dev/user/0d65df4f/avatar_lg.jpg
catmap/dev/cat/9a3f6d8e/asset_id/thumb_md.jpg
catmap/dev/map_point/6bf9e8aa/asset_id/route_display.jpg
catmap/dev/temporary/uploaded_by_user_id/asset_id/display.jpg
```

说明：

1. `env` 区分 `dev`、`test`、`prod`。
2. `owner_type` 用于按业务归类。
3. `owner_id_or_temp` 用于业务对象或临时对象归类。
4. `asset_id` 保证唯一性。
5. `variant_key` 直接表达图片版本。
6. V1 不因后续绑定 owner 而移动 COS 对象，避免多一步复制和删除。

# 15. 数据一致性规则

## 15.1 上传成功定义

一次图片上传成功必须满足：

```text
1. 图片文件解析成功。
2. 所需变体全部生成成功。
3. 所需变体全部上传 COS 成功。
4. file_assets 写入成功。
5. file_asset_variants 写入成功。
6. file_assets.default_url 和 default_thumb_url 回填成功。
```

## 15.2 失败回滚

| 失败点 | 处理方式 |
|---|---|
| 图片解析失败 | 直接返回错误，不写库 |
| 图片处理失败 | 直接返回错误，不写库 |
| COS 上传失败 | 返回错误，删除已上传的本次 COS 对象 |
| `file_assets` 写入失败 | 尝试删除已上传 COS 对象 |
| `file_asset_variants` 写入失败 | 回滚数据库事务，尝试删除 COS 对象 |
| 业务表写入失败 | 文件资产保留为 `temporary`，后续可由人工或后期清理机制处理 |

## 15.3 临时图片规则

V1 允许临时图片：

```text
owner_type = temporary
owner_id = null
```

建议规则：

1. 临时图片只允许上传者本人和管理员查看。
2. 临时图片可以绑定到业务对象。
3. 临时图片不在普通业务列表展示。
4. 临时图片超时清理先不做自动任务，但可以预留查询条件。

# 18. 最终推荐落地方案

V1 文件上传模块最终建议按以下方式落地：

1. 新增独立文件模块：`backend/app/modules/files`。
2. 新增接口前缀：`/api/v1/files`。
3. 新增核心表：`file_assets`、`file_asset_variants`。
4. 不创建图片上传日志表。
5. 不创建文件清理任务表。
6. 所有图片上传都走 `POST /files/images` 或 `POST /files/images/batch`。
7. 所有上传图片都由后端统一处理后上传 COS。
8. 所有图片至少生成默认展示图和默认缩略图。
9. 普通图片生成 `thumb_sm`、`thumb_md`、`preview_lg`、`display`。
10. 头像生成 `avatar_sm`、`avatar_md`、`avatar_lg`。
11. 路线图生成 `thumb_md`、`preview_lg`、`route_display`。
12. 业务表只保存 `asset_id`，并可冗余常用 URL。
13. 地图 Marker、列表、头像小图必须使用缩略图变体，避免加载大图。
14. 详情页按场景使用 `preview_lg` 或 `display`。
15. V1 只做逻辑删除，不立即物理删除 COS 对象。

这套接口可以覆盖当前项目中的全部图片上传场景，并且不会让猫咪库、地图模块、任务模块、个人中心各自维护一套图片压缩和缩略图规则。
