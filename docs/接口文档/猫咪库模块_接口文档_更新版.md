# 校园猫协地图任务系统｜猫咪库模块接口文档

> 版本：V1.1 管理员录入流程调整版  
> Base URL：`/api/v1`  
> 字段命名：统一使用 `snake_case`  
> 响应结构：统一使用 `{ code, message, data, trace_id }`  
> 本次调整重点：删除 `friendliness_level`；新增性格标签规则；新增管理员四页式录入流程；V1 草稿由前端本地暂存，后端提供最终创建和后续维护接口。

---

# 1. 文档定位

本文档定义猫咪库模块接口，包括成员端猫咪浏览、搜索、筛选、详情、观察记录、收藏，以及管理员新增猫咪、编辑档案、维护照片、绑定点位、维护健康与注意事项等接口。

管理员新增猫咪采用：

```text
第 1 步：基础信息
第 2 步：地图点位，可跳过
第 3 步：健康与注意事项，可跳过
第 4 步：预览确认并保存
```

V1 支持草稿暂存，但草稿优先由前端本地实现，不强制提供服务端草稿接口。

# 2. 通用约定

## 2.1 Base URL

```text
/api/v1
```

## 2.2 鉴权

除公开说明外，猫咪库接口均需要登录：

```http
Authorization: Bearer <access_token>
```

管理员接口要求当前用户角色为：

```text
admin / super_admin
```

## 2.3 统一响应结构

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

## 2.4 分页结构

请求参数：

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---:|---|
| `page` | int | 1 | 页码 |
| `page_size` | int | 20 | 每页数量，最大 100 |

响应结构：

```json
{
  "items": [],
  "page": 1,
  "page_size": 20,
  "total": 0,
  "has_more": false
}
```

# 4. 枚举定义

## 4.1 `sex`

| 值 | 说明 |
|---|---|
| `unknown` | 未知 |
| `male` | 公 |
| `female` | 母 |

| 值 | 说明 |
|---|---|
| `unknown` | 未知 |
| `not_neutered` | 未绝育 |

| `scheduled` | 已预约 |
| `not_suitable` | 暂不适合 |

| 值 | 说明 |
|---|---|
| `unknown` | 未知 |
| `healthy` | 健康 |
| `watching` | 需观察 |
| `abnormal` | 异常 |
| `injured` | 受伤 |
| `sick` | 疑似生病 |
| `in_treatment` | 治疗中 |
| `recovered` | 已恢复 |

| 值 | 说明 |
|---|---|

| `deceased` | 已死亡 |

| 值 | 说明 |

# 8. 管理员新增猫咪接口设计

## 8.1 前端草稿机制说明

V1 草稿暂存由前端负责，不需要调用后端草稿 API。

建议前端本地草稿 key：

```text
cat_create_draft_v1
```

草稿保存内容包括：

```text
当前步骤
基础信息
已上传图片 URL
可选地图点位
可选健康与注意事项
更新时间
```

管理员进入新增猫咪页时，如果检测到本地草稿，前端提示：

```text
检测到未完成的猫咪档案
继续编辑 / 丢弃草稿
```

第 4 步确认保存成功后，前端清空草稿。

## 8.2 创建猫咪档案

### 基本信息

```http
POST /api/v1/admin/cats
Authorization: Bearer <access_token>
Content-Type: application/json
```

### 权限

管理员。

### 说明

管理员第 4 步预览确认后调用。后端一次性创建猫咪档案、照片、可选地图点位、可选健康与注意事项。

### 请求参数

使用 `AdminCatCreateRequest`。

### 请求示例

```json
{
  "basic_info": {
    "name": "小橘",
    "coat_color": "橘猫",
    "sex": "unknown",
    "neuter_status": "neutered",
    "status": "active",
    "personality_tags": ["亲人", "贪吃"],
    "resident_area_text": "学生宿舍区东侧草坪",
    "story": "小橘常在宿舍区晒太阳，比较亲人。"
  },
  "photos": [
    {
      "file_url": "https://cos.example.com/cats/x.jpg",
      "thumbnail_url": "https://cos.example.com/cats/x_thumb.jpg",
      "photo_type": "avatar",
      "is_avatar": true,
      "caption": "正脸照",
      "sort_order": 0
    }
  ],
  "map_point": null,
  "health_info": null
}
```

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "cat_id": "uuid",
    "created_map_point_id": null,
    "next_action": "view_detail",
    "detail_path": "/pages/cats/detail?id=uuid"
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 业务规则

1. `basic_info.name` 必填。
2. `basic_info.coat_color` 必填。
3. `basic_info.sex` 必填，可传 `unknown`。
4. `basic_info.neuter_status` 必填，可传 `unknown`。
5. `basic_info.status` 必填。
6. `basic_info.personality_tags` 至少 1 个。
7. 每个性格标签 1-4 字。
8. `basic_info.resident_area_text` 必填。
9. `basic_info.story` 最多 300 字。
10. `photos` 至少 1 张，最多 5 张。
11. `photos` 中必须且只能有 1 张 `is_avatar = true`。
12. `map_point = null` 表示跳过第 2 步。
13. `health_info = null` 表示跳过第 3 步，默认 `health_status = unknown`。
14. 如果 `map_point` 不为空，后端创建 `map_points` 与 `cat_map_points`。
15. 创建成功后写入管理员操作日志。

# 13. 前后端协作说明

## 13.1 管理员新增猫咪推荐流程

```text
进入新增猫咪页
  ↓
检查前端本地草稿
  ↓
第 1 步填写基础信息
  ↓
自动保存本地草稿
  ↓
第 2 步可选绑定地图点位
  ↓
自动保存本地草稿
  ↓
第 3 步可选填写健康注意事项
  ↓
自动保存本地草稿
  ↓
第 4 步预览确认
  ↓
POST /api/v1/admin/cats
  ↓
保存成功后清空本地草稿
  ↓
跳转猫咪详情页
```

## 13.2 图片上传协作

1. 前端在第 1 步选择照片。
2. 调用统一文件上传模块上传图片。
3. 文件模块返回 `file_url` 和 `thumbnail_url`。
4. 前端把图片信息写入本地草稿。
5. 第 4 步确认保存时，把图片 URL 提交给 `POST /admin/cats`。
6. 后端创建 `cat_photos`，并同步主头像到 `cats`。

## 13.3 跳过步骤处理

| 步骤 | 跳过后提交内容 | 后端处理 |
|---|---|---|
| 第 2 步地图点位 | `map_point = null` | 不创建地图点；`primary_map_point_id = null` |
| 第 3 步健康注意事项 | `health_info = null` | `health_status = unknown`；注意事项为空 |

## 13.4 服务端草稿后期扩展

V1 不提供草稿 API。后期如需多端同步，可扩展：

```http
POST /api/v1/admin/cats/drafts
PATCH /api/v1/admin/cats/drafts/{draft_id}
GET /api/v1/admin/cats/drafts/latest
DELETE /api/v1/admin/cats/drafts/{draft_id}
POST /api/v1/admin/cats/drafts/{draft_id}/submit
```

后期扩展不影响 V1 接口设计。
