# 校园猫协地图任务系统｜用户与个人中心模块接口文档

> 版本：V1.1.1 接口设计稿
> Base URL：`/api/v1`
> 适用范围：Auth、Profile、Me、Notifications、Assignments、Admin Users
> 字段命名：统一使用 `snake_case`
> 响应结构：统一使用 `{ code, message, data, trace_id }`

---

# 1. 接口范围

本文档覆盖用户与个人中心模块相关接口：

1. 登录验证码。
2. 喵喵号密码登录与微信 OpenID 绑定登录。
3. 当前用户信息。
4. 修改密码。
5. 解绑当前用户自己的微信。
6. 退出登录。
7. 当前用户资料查询与编辑。
8. 用户资料初始化。
9. 任务偏好查询与保存。
10. 个人中心首页。
11. 我的任务。
12. 我的打卡记录。
13. 我的观察记录。
14. 我的排班。
15. 指派给我的任务。
16. 通知中心。
17. 个人统计。
18. 管理员成员账号管理。

不提供成员自主注册接口。1.1.1 起，成员首次仍需通过喵喵号密码登录，绑定微信 OpenID 后可在小程序启动时自动登录；管理员可为普通成员重置绑定，所有用户也可在账号设置中解绑本人。解绑成功后立即退出登录，下一次通过喵喵号密码兜底登录并重新绑定。

# 2. 通用约定

## 2.1 统一响应结构

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

## 2.2 鉴权 Header

除公开接口外，统一使用：

```http
Authorization: Bearer <access_token>
```

## 2.3 分页参数

列表接口统一使用：

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---:|---|
| page | int | 1 | 当前页，从 1 开始 |
| page_size | int | 20 | 每页数量，最大 100 |

分页响应：

```json
{
  "items": [],
  "page": 1,
  "page_size": 20,
  "total": 0,
  "has_more": false
}
```

## 2.4 首次登录改密限制

当当前用户：

```text
must_change_password = true
```

只允许访问：

```http
GET /api/v1/auth/me
PATCH /api/v1/auth/password
POST /api/v1/auth/logout
```

访问其他业务接口返回：

```json
{
  "code": 40301,
  "message": "请先修改初始密码",
  "data": null,
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

## 2.5 资料初始化限制

当当前用户：

```text
profile_completed = false
```

建议只允许访问：

```http
GET /api/v1/auth/me
GET /api/v1/profile/me
PATCH /api/v1/profile/me
POST /api/v1/profile/me/complete
POST /api/v1/auth/logout
```

访问其他业务接口可以返回：

```json
{
  "code": 63006,
  "message": "请先完成个人资料初始化",
  "data": null,
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

## 2.6 微信 OpenID 绑定登录

1. 前端只提交微信临时 code，OpenID 由后端使用 `code2Session` 获取。
2. 首次绑定前必须通过喵喵号、密码和验证码校验，保持邀请制。
3. 用户在密码登录页点击登录后，前端先使用当前微信 code 发起不确认绑定的登录；已绑定且 OpenID 匹配时直接登录。只有后端返回 `40006 WECHAT_BINDING_CONFIRMATION_REQUIRED` 时才使用原生模态框告知“登录后，当前微信将自动与该喵喵号绑定，用于后续自动登录和账号保护”；用户确认后以新的 code 重试并提交绑定，取消时不再发起第二次登录请求。
4. 已绑定用户启动小程序时由加载页静默尝试微信自动登录，不显示绑定提示或登录结果提示；成功后继续沿用 `must_change_password` 和 `profile_completed` 的页面流转，未匹配时进入喵喵号密码登录页。
5. 用户确认绑定后若无法取得微信 code，前端提示重试，不提交空 code，也不降级为未绑定的密码登录。
6. 管理员清除绑定后，后端递增 `token_version`，成员下一次需用喵喵号密码重新登录并绑定。
7. OpenID 仅用于本小程序的自动登录和账号保护，保存在后端 `users.wechat_openid`；前端不接收、不缓存、不展示 OpenID 或 session key。
8. 管理员解绑会清除后端微信标识及绑定时间；正式发布前需在小程序隐私保护指引和平台配置中说明该标识的用途与处理方式，并完成真机验收。
9. 管理员之间不能相互清除绑定；管理员本人和普通成员本人统一通过 `/auth/wechat-binding` 自助解绑。
10. 自助解绑前必须二次确认并告知用户会立即退出登录；成功后前端清除本地会话，失败时保留会话。

# 5. Auth 接口

## 5.1 获取登录验证码

### 基本信息

- 方法：GET
- 路径：`/api/v1/auth/captcha`
- 权限：公开
- 说明：获取登录页字母验证码。

### 请求参数

无。

### 响应字段

| 字段 | 类型 | 说明 |
|---|---|---|
| captcha_id | string | 验证码 ID |
| image_base64 | string | 验证码图片 base64 |
| expires_in | int | 有效秒数 |

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "captcha_id": "b5f7a6d8-2a3e-4f91-98e5-4c22f47a1f31",
    "image_base64": "data:image/png;base64,iVBORw0KGgoAAA...",
    "expires_in": 300
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 错误码

| code | 说明 |
|---:|---|
| 50001 | 服务端异常 |

### 业务规则

1. 后端生成验证码明文后，只保存哈希到 `auth_captchas.code_hash`。
2. 验证码默认 5 分钟有效。
3. 验证码只能使用一次。

---

## 5.2 用户登录

### 基本信息

- 方法：POST
- 路径：`/api/v1/auth/login`
- 权限：公开
- 说明：使用学号、密码、字母验证码登录。

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| meow_no | string | 是 | 喵喵号，格式如 `trmx0001` |
| student_no | string | 否 | 兼容旧字段，含义同喵喵号 |
| password | string | 是 | 密码 |
| captcha_id | string | 是 | 验证码 ID |
| captcha_code | string | 是 | 用户输入的验证码 |
| agreement_checked | boolean | 是 | 是否勾选协议 |

### 请求示例

```json
{
  "meow_no": "trmx0001",
  "password": "123456",
  "captcha_id": "b5f7a6d8-2a3e-4f91-98e5-4c22f47a1f31",
  "captcha_code": "ABCD",
  "agreement_checked": true
}
```

### 响应字段

| 字段 | 类型 | 说明 |
|---|---|---|
| access_token | string | JWT Token |
| token_type | string | 固定为 `Bearer` |
| expires_in | int | Token 有效秒数 |
| user | object | 当前用户摘要 |
| profile | object | 当前用户资料 |
| next_action | string | 前端下一步动作 |

`next_action` 可选：

| 值 | 说明 |
|---|---|
| change_password | 必须先改密 |
| complete_profile | 必须先初始化资料 |
| enter_app | 进入应用 |

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.xxx",
    "token_type": "Bearer",
    "expires_in": 7200,
    "user": {
      "id": "0d65df4f-10c0-45e4-a48d-b04e0af7d3d2",
      "student_no": "trmx0001",
      "meow_no": "trmx0001",
      "role": "member",
      "status": "active",
      "must_change_password": true,
      "profile_completed": false
    },
    "profile": {
      "nickname": "猫协成员1010",
      "avatar_url": null,
      "department": null,
      "profile_completed": false
    },
    "next_action": "change_password"
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 错误码

| code | HTTP | 说明 |
|---:|---:|---|
| 40003 | 400 | 验证码错误 |
| 40004 | 400 | 验证码已过期 |
| 40005 | 400 | 未勾选协议 |
| 40103 | 401 | 学号或密码错误 |
| 40303 | 403 | 账号已被禁用 |
| 42301 | 423 | 登录失败次数过多，账号临时锁定 |

### 业务规则

1. 登录前先校验验证码。
2. 验证码校验成功后写入 `auth_captchas.used_at`。
3. 学号或密码错误时增加 `users.login_failed_count`。
4. 连续失败达到阈值后设置 `users.locked_until`。
5. 登录成功后更新 `users.last_login_at` 并清零 `login_failed_count`。
6. JWT payload 必须包含 `sub`、`student_no`、`role`、`token_version`、`iat`、`exp`。
7. 不提供注册接口。

## 5.4 修改当前用户密码

### 基本信息

- 方法：PATCH
- 路径：`/api/v1/auth/password`
- 权限：需要登录
- 说明：当前用户修改自己的登录密码。

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| old_password | string | 是 | 原密码 |
| new_password | string | 是 | 新密码 |
| confirm_password | string | 是 | 确认新密码 |

### 请求示例

```json
{
  "old_password": "123456",
  "new_password": "CatMap@2026",
  "confirm_password": "CatMap@2026"
}
```

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "must_change_password": false,
    "token_invalidated": true,
    "next_action": "complete_profile"
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 错误码

| code | HTTP | 说明 |
|---:|---:|---|
| 40001 | 400 | 新密码格式不合法 |
| 40103 | 401 | 原密码错误 |
| 40102 | 401 | Token 无效或已过期 |

### 业务规则

1. 修改成功后更新 `password_hash`。
2. 修改成功后设置 `must_change_password = false`。
3. 修改成功后更新 `password_updated_at`。
4. 修改成功后 `token_version + 1`，旧 Token 失效。
5. 前端应引导用户重新登录，或后端返回新 Token。V1 建议要求重新登录，逻辑更简单。

## 5.5 解绑当前用户自己的微信

### 基本信息

- 方法：DELETE
- 路径：`/api/v1/auth/wechat-binding`
- 权限：需要登录，包括管理员
- 说明：解绑当前 Token 所属用户自己的微信，不接受其他用户 ID。

### 请求参数

无。

### 响应示例

```json
{
  "code": 0,
  "message": "微信绑定已清除",
  "data": {
    "user_id": "0d65df4f-10c0-45e4-a48d-b04e0af7d3d2",
    "wechat_bound": false,
    "token_version": 3,
    "token_invalidated": true
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 错误码

| code | HTTP | 说明 |
|---:|---:|---|
| 40001 | 400 | 当前账号尚未绑定微信 |
| 40101 | 401 | 未登录 |
| 40102 | 401 | Token 无效或已过期 |

### 业务规则

1. 后端从 JWT 取得本人，不允许指定其他用户。
2. 普通成员和管理员均可解绑本人。
3. 成功后清空三个微信字段并执行 `token_version + 1`。
4. 前端二次确认必须明确提示“解绑后将立即退出登录”；成功后清除本地会话并跳转登录页。
5. 未绑定时不修改账号和会话状态。

## 5.6 退出登录

### 基本信息

- 方法：POST
- 路径：`/api/v1/auth/logout`
- 权限：需要登录
- 说明：退出当前登录态。

### 请求参数

无。

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": null,
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 业务规则

V1 没有 refresh token 和 session 表，退出登录主要由前端清除本地 Token。后续如果引入 `auth_sessions`，可在此接口中使服务端会话失效。

| department | string | 否 | 猫协部门：生存保障部、活动部、宣传部、秘书部、养护部 |
| contact_info | string | 否 | 手机号，仅管理员可见 |

### 请求示例

```json
{
  "nickname": "小橘观察员",
  "avatar_url": "/uploads/avatar/u1.jpg",
```

### 错误码

| code | HTTP | 说明 |
|---:|---:|---|
| 63002 | 400 | 昵称不合法 |
| 63003 | 400 | 头像地址不合法 |

### 业务规则

1. 普通成员只能修改自己的资料。
2. 当前资料文字字段为 `nickname`、`department`、`contact_info`；头像通过 `/files/images` 独立提交审核并由服务器自动生效。
3. `nickname` 不超过 20 个字符；`contact_info` 使用手机号正则校验。
4. `contact_info` 仅管理员可见，用于任务协作时必要联系。
5. 真实姓名、加入时间等字段后续由管理员维护接口扩展。
6. 客户端个人资料页仅在昵称、部门或联系方式相对已保存快照变化后启用离页保护；头像上传成功即已暂存在后端，不作为未保存表单。确认放弃必须等待微信 `page-container.afterleave` 后再返回，取消只关闭一次弹窗并恢复编辑。
7. 保存文字资料不得覆盖当前页面的待审头像预览或把 `pending` 改成失败；头像审核失败在客户端按审核资产只提示一次，重新进入不常驻显示失败文案。

- 权限：需要登录
- 说明：当前用户接受管理员指派。

### 请求参数

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "assignment_id": "537d7a9b-79b3-4304-9a0e-31591314d597",
    "status": "accepted",
    "task_id": "9a3f6d8e-8aa7-4b69-8501-4c2b3619aa3d",

  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 错误码

| code | HTTP | 说明 |
|---:|---:|---|
| 63009 | 404 | 指派记录不存在 |
| 63010 | 409 | 指派状态已变化 |
| 63011 | 409 | 当前任务人数已满，无法接受指派 |

### 业务规则

1. 只能接受指派给自己的任务。
2. 只有 `pending` 状态可以接受。
3. 如果 `expires_at < now()`，应先置为 `expired`，再返回状态冲突。
4. 接受成功后：
   - `task_assignments.status = accepted`
   - `task_assignments.responded_at = now()`
   - 写入或更新 `task_participants`
   - 更新 `tasks.participant_count`
   - 写入任务动态
   - 生成通知

> 管理员成员管理接口统一放在 `/api/v1/admin/users`。
> 访问要求：已登录、`role IN ('admin', 'super_admin')`、`status = active`、`must_change_password = false`。

## 9.1 创建成员账号

### 基本信息

- 方法：POST
- 路径：`/api/v1/admin/users`
- 权限：管理员
- 说明：管理员创建成员账号。

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| student_no | string | 否 | 兼容字段，与 `meow_no` 同值；两者都不传时自动分配喵喵号 |
| meow_no | string | 否 | 可手动指定喵喵号；两者都不传时自动分配 |
| initial_password | string | 否 | 8-20 位；不传时默认与最终喵喵号相同 |
| role | string | 否 | 默认 `member`；普通管理员仅可创建 `member` / `summer_volunteer` |
| profile | object | 否 | 成员资料对象 |
| profile.nickname | string | 否 | 默认昵称 |
| profile.real_name | string | 否 | 真实姓名 |
| profile.department | string | 否 | 学院、部门或专业 |
| profile.grade | string | 否 | 年级 |
| profile.joined_at | string | 否 | 加入猫协日期 |
| must_change_password | boolean | 否 | 默认 `true` |

### 请求示例

```json
{
  "meow_no": "trmx0010",
  "initial_password": "trmx0010",
  "role": "member",
  "profile": {
    "nickname": "猫协成员0010",
    "real_name": "张三",
    "department": "计算机学院",
    "grade": "2023级",
    "joined_at": "2026-06-01"
  }
}
```

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": "0d65df4f-10c0-45e4-a48d-b04e0af7d3d2",
    "student_no": "trmx0010",
    "meow_no": "trmx0010",
    "role": "member",
    "status": "active",
    "must_change_password": true
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 错误码

| code | HTTP | 说明 |
|---:|---:|---|
| 40302 | 403 | 权限不足 |
| 40901 | 409 | 喵喵号已被当前账号使用 |
| 40904 | 409 | 喵喵号属于可恢复的软删除账号 |
| 64004 | 400 | 角色不合法 |

### 业务规则

1. 创建账号时写入 `users`。
2. 同步创建 `user_profiles`。
3. 默认 `must_change_password = true`。
4. 默认 `token_version = 1`。
5. 重要操作写入 `admin_operation_logs`。
6. 喵喵号对软删除记录仍保持唯一；发现历史账号时不新增记录，返回 `40904`、旧昵称和原账号 ID 供管理员确认恢复。
7. 自动分配使用独立 PostgreSQL 序列 `meow_no_auto_sequence`，并跳过已占用号码；手动指定高位喵喵号不会推进该序列。例如自动号已使用到 `trmx0009`，手动创建 `trmx2313` 后，下一个自动号仍为 `trmx0010`。
8. 序列由迁移 `20260716_0015` 创建；手动指定的喵喵号仍需满足唯一约束。

## 9.1.1 重新启用软删除成员账号

- 方法：POST
- 路径：`/api/v1/admin/users/{user_id}/restore`
- 权限：管理员
- 说明：恢复创建接口发现的历史账号，保留原 `user.id`、个人资料和历史业务关联。

请求体可传 `initial_password`；留空时使用喵喵号。恢复操作将账号设为 `active`、清除 `deleted_at`、重置登录锁定和密码、强制首次改密、清除旧微信绑定、递增 `token_version`，并写入 `user_restore` 管理员操作日志。创建页本次填写的昵称、部门和角色不会覆盖历史资料。

# 10. 前端页面调用建议

## 10.1 登录页

小程序启动自动登录顺序：

```text
uni.login(provider = weixin)
        ↓
POST /auth/wechat/login
        ↓
成功：保存 JWT 并按用户状态跳转
未绑定：清理旧会话并进入喵喵号密码登录页
```

喵喵号密码登录顺序：

```text
GET /auth/captcha
        ↓
用户点击登录，通过表单校验
        ↓
uni.login(provider = weixin)，POST /auth/login（`agree_wechat_bind=false`）
        ↓
已绑定且匹配：直接根据 next_action 跳转
未绑定：后端返回 40006，再弹出原生确认框
        ↓
取消：停留登录页；确认：重新 uni.login 并以 `agree_wechat_bind=true` 重试
        ↓
根据 next_action 跳转
```

若用户确认后 `uni.login` 暂不可用，前端提示重试，不提交空 `wechat_code`，也不降级为未绑定的密码登录。生产后端仍使用 `optional` 模式兼容尚未升级的前端和紧急回滚。

登录页仅展示一个《团绒喵记本小程序隐私保护指引》链接，并调用微信官方 `wx.openPrivacyContract()` 打开公众平台配置的指引全文；不再展示内容为空或仅作占位的自定义《用户协议》《隐私政策》《校园猫协成员规范》页面。勾选状态仍按原流程提交为 `agree_terms`，不改变密码登录和微信绑定逻辑。

跳转规则：

|---|---|
| change_password | 强制修改密码页 |
| complete_profile | 初始化资料页 |

# 13. 头衔系统当前接口（2026-07-18）

以下为当前代码实现，优先级高于本文早期规划段落。所有响应继续使用统一 `{code,message,data,trace_id}` 信封。

## 13.1 通用头衔字段

`GET /auth/me` 的 `profile`、`GET|PATCH /profile/me`、`GET /me/dashboard` 的 `profile`、管理员成员详情和列表的 `profile` 均返回：

```json
{
  "title": "vice_president",
  "title_label": "副会长",
  "title_shield": "purple"
}
```

无头衔时三个字段均为 `null`。

## 13.2 查询头衔占用目录

`GET /api/v1/admin/titles`，仅会长可访问。`data.items` 固定返回 12 个非空头衔；每项包含 `key`、`label`、`shield`、`is_available`，占用时另含 `holder.user_id/meow_no/nickname`。

## 13.3 授予或剥夺头衔

`PATCH /api/v1/admin/users/{user_id}/title`，仅会长可调用。

```json
{ "title": "publicity_head" }
```

传 `null` 或 `none` 表示剥夺。该接口不接受 `president`，也不能修改当前会长；成功返回完整管理员成员详情。

## 13.4 转让会长

`POST /api/v1/admin/titles/transfer`，仅当前会长可调用。

```json
{ "successor_id": "uuid" }
```

成功返回 `previous_president` 与 `successor` 两份管理员成员详情。转让与接任者角色提升在同一事务中完成。

## 13.5 主动退出头衔

`POST /api/v1/profile/me/title/resign`。无请求体；无头衔时幂等成功，会长调用返回 403/63012。

## 13.6 创建成员时指定头衔

`POST /api/v1/admin/users` 的 `profile` 可增加 `title`。只有会长可以提交非空头衔，且头衔必须空闲并且不是 `president`；普通管理员提交非空值返回 403/63010。

## 13.7 头衔错误码

| code | 含义 | HTTP |
|---|---|---|
| 63009 | 头衔已被占用 | 409 |
| 63010 | 需要会长权限 | 403 |
| 63011 | 头衔或流转方式非法 | 422 |
| 63012 | 会长必须先完成转让 | 403 |
