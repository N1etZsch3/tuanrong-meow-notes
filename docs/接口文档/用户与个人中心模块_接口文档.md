# 校园猫协地图任务系统｜用户与个人中心模块接口文档

> 版本：V1.1.1 接口设计稿
> Base URL：`/api/v1`
> 适用范围：Auth、Profile、Me、Notifications、Assignments、Admin Users
> 字段命名：统一使用 `snake_case`
> 响应结构：统一使用 `{ code, message, data, trace_id }`

---

# 1. 接口范围

本文档覆盖用户与个人中心模块相关接口：

1. 登录验证码。
2. 喵喵号密码登录与微信 OpenID 绑定登录。
3. 当前用户信息。
4. 修改密码。
5. 退出登录。
6. 当前用户资料查询与编辑。
7. 用户资料初始化。
8. 任务偏好查询与保存。
9. 个人中心首页。
10. 我的任务。
11. 我的打卡记录。
12. 我的观察记录。
13. 我的排班。
14. 指派给我的任务。
15. 通知中心。
16. 个人统计。
17. 管理员成员账号管理。

不提供成员自主注册接口。1.1.1 起，成员首次仍需通过喵喵号密码登录，绑定微信 OpenID 后可在小程序启动时自动登录；管理员清除绑定后，成员回到喵喵号密码兜底登录并重新绑定。

# 2. 通用约定

## 2.1 统一响应结构

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

## 2.2 鉴权 Header

除公开接口外，统一使用：

```http
Authorization: Bearer <access_token>
```

## 2.3 分页参数

列表接口统一使用：

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---:|---|
| page | int | 1 | 当前页，从 1 开始 |
| page_size | int | 20 | 每页数量，最大 100 |

分页响应：

```json
{
  "items": [],
  "page": 1,
  "page_size": 20,
  "total": 0,
  "has_more": false
}
```

## 2.4 首次登录改密限制

当当前用户：

```text
must_change_password = true
```

只允许访问：

```http
GET /api/v1/auth/me
PATCH /api/v1/auth/password
POST /api/v1/auth/logout
```

访问其他业务接口返回：

```json
{
  "code": 40301,
  "message": "请先修改初始密码",
  "data": null,
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

## 2.5 资料初始化限制

当当前用户：

```text
profile_completed = false
```

建议只允许访问：

```http
GET /api/v1/auth/me
GET /api/v1/profile/me
PATCH /api/v1/profile/me
POST /api/v1/profile/me/complete
POST /api/v1/auth/logout
```

访问其他业务接口可以返回：

```json
{
  "code": 63006,
  "message": "请先完成个人资料初始化",
  "data": null,
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

---

## 2.6 微信 OpenID 绑定登录

1. 前端只提交微信临时 code，OpenID 由后端使用 `code2Session` 获取。
2. 首次绑定前必须通过喵喵号、密码和验证码校验，保持邀请制。
3. 绑定前必须告知用户“当前微信将与该喵喵号绑定，用于后续自动登录和账号保护”。
4. 已绑定用户启动小程序时可先尝试微信自动登录，成功后继续沿用 `must_change_password` 和 `profile_completed` 的页面流转。
5. 管理员清除绑定后，后端递增 `token_version`，成员下一次需用喵喵号密码重新登录并绑定。
6. OpenID 仅用于本小程序的自动登录和账号保护，保存在后端 `users.wechat_openid`；前端不接收、不缓存、不展示 OpenID 或 session key。
7. 管理员解绑会清除后端微信标识及绑定时间；正式发布前需在小程序隐私保护指引和平台配置中说明该标识的用途与处理方式，并完成真机验收。

# 5. Auth 接口

## 5.1 获取登录验证码

### 基本信息

- 方法：GET
- 路径：`/api/v1/auth/captcha`
- 权限：公开
- 说明：获取登录页字母验证码。

### 请求参数

无。

### 响应字段

| 字段 | 类型 | 说明 |
|---|---|---|
| captcha_id | string | 验证码 ID |
| image_base64 | string | 验证码图片 base64 |
| expires_in | int | 有效秒数 |

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "captcha_id": "b5f7a6d8-2a3e-4f91-98e5-4c22f47a1f31",
    "image_base64": "data:image/png;base64,iVBORw0KGgoAAA...",
    "expires_in": 300
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 错误码

| code | 说明 |
|---:|---|
| 50001 | 服务端异常 |

### 业务规则

1. 后端生成验证码明文后，只保存哈希到 `auth_captchas.code_hash`。
2. 验证码默认 5 分钟有效。
3. 验证码只能使用一次。

---

## 5.2 用户登录

### 基本信息

- 方法：POST
- 路径：`/api/v1/auth/login`
- 权限：公开
- 说明：使用学号、密码、字母验证码登录。

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| meow_no | string | 是 | 喵喵号，格式如 `trmx0001` |
| student_no | string | 否 | 兼容旧字段，含义同喵喵号 |
| password | string | 是 | 密码 |
| captcha_id | string | 是 | 验证码 ID |
| captcha_code | string | 是 | 用户输入的验证码 |
| agreement_checked | boolean | 是 | 是否勾选协议 |

### 请求示例

```json
{
  "meow_no": "trmx0001",
  "password": "123456",
  "captcha_id": "b5f7a6d8-2a3e-4f91-98e5-4c22f47a1f31",
  "captcha_code": "ABCD",
  "agreement_checked": true
}
```

### 响应字段

| 字段 | 类型 | 说明 |
|---|---|---|
| access_token | string | JWT Token |
| token_type | string | 固定为 `Bearer` |
| expires_in | int | Token 有效秒数 |
| user | object | 当前用户摘要 |
| profile | object | 当前用户资料 |
| next_action | string | 前端下一步动作 |

`next_action` 可选：

| 值 | 说明 |
|---|---|
| change_password | 必须先改密 |
| complete_profile | 必须先初始化资料 |
| enter_app | 进入应用 |

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.xxx",
    "token_type": "Bearer",
    "expires_in": 7200,
    "user": {
      "id": "0d65df4f-10c0-45e4-a48d-b04e0af7d3d2",
      "student_no": "trmx0001",
      "meow_no": "trmx0001",
      "role": "member",
      "status": "active",
      "must_change_password": true,
      "profile_completed": false
    },
    "profile": {
      "nickname": "猫协成员1010",
      "avatar_url": null,
      "department": null,
      "profile_completed": false
    },
    "next_action": "change_password"
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 错误码

| code | HTTP | 说明 |
|---:|---:|---|
| 40003 | 400 | 验证码错误 |
| 40004 | 400 | 验证码已过期 |
| 40005 | 400 | 未勾选协议 |
| 40103 | 401 | 学号或密码错误 |
| 40303 | 403 | 账号已被禁用 |
| 42301 | 423 | 登录失败次数过多，账号临时锁定 |

### 业务规则

1. 登录前先校验验证码。
2. 验证码校验成功后写入 `auth_captchas.used_at`。
3. 学号或密码错误时增加 `users.login_failed_count`。
4. 连续失败达到阈值后设置 `users.locked_until`。
5. 登录成功后更新 `users.last_login_at` 并清零 `login_failed_count`。
6. JWT payload 必须包含 `sub`、`student_no`、`role`、`token_version`、`iat`、`exp`。
7. 不提供注册接口。

## 5.4 修改当前用户密码

### 基本信息

- 方法：PATCH
- 路径：`/api/v1/auth/password`
- 权限：需要登录
- 说明：当前用户修改自己的登录密码。

### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| old_password | string | 是 | 原密码 |
| new_password | string | 是 | 新密码 |
| confirm_password | string | 是 | 确认新密码 |

### 请求示例

```json
{
  "old_password": "123456",
  "new_password": "CatMap@2026",
  "confirm_password": "CatMap@2026"
}
```

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "must_change_password": false,
    "token_invalidated": true,
    "next_action": "complete_profile"
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 错误码

| code | HTTP | 说明 |
|---:|---:|---|
| 40001 | 400 | 新密码格式不合法 |
| 40103 | 401 | 原密码错误 |
| 40102 | 401 | Token 无效或已过期 |

### 业务规则

1. 修改成功后更新 `password_hash`。
2. 修改成功后设置 `must_change_password = false`。
3. 修改成功后更新 `password_updated_at`。
4. 修改成功后 `token_version + 1`，旧 Token 失效。
5. 前端应引导用户重新登录，或后端返回新 Token。V1 建议要求重新登录，逻辑更简单。

---

## 5.5 退出登录

### 基本信息

- 方法：POST
- 路径：`/api/v1/auth/logout`
- 权限：需要登录
- 说明：退出当前登录态。

### 请求参数

无。

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": null,
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 业务规则

V1 没有 refresh token 和 session 表，退出登录主要由前端清除本地 Token。后续如果引入 `auth_sessions`，可在此接口中使服务端会话失效。

```json
{
```

```json
    "assignment_id": "537d7a9b-79b3-4304-9a0e-31591314d597",

  },
```

```json
}
```

### 错误码

| code | HTTP | 说明 |
|---:|---:|---|
| 63009 | 404 | 指派记录不存在 |
| 63010 | 409 | 指派状态已变化 |
| 63011 | 409 | 当前任务人数已满，无法接受指派 |

### 业务规则

3. 如果 `expires_at < now()`，应先置为 `expired`，再返回状态冲突。
4. 接受成功后：
   - `task_assignments.status = accepted`
   - `task_assignments.responded_at = now()`
   - 写入或更新 `task_participants`
   - 更新 `tasks.participant_count`
   - 写入任务动态
   - 生成通知

## 9.8 清除成员微信绑定

### 基本信息

- 方法：DELETE
- 路径：`/api/v1/admin/users/{user_id}/wechat-binding`
- 权限：管理员
- 说明：管理员为成员清除当前微信绑定，使成员重新通过喵喵号密码登录并绑定微信。

```json

  "data": {
    "user_id": "0d65df4f-10c0-45e4-a48d-b04e0af7d3d2",
```

4. 管理员操作日志只记录是否已绑定，不记录 OpenID 原值。
5. 成员下次使用喵喵号密码登录时需要再次确认并绑定当前微信。

# 10. 前端页面调用建议

## 10.1 登录页

小程序启动自动登录顺序：

```text
uni.login(provider = weixin)
        ↓
POST /auth/wechat/login
        ↓
成功：保存 JWT 并按用户状态跳转
未绑定：清理旧会话并进入喵喵号密码登录页
```

喵喵号密码登录顺序：

```text
GET /auth/captcha
        ↓
显示并确认“当前微信将与该喵喵号绑定”
        ↓
uni.login(provider = weixin)
        ↓
POST /auth/login（包含 wechat_code、agree_wechat_bind）
        ↓
根据 next_action 跳转
```

若 `uni.login` 暂不可用，前端不提交 `wechat_code`，也不把 `agree_wechat_bind` 标记为 true；生产首次部署使用 `optional` 模式保留原密码登录兜底。

跳转规则：

| next_action | 页面 |
|---|---|
| change_password | 强制修改密码页 |
| complete_profile | 初始化资料页 |
| enter_app | 地图首页 |

成功后后端返回新的 access token，前端替换旧 token 后进入资料初始化页。
