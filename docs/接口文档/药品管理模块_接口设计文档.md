# 校园猫协地图任务系统｜药品管理模块接口设计文档

> 版本：V1.0
> 模块名称：药品管理模块
> Base URL：`/api/v1`
> 适用范围：药品主档、成员持有库存、库存流水、用药申请、申请审核、药品归档与删除
> 字段命名：统一使用 `snake_case`
> 响应结构：统一使用 `{ code, message, data, trace_id }`
> 登录方式：复用系统 JWT 鉴权。

---

# 1. 接口范围

药品管理模块接口覆盖以下能力：

1. 药品分类查询与管理。
2. 药品主档列表、搜索、详情。
3. 新增药品主档并创建持有库存。
4. 选择已有药品主档并新增持有库存。
5. 持有人库存详情。
6. 药品动态 / 库存流水查询。
7. 持有人记录购入、使用、报废。
8. 持有人分配药品。
9. 持有人转交库存。
10. 用药申请提交。
11. 用药申请审核、拒绝、取消、失效。
12. 管理员编辑药品主档。
13. 管理员归档或删除无库存药品。
14. 管理员删除成员持有库存。

V1 暂不重点实现：

1. 任务创建时预指定药品。
2. 任务搜索完整联动。
3. WebSocket 实时通知。
4. 药品有效期和批次。
5. 药品财务统计。
6. 独立药品流水页高级筛选。

# 2. 通用约定

## 2.1 统一响应结构

所有接口统一返回：

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

失败示例：

```json
{
  "code": 66005,
  "message": "库存不足",
  "data": null,
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

## 2.2 鉴权方式

除特殊说明外，药品模块所有接口都需要登录。

```http
Authorization: Bearer <access_token>
```

## 2.3 分页参数

列表接口统一使用：

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---:|---|
| `page` | int | 1 | 当前页，从 1 开始 |
| `page_size` | int | 20 | 每页数量，最大 100 |

分页响应：

```json
{
  "items": [],
  "page": 1,
  "page_size": 20,
  "total": 0,
  "has_more": false
}
```

## 2.4 时间格式

所有时间统一使用 ISO 8601 / RFC3339。

示例：

```json
{
  "created_at": "2026-07-06T20:00:00+08:00"
}
```

## 2.5 数量格式

数量字段统一使用 number。

示例：

```json
{
  "quantity": 3,
  "current_quantity": 12,
  "total_in_quantity": 30
}
```

# 4. 枚举定义

## 4.1 medicine_status 药品主档状态

| 值 | 说明 |
|---|---|
| `active` | 正常使用 |
| `archived` | 已归档 |
| `deleted` | 已软删除 |

## 4.2 holding_status 持有库存状态

| 值 | 说明 |
|---|---|
| `active` | 正常持有 |
| `empty` | 当前库存为 0 |
| `transferred` | 已转交或合并 |
| `deleted` | 已软删除 |

## 4.3 stock_status 库存状态

| 值 | 说明 |
|---|---|
| `sufficient` | 库存充足 |
| `low` | 库存紧张 |
| `critical` | 库存告急 |
| `empty` | 无库存 |

## 4.4 holding_relation 持有关系筛选

| 值 | 说明 |
|---|---|
| `all` | 全部药品 |
| `mine` | 我持有的 |
| `others` | 他人持有的 |

## 4.5 medicine_sort_by 排序字段

| 值 | 说明 |
|---|---|
| `recent_activity` | 最近有动态的药 |
| `name` | 药品名称 |
| `total_current_quantity` | 当前总库存 |
| `created_at` | 创建时间 |

## 4.6 stock_operation_type 库存操作类型

| 值 | 说明 |
|---|---|
| `initial_in` | 初始入库 |
| `purchase` | 购入 |
| `use_self` | 使用 |
| `scrap` | 报废 |
| `distribute_out` | 分配转出 |
| `distribute_in` | 分配转入 |
| `transfer_out` | 转交转出 |
| `transfer_in` | 转交转入 |
| `application_use` | 申请使用 |
| `adjustment` | 库存校正 |
| `archive` | 归档记录 |
| `delete_holding` | 删除持有库存记录 |

## 4.7 application_status 用药申请状态

| 值 | 说明 |
|---|---|
| `pending` | 待审核 |
| `approved` | 已通过 |
| `rejected` | 已拒绝 |
| `cancelled` | 已取消 |
| `expired` | 已失效 |

# 6. 接口总览

## 6.1 成员端接口

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/medicine-categories` | 获取药品分类 |

| GET | `/medicine-holdings/{holding_id}` | 获取持有人库存详情 |
| GET | `/medicine-holdings/{holding_id}/logs` | 获取某持有人库存动态 / 流水 |
| POST | `/medicine-holdings/{holding_id}/purchase` | 记录购入 |
| POST | `/medicine-holdings/{holding_id}/use` | 记录使用 |
| POST | `/medicine-holdings/{holding_id}/scrap` | 记录报废 |
| POST | `/medicine-holdings/{holding_id}/distribute` | 分配给其他成员 |

| POST | `/medicine-holdings/{holding_id}/adjust` | 库存校正 |
| POST | `/medicine-holdings/{holding_id}/applications` | 提交用药申请 |
| GET | `/medicine-applications` | 查询用药申请列表 |
| GET | `/medicine-applications/{application_id}` | 查看用药申请详情 |
| POST | `/medicine-applications/{application_id}/approve` | 通过用药申请 |
| POST | `/medicine-applications/{application_id}/reject` | 拒绝用药申请 |
| POST | `/medicine-applications/{application_id}/cancel` | 取消用药申请 |

| 方法 | 路径 | 说明 |
|---|---|---|

# 8. 药品主档与列表接口

## 8.1 获取药品列表

### 基本信息

```http
GET /api/v1/medicines
```

| 项目 | 内容 |
|---|---|
| 权限 | 登录用户 |
| 说明 | 获取药品主档维度列表，包含总库存和持有人标签 |
| 涉及表 / 视图 | `v_medicine_catalog_summary`、`medicine_holdings`、`user_profiles` |

### Query 参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---:|---|---|
| `keyword` | string | 否 | null | 搜索药品名称、别名、分类、说明、持有人昵称 |
| `category_id` | uuid | 否 | null | 药品分类 |
| `holding_relation` | string | 否 | `all` | `all` / `mine` / `others` |
| `sort_by` | string | 否 | `recent_activity` | 排序字段 |
| `sort_order` | string | 否 | `desc` | `asc` / `desc` |
| `include_archived` | boolean | 否 | false | 是否包含归档药品，管理员可用 |
| `page` | int | 否 | 1 | 页码 |
| `page_size` | int | 否 | 20 | 每页数量 |

### 请求示例

```http
GET /api/v1/medicines?category_id=uuid&holding_relation=mine&sort_by=recent_activity&page=1&page_size=20
Authorization: Bearer <access_token>
```

### 响应示例

```json
{

        "total_in_quantity": 80,
        "holder_count": 3,
        "stock_status": "sufficient",
        "stock_status_label": "库存充足",
        "last_operation_at": "2026-07-06T20:00:00+08:00",
        "holders": [
          {
            "holding_id": "uuid",
            "holder_id": "uuid",
            "holder_nickname": "小橘观察员",
            "current_quantity": 12,
            "unit": "片"
          }
        ]
      }
    ],
    "page": 1,
    "page_size": 20,
    "total": 1,
```

### 业务规则

1. 普通成员可查看所有未归档药品。
2. `holding_relation = mine` 表示当前用户是持有人之一的药品。
3. `holding_relation = others` 表示当前用户不是持有人，但其他成员持有的药品。
4. 库存状态由当前总库存和累计入库动态计算。
5. 默认不返回已归档药品。

## 8.2 搜索已有药品主档

### 基本信息

```http
GET /api/v1/medicines/search
```

| 项目 | 内容 |
|---|---|
| 权限 | 登录用户 |

### Query 参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `keyword` | string | 是 | 搜索关键词 |
| `limit` | int | 否 | 默认 10，最大 20 |

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "items": [
      {
        "medicine_id": "uuid",
        "name": "阿莫西林",
        "category_name": "抗生素",
        "specification": "250mg/片",
        "unit": "片",
        "description": "常用抗生素。",
        "usage_notes": "遵医嘱使用。",
        "cover_image_url": "/uploads/medicine/amoxicillin.jpg"
      }
    ]
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 业务规则

1. 用于“输入药品名，有结果则选择已有主档”的表单交互。
2. 已归档和已删除药品默认不返回。
3. 选择已有主档后，前端只允许填写持有库存信息，主档字段只读展示。

## 8.3 创建药品或新增持有库存

### 基本信息

```http
POST /api/v1/medicines
```

| 项目 | 内容 |
|---|---|
| 权限 | 登录用户 |
| 说明 | 支持创建全新药品主档，也支持选择已有主档后新增持有库存 |
| 涉及表 | `medicine_catalogs`、`medicine_holdings`、`medicine_stock_logs` |

### 请求体

请求分两种模式。

#### 模式一：选择已有药品主档

```json
{
  "medicine_id": "uuid",
  "holder_id": null,
  "initial_quantity": 20,
  "remark": "自己手里现有库存"
}
```

#### 模式二：创建全新药品主档

```json
{
  "medicine_id": null,
  "catalog": {
    "name": "阿莫西林",
    "category_id": "uuid",
    "specification": "250mg/片",
    "unit": "片",
    "description": "常用抗生素。",
    "usage_notes": "遵医嘱使用。",
    "cover_image_url": "/uploads/medicine/amoxicillin.jpg"
  },
  "holder_id": null,
  "initial_quantity": 20,
  "remark": "第一次建档"
}
```

### 请求字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `medicine_id` | uuid/null | 否 | 已有药品主档 ID；为空表示创建新主档 |
| `catalog` | object | 条件必填 | 创建新主档时必填 |

### 请求体

```json
{
  "cancel_reason": "线下已解决，不需要用药。"
}
```

### 业务规则

1. 只有申请人可以取消自己的申请。
2. 只能取消 `pending` 状态申请。
3. 已通过、已拒绝、已失效的申请不能取消。
4. 取消后库存不变。

## 12.5 归档药品主档

### 基本信息

```http
POST /api/v1/admin/medicines/{medicine_id}/archive
```

| 项目 | 内容 |
|---|---|
| 权限 | 管理员 |
| 说明 | 药品无库存后归档主档 |
| 涉及表 | `medicine_catalogs`、`medicine_holdings`、`medicine_use_applications`、`medicine_stock_logs` |

### 请求体

```json
{
  "archive_reason": "该药品已无库存，暂不继续使用。"
}
```

### 业务规则

1. 所有持有人当前库存必须为 0。
2. 不能存在 `pending` 用药申请。
3. 药品状态变为 `archived`。
4. 写入 `archive` 流水。
5. 归档后不出现在默认药品列表。

## 12.6 删除药品主档

### 基本信息

```http
DELETE /api/v1/admin/medicines/{medicine_id}
```

| 项目 | 内容 |
|---|---|
| 权限 | 管理员 |
| 说明 | 软删除药品主档 |
| 涉及表 | `medicine_catalogs` |

### Query 参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `reason` | string | 是 | 删除原因 |

### 业务规则

1. 删除应谨慎，优先归档。
2. 当前药品所有持有人库存必须为 0。
3. 不能存在待审核申请。
4. 如果存在历史流水或任务关联，仍建议只归档。
5. 删除采用软删除，写入 `deleted_at`，不物理删除。

---

## 12.7 删除成员持有库存

### 基本信息

```http
DELETE /api/v1/admin/medicine-holdings/{holding_id}
```

| 项目 | 内容 |
|---|---|
| 权限 | 管理员 |
| 说明 | 删除某位成员的持有库存记录，用于成员退会后数据清理 |
| 涉及表 | `medicine_holdings`、`medicine_stock_logs` |

### Query 参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `reason` | string | 是 | 删除原因 |

### 业务规则

1. 如果当前库存大于 0，不允许直接删除，返回 `66022`。
2. 非空库存应先转交或分配给其他成员。
3. 删除采用软删除。
4. 写入 `delete_holding` 流水。
5. 历史流水仍可追溯。

# 13. 任务关联预留

V1 保留以下字段：

```text
related_task_id
related_task_snapshot
```

使用场景：

1. 持有人记录使用时关联任务。
2. 申请人提交用药申请时关联任务。
3. 申请通过后生成的流水关联任务。

V1 处理建议：

1. 接口允许传 `related_task_id = null`。
2. 任务搜索功能可以暂不做。
3. 如果前端已经有任务 ID，可以传入并保存。
4. 任务动态同步可以后置。
5. 药品流水中保留任务链接字段即可。

# 14. 前端调用流程

## 14.1 药品列表页

```text
1. 调用 GET /medicine-categories 获取分类
2. 调用 GET /medicines 获取药品列表
3. 用户切换分类 / 持有关系 / 最近动态排序时重新请求列表
4. 点击药品卡片进入 GET /medicines/{medicine_id}
5. 点击持有人标签进入 GET /medicine-holdings/{holding_id}
```

## 14.2 新增药品页

```text
1. 用户输入药品名称
2. 调用 GET /medicines/search?keyword=xxx
3. 如果选择已有主档：填充主档信息为只读，只填写初始库存
4. 如果没有选择主档：填写完整药品主档信息和初始库存
5. 调用 POST /medicines
6. 创建成功后跳转药品详情或持有人库存详情
```

## 14.3 持有人记录库存变化

```text
1. 进入 GET /medicine-holdings/{holding_id}
2. 判断 permissions.can_record
3. 点击“记录”按钮打开模态框
4. 根据 operation_type 展示不同表单
5. 根据操作调用 purchase / use / scrap / distribute / transfer / adjust 接口
6. 成功后刷新持有人库存详情和药品详情动态
```

## 14.4 非持有人申请用药

```text
1. 进入 GET /medicine-holdings/{holding_id}
2. 判断 permissions.can_apply
3. 点击“申请”按钮打开模态框
4. 填写申请数量、申请原因、详细说明、使用时间
5. 调用 POST /medicine-holdings/{holding_id}/applications
6. 申请进入 pending 状态，24 小时未处理后失效
```

## 14.5 持有人审核申请

```text
1. 调用 GET /medicine-applications?scope=to_review&status=pending
2. 点击申请查看详情
3. 通过：POST /medicine-applications/{id}/approve
4. 拒绝：POST /medicine-applications/{id}/reject
5. 成功后刷新库存和申请列表
```
