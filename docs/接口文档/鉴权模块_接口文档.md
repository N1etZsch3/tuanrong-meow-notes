# 校园猫协地图任务系统｜鉴权模块接口文档

> 版本：V1 鉴权接口设计稿
> Base URL：`/api/v1`
> 登录方式：喵喵号 + 密码 + 字母验证码 + JWT
> 账号来源：管理员创建或批量导入

---

## 1. 模块边界

### 1.1 V1 做什么

| 功能 | 是否 V1 做 | 说明 |
|---|---:|---|
| 获取字母验证码 | 是 | 登录页展示 |
| 喵喵号密码登录 | 是 | 内部成员登录 |
| 获取当前用户 | 是 | 前端恢复登录态 |
| 修改自己的密码 | 是 | 默认密码登录后必须改密 |
| 退出登录 | 是 | 前端清 token，后端可轻量返回成功 |
| 管理员创建成员账号 | 是 | 因为 V1 不开放注册 |
| 管理员重置成员密码 | 是 | 替代找回密码 |
| 管理员禁用/恢复账号 | 是 | 内部成员管理 |

### 1.2 V1 不做什么

```text
成员自主注册
手机号登录
邮箱登录
微信 / QQ / 第三方登录
找回密码
refresh token
复杂会话管理
```

## 2. 接口总览

### 2.1 公开接口

| 方法 | 路径 | 说明 |
|---|---|---|

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/admin/users` | 查询成员账号列表 |

| PATCH | `/admin/users/{user_id}/status` | 禁用/恢复账号 |
| PATCH | `/admin/users/{user_id}/role` | 调整成员角色 |

## 4. 登录状态与权限规则

### 4.1 Token Header

需要登录的接口统一传：

```http
Authorization: Bearer <access_token>
```

### 4.2 JWT Payload

```json
{
  "sub": "0d65df4f-10c0-45e4-a48d-b04e0af7d3d2",
  "student_no": "trmx0001",
  "meow_no": "trmx0001",
  "role": "member",
  "token_version": 1,
  "iat": 1781750000,
  "exp": 1782354800
}
```

### 4.3 首次登录强制改密

当用户：

```json
{
  "must_change_password": true
}
```

前端应跳转到修改密码页。

后端只允许访问：

```http
GET /api/v1/auth/me
PATCH /api/v1/auth/password
POST /api/v1/auth/logout
```

访问其他接口返回：

```json
{
  "code": 40301,
  "message": "请先修改初始密码",
  "data": null,
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

# 5. 公开接口

## 5.1 获取登录验证码

### 基本信息

```http
GET /api/v1/auth/captcha
```

| 项目 | 内容 |
|---|---|
| 权限 | 公开 |
| 说明 | 生成登录页字母验证码 |
| 涉及表 | `auth_captchas` |

### Query 参数

无。

### 响应字段

| 字段 | 类型 | 说明 |
|---|---|---|
| captcha_id | string | 验证码 ID，登录时提交 |
| captcha_image | string | base64 图片，可以是 SVG 或 PNG |
| expires_in | int | 有效期，单位秒 |

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "captcha_id": "b5f7a6d8-2a3e-4f91-98e5-4c22f47a1f31",
    "captcha_image": "data:image/svg+xml;base64,PHN2ZyB4bWxucz0iLi4u",
    "expires_in": 300
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 业务规则

1. 验证码长度建议 4 位。
2. 只保存验证码 hash，不保存明文。
3. 默认有效期 5 分钟。
4. 同一个验证码登录成功后标记 `used_at`。
5. 前端点击“换一张”时重新请求本接口。

## 5.2 成员登录

### 基本信息

```http
POST /api/v1/auth/login
```

| 项目 | 内容 |
|---|---|
| 权限 | 公开 |
| 说明 | 使用学号、密码、验证码登录 |
| 涉及表 | `users`、`auth_captchas`、`user_profiles` |

### 请求体

```json
{
  "student_no": "20252160A1010",
  "password": "123456",
  "captcha_id": "b5f7a6d8-2a3e-4f91-98e5-4c22f47a1f31",
  "captcha_code": "A7KD",
  "agree_terms": true
}
```

### 请求字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| meow_no | string | 是 | 喵喵号，格式如 `trmx0001` |
| student_no | string | 否 | 兼容旧字段，含义同喵喵号 |
| password | string | 是 | 密码 |
| captcha_id | string | 是 | 验证码 ID |
| captcha_code | string | 是 | 用户输入的验证码 |
| agree_terms | boolean | 是 | 是否勾选协议 |

### 响应字段

| 字段 | 类型 | 说明 |
|---|---|---|
| access_token | string | JWT |
| token_type | string | 固定为 Bearer |
| expires_in | int | access token 有效期，单位秒 |
| must_change_password | boolean | 是否必须修改初始密码 |
| user | object | 当前用户信息 |

### 响应示例

```json
{
  "code": 0,
  "message": "login success",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.xxx",
    "token_type": "Bearer",
    "expires_in": 604800,
    "must_change_password": true,
    "user": {
      "id": "0d65df4f-10c0-45e4-a48d-b04e0af7d3d2",
      "student_no": "trmx0001",
      "meow_no": "trmx0001",
      "nickname": "金炜然",
      "avatar_url": null,
      "role": "member",
      "status": "active",
      "profile_completed": false
    },
    "next_action": "change_password"
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 后端处理流程

```text
1. 校验 agree_terms 是否为 true
2. 校验验证码是否存在、未过期、未使用
3. 校验验证码内容
4. 根据 student_no 查询 users
5. 校验用户状态 status 是否为 active
6. 校验 locked_until 是否仍在锁定期
7. 校验密码
8. 失败则增加 login_failed_count，必要时锁定账号
9. 成功则清空 login_failed_count 和 locked_until
10. 更新 last_login_at
11. 标记验证码 used_at
12. 签发 JWT，payload 写入 token_version
13. 返回 token 和用户信息
```

### 错误码

| code | HTTP | 说明 |
|---:|---:|---|
| 40002 | 400 | 缺少必要参数 |
| 40003 | 400 | 验证码错误 |
| 40004 | 400 | 验证码已过期 |
| 40005 | 400 | 未勾选协议 |
| 40103 | 401 | 学号或密码错误 |
| 40303 | 403 | 账号已被禁用 |
| 42301 | 423 | 登录失败次数过多，账号临时锁定 |

# 6. 登录后接口

## 6.1 获取当前登录用户

### 基本信息

```http
GET /api/v1/auth/me
Authorization: Bearer <access_token>
```

| 项目 | 内容 |
|---|---|
| 权限 | 登录用户 |
| 说明 | 前端刷新页面后恢复登录态，也用于判断管理员入口 |
| 涉及表 | `users`、`user_profiles` |

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": "0d65df4f-10c0-45e4-a48d-b04e0af7d3d2",
    "student_no": "20252160A1010",
    "role": "member",
    "status": "active",
    "must_change_password": true,
    "profile": {
      "nickname": "金炜然",
      "avatar_url": null,
      "real_name": "金炜然",
      "department": "计算机学院",
      "grade": "2025"
    }
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 业务规则

1. 不返回 `password_hash`。
2. 如果 token 过期或无效，返回 `40102`。
3. 如果 token 中的 `token_version` 与数据库不一致，返回 `40102`。
4. 如果 `role = admin`，前端可以在个人中心显示管理员入口。

## 6.2 续签 Access Token

### 基本信息

```http
POST /api/v1/auth/renew
Authorization: Bearer <access_token>
```

| 项目 | 内容 |
|---|---|
| 权限 | 已完成首次改密的登录用户 |
| 说明 | 用仍有效的 access token 换取新的 access token，新的有效期从续签时间重新计算 |
| 涉及表 | `users` |

### 请求体

无。

### 响应字段

| 字段 | 类型 | 说明 |
|---|---|---|
| access_token | string | 新签发的 access token |
| token_type | string | 固定为 Bearer |
| expires_in | int | 新 token 有效期，单位秒，当前为 604800 |

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.renewed",
    "token_type": "Bearer",
    "expires_in": 604800
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 业务规则

1. 只接受尚未过期、签名合法、`token_version` 与数据库一致的 access token。
2. 续签不引入 `refresh_tokens` 表，不保存服务端会话。
3. 新 token 的 `iat` 和 `exp` 从续签请求时间重新计算。
4. `must_change_password = true` 时不允许续签，仍只能访问 `/auth/me`、`/auth/password`、`/auth/logout`。
5. 前端建议在 token 剩余有效期小于 1 天时调用本接口；如果已经返回 401，则清空登录态并跳转登录页。

### 错误码

| code | HTTP | 说明 |
|---|---:|---|
| 40101 | 401 | 未登录 |
| 40102 | 401 | Token 无效或已过期 |
| 40301 | 403 | 请先修改初始密码 |
| 40303 | 403 | 账号已被禁用 |

## 6.3 修改自己的密码

### 基本信息

```http
PATCH /api/v1/auth/password
Authorization: Bearer <access_token>
```

| 项目 | 内容 |
|---|---|
| 权限 | 登录用户 |
| 说明 | 用户修改自己的密码，首次登录默认密码时必须调用 |
| 涉及表 | `users` |

### 请求体

```json
{
  "old_password": "123456",
  "new_password": "N1etzsche@2026",
  "confirm_password": "N1etzsche@2026"
}
```

### 请求字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| old_password | string | 是 | 旧密码 |
| new_password | string | 是 | 新密码 |
| confirm_password | string | 是 | 确认新密码 |

### 密码规则建议

| 规则 | 建议 |
|---|---|
| 最小长度 | 8 位 |
| 最大长度 | 20 位 |
| 字符要求 | 至少包含字母和数字，仅允许大小写字母、数字和 `@_!` |
| 与旧密码关系 | 不能与旧密码相同 |
| 空白字符 | 不允许全空白 |

### 响应示例

```json
{
  "code": 0,
  "message": "密码修改成功，请重新登录",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.new",
    "token_type": "Bearer",
    "expires_in": 604800,
    "must_change_password": false,
    "profile_completed": false,
    "token_invalidated": true,
    "next_action": "complete_profile"
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 后端处理流程

```text
1. 校验旧密码是否正确
2. 校验 new_password 和 confirm_password 是否一致
3. 校验新密码强度
4. 校验新密码不能与旧密码相同
5. 更新 users.password_hash
6. 更新 users.password_updated_at
7. 设置 users.must_change_password = false
8. users.token_version + 1
9. 清空 login_failed_count 和 locked_until
10. 返回新 access token，前端替换旧 token 并进入资料初始化页
```

### 错误码

| code | HTTP | 说明 |
|---:|---:|---|
| 40001 | 400 | 参数错误 |
| 40103 | 401 | 旧密码错误 |
| 40902 | 409 | 新密码不能与旧密码相同 |

## 7.2 创建成员账号

### 基本信息

```http
POST /api/v1/admin/users
Authorization: Bearer <admin_token>
```

| 项目 | 内容 |
|---|---|
| 权限 | 管理员 |
| 说明 | 创建内部成员账号 |
| 涉及表 | `users`、`user_profiles`、`admin_operation_logs` |

### 请求体

```json
{
  "student_no": "20252160A1010",
  "initial_password": "123456",
  "role": "member",
  "profile": {
    "nickname": "金炜然",
    "real_name": "金炜然",
    "department": "计算机学院",
    "grade": "2025",
    "joined_at": "2026-06-18"
  },
  "must_change_password": true
}
```

### 请求字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| student_no | string | 是 | 学号，唯一 |
| initial_password | string | 是 | 初始密码 |
| role | string | 否 | 默认 member |
| profile.nickname | string | 是 | 昵称 |
| profile.real_name | string | 否 | 真实姓名 |
| profile.department | string | 否 | 学院或专业 |
| profile.grade | string | 否 | 年级 |
| profile.joined_at | string | 否 | 加入猫协日期 |
| must_change_password | boolean | 否 | 默认 true |

### 响应示例

```json
{
  "code": 0,
  "message": "成员账号创建成功",
  "data": {
    "id": "0d65df4f-10c0-45e4-a48d-b04e0af7d3d2",
    "student_no": "20252160A1010",
    "role": "member",
    "status": "active",
    "must_change_password": true
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 业务规则

1. `student_no` 必须唯一。
2. `initial_password` 必须哈希后写入 `password_hash`。
3. 默认 `status = active`。
4. 默认 `must_change_password = true`。
5. 默认 `token_version = 1`。
6. 创建成功后写入管理员操作日志。
7. 接口永远不返回初始密码和密码哈希。

### 错误码

| code | HTTP | 说明 |
|---:|---:|---|
| 40001 | 400 | 参数错误 |
| 40302 | 403 | 权限不足 |
| 40901 | 409 | 学号已存在 |

## 7.3 批量导入成员账号

### 基本信息

```http
POST /api/v1/admin/users/import
Authorization: Bearer <admin_token>
Content-Type: multipart/form-data
```

| 项目 | 内容 |
|---|---|
| 权限 | 管理员 |
| 说明 | 批量导入内部成员账号 |
| 涉及表 | `users`、`user_profiles`、`admin_operation_logs` |

### 请求字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| file | file | 是 | xlsx/csv 文件 |
| default_password | string | 是 | 默认初始密码 |
| must_change_password | boolean | 否 | 默认 true |

### 导入文件字段建议

| 列名 | 必填 | 说明 |
|---|---:|---|
| student_no | 是 | 学号 |
| real_name | 否 | 真实姓名 |
| nickname | 否 | 昵称；为空时可用真实姓名或学号生成 |
| department | 否 | 学院或专业 |
| grade | 否 | 年级 |
| role | 否 | 默认为 member |

### 响应示例

```json
{
  "code": 0,
  "message": "导入完成",
  "data": {
    "total": 30,
    "success_count": 28,
    "failed_count": 2,
    "failed_items": [
      {
        "row": 6,
        "student_no": "20250001",
        "reason": "学号已存在"
      }
    ]
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 业务规则

1. 导入时应逐行校验，返回失败行信息。
2. 可以选择部分成功，不必一行失败导致全部失败。
3. 所有导入账号默认 `must_change_password = true`。
4. 所有初始密码都必须哈希存储。
5. 批量导入操作写入管理员操作日志。

---

## 7.4 管理员重置成员密码

### 基本信息

```http
PATCH /api/v1/admin/users/{user_id}/password
Authorization: Bearer <admin_token>
```

| 项目 | 内容 |
|---|---|
| 权限 | 管理员 |
| 说明 | 管理员为成员重置密码 |
| 涉及表 | `users`、`admin_operation_logs` |

### Path 参数

| 参数 | 类型 | 说明 |
|---|---|---|
| user_id | string | 目标用户 ID |

### 请求体

```json
{
  "new_password": "123456",
  "must_change_password": true
}
```

### 响应示例

```json
{
  "code": 0,
  "message": "密码已重置",
  "data": {
    "user_id": "0d65df4f-10c0-45e4-a48d-b04e0af7d3d2",
    "must_change_password": true
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 后端处理流程

```text
1. 校验当前用户是 admin
2. 查询目标用户
3. 哈希 new_password
4. 更新 password_hash
5. must_change_password = true
6. password_updated_at = now()
7. login_failed_count = 0
8. locked_until = null
9. token_version + 1
10. 写入 admin_operation_logs
```

### 业务规则

1. 重置密码后，目标用户旧 token 必须失效。
2. 重置密码后，目标用户下次登录必须改密。
3. 不返回新密码和密码哈希。
4. 如果目标用户是 super_admin，普通 admin 不应允许重置。

## 7.5 禁用 / 恢复账号

### 基本信息

```http
PATCH /api/v1/admin/users/{user_id}/status
Authorization: Bearer <admin_token>
```

| 项目 | 内容 |
|---|---|
| 权限 | 管理员 |
| 说明 | 禁用、恢复或标记成员离开猫协 |
| 涉及表 | `users`、`admin_operation_logs` |

### 请求体

```json
{
  "status": "blocked",
  "reason": "已退出猫协"
}
```

### status 可选值

| 值 | 说明 |
|---|---|
| active | 恢复正常 |
| blocked | 禁用 |
| left | 已退出猫协 |
| deleted | 已删除，软删除语义 |

### 响应示例

```json
{
  "code": 0,
  "message": "账号状态已更新",
  "data": {
    "user_id": "0d65df4f-10c0-45e4-a48d-b04e0af7d3d2",
    "status": "blocked"
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 业务规则

1. `blocked`、`left`、`deleted` 用户不能登录。
2. 禁用账号时建议 `token_version + 1`，让旧 token 失效。
3. 恢复账号为 `active` 后，可以正常登录。
4. 禁止管理员禁用自己，避免系统无人管理。
5. 状态变更写入管理员操作日志。

```json
}
```

# 8. 前端页面对应关系

## 8.1 登录页

页面字段：

```text
喵喵号
密码
字母验证码
协议勾选项
登录按钮
```

对应接口：

```http
GET /api/v1/auth/captcha
POST /api/v1/auth/login
```

前端逻辑：

```text
1. 页面进入时请求验证码
2. 用户点击“换一张”重新请求验证码
3. 用户点击登录时提交喵喵号、密码、captcha_id、captcha_code、agree_terms
4. 登录成功后保存 access_token
5. 如果 must_change_password = true，跳转修改密码页
6. 如果 must_change_password = false，进入地图首页
```

## 8.2 修改密码页

对应接口：

```http
PATCH /api/v1/auth/password
```

前端逻辑：

```text
1. 输入旧密码、新密码、确认新密码
2. 修改成功后替换为后端返回的新 access_token
3. 跳转初始化身份页
```

## 8.3 我的页面

对应接口：

```http
GET /api/v1/auth/me
```

前端逻辑：

```text
1. 获取当前用户资料
2. 展示昵称、头像、角色、喵喵号
3. 如果 role = admin，显示管理员入口
```

# 9. 错误码汇总

| code | HTTP | 说明 | 前端建议 |
|---:|---:|---|---|
| 0 | 200 | 成功 | 正常处理 |

# 10. MVP 落地顺序

建议按下面顺序实现：

```text
1. users / user_profiles / auth_captchas 表
2. 密码哈希工具
3. JWT 签发和解析工具
4. GET /auth/captcha
5. POST /auth/login
6. GET /auth/me
7. PATCH /auth/password
8. 管理员创建成员账号
9. 管理员重置密码
10. 管理员禁用 / 恢复账号
```

这样可以先保证成员能够登录系统，再继续开发地图、猫咪库、任务模块。
