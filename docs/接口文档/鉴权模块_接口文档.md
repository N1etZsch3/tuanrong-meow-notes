# 校园猫协地图任务系统｜鉴权模块接口文档

> 版本：V1.1.1 鉴权接口设计稿
> Base URL：`/api/v1`
> 登录方式：喵喵号 + 密码 + 字母验证码 + 微信 OpenID 绑定 + JWT
> 账号来源：管理员创建或批量导入

---

## 1. 模块边界

### 1.1 V1.1.1 做什么

| 功能 | 是否 V1.1.1 做 | 说明 |
|---|---:|---|
| 获取字母验证码 | 是 | 登录页展示 |
| 喵喵号密码登录 | 是 | 内部成员登录 |
| 微信 OpenID 自动登录 | 已实现，待真机验收 | 仅限已通过喵喵号密码验证并绑定过的成员 |
| 首次登录绑定微信 OpenID | 已实现，待真机验收 | 后端使用微信 code 换 OpenID，前端不得提交 OpenID |
| 获取当前用户 | 是 | 前端恢复登录态 |
| 修改自己的密码 | 是 | 默认密码登录后必须改密 |
| 解绑自己的微信 | 已实现，待真机验收 | 普通成员和管理员均只能解绑本人，成功后立即退出登录 |
| 退出登录 | 是 | 前端清 token，后端可轻量返回成功 |
| 管理员创建成员账号 | 是 | 因为 V1 不开放注册 |
| 管理员重置成员密码 | 是 | 替代找回密码 |
| 管理员禁用/恢复账号 | 是 | 内部成员管理 |
| 管理员清除微信绑定 | 已实现，待真机验收 | 成员换微信或误绑后的兜底恢复路径 |

### 1.2 V1.1.1 不做什么

```text
成员自主注册
手机号登录
邮箱登录
QQ / 第三方登录
成员自主绑定多个微信
前端直接提交 OpenID
找回密码
refresh token
复杂会话管理
```

## 2. 接口总览

### 2.1 公开接口

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/auth/captcha` | 获取登录验证码 |
| POST | `/auth/login` | 成员登录 |
| POST | `/auth/wechat/login` | 微信 OpenID 自动登录 |

### 2.2 登录后接口

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/auth/me` | 获取当前登录用户 |

| PATCH | `/auth/password` | 修改自己的密码 |
| DELETE | `/auth/wechat-binding` | 解绑当前用户自己的微信并使现有 Token 失效 |
| POST | `/auth/logout` | 退出登录 |

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/admin/users` | 查询成员账号列表 |

| POST | `/admin/users/import` | 批量导入成员账号 |
| PATCH | `/admin/users/{user_id}/password` | 重置成员密码 |
| PATCH | `/admin/users/{user_id}/status` | 禁用/恢复账号 |
| PATCH | `/admin/users/{user_id}/role` | 调整成员角色 |
| DELETE | `/admin/users/{user_id}/wechat-binding` | 清除成员微信绑定 |

## 4. 登录状态与权限规则

### 4.1 Token Header

需要登录的接口统一传：

```http
Authorization: Bearer <access_token>
```

### 4.2 JWT Payload

```json
{
  "sub": "0d65df4f-10c0-45e4-a48d-b04e0af7d3d2",
  "student_no": "trmx0001",
  "meow_no": "trmx0001",
  "role": "member",
  "token_version": 1,
  "iat": 1781750000,
  "exp": 1782354800
}
```

### 4.3 首次登录强制改密

当用户：

```json
{
  "must_change_password": true
}
```

前端应跳转到修改密码页。

后端只允许访问：

```http
GET /api/v1/auth/me
PATCH /api/v1/auth/password
POST /api/v1/auth/logout
```

访问其他接口返回：

```json
{
  "code": 40301,
  "message": "请先修改初始密码",
  "data": null,
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

---

### 4.4 微信 OpenID 绑定规则

1. OpenID 只能由后端使用微信小程序 `code2Session` 换取，前端只提交 `wx.login` / `uni.login` 得到的临时 `code`。
2. 成员首次绑定前仍必须通过喵喵号、密码、验证码校验，确保邀请制入口不被绕过。
3. 未绑定账号第一次带 `wechat_code` 登录时，必须同时提交 `agree_wechat_bind = true`。
4. 已绑定账号的 OpenID 必须一致；不一致时拒绝登录并提示联系管理员。
5. 同一个 OpenID 只能绑定一个未删除用户。
6. 管理员清除绑定后，后端必须递增 `users.token_version`，使既有 JWT 失效。
7. 清除绑定后，成员下一次使用喵喵号密码登录并重新绑定当前微信。
8. 所有登录用户都可通过 `/auth/wechat-binding` 解绑本人，包括管理员；该接口不接收目标用户 ID，解绑成功后递增本人 `token_version` 并立即退出登录。
9. 管理员接口只允许重置非管理员成员绑定；管理员之间不能相互解绑，管理员本人必须走自助解绑接口。

### 4.5 微信鉴权模式

后端通过环境变量控制微信登录启用程度：

| 模式 | 说明 |
|---|---|
| `off` | 关闭微信绑定和自动登录，保留现有密码登录作为回滚模式 |
| `optional` | 允许绑定和自动登录，同时保留旧密码登录兼容生产前端；1.1.1 首次部署使用 |
| `enforced` | 已绑定账号必须通过匹配的微信 OpenID 登录；密码登录仅作为未绑定或管理员解绑后的兜底 |

因为开发服务器和生产服务器当前是同一台，生产部署必须先使用 `optional`，待真机验证通过后再评估是否切换到 `enforced`。

### 4.6 小程序前端登录顺序

1. 加载页启动时调用 `uni.login({ provider: "weixin" })` 获取临时 code，并优先请求 `POST /auth/wechat/login`。
2. 自动登录成功后保存后端返回的 JWT 和当前用户信息，再按 `next_action` / 当前用户状态进入改密页、资料初始化页或首页。
3. 当前微信未绑定或账号不可用时清除旧本地会话并进入喵喵号密码登录页。
4. 无法取得微信 code 时不提交伪造或空 OpenID；若本地已有有效 JWT，可继续使用既有登录态，否则进入密码登录页。
5. 用户在密码登录页点击登录并通过表单校验后，前端弹出原生确认框，告知“登录后，当前微信将自动与该喵喵号绑定，用于后续自动登录和账号保护”。用户确认后才获取微信 code，并提交 `wechat_code` 和 `agree_wechat_bind = true`；用户取消时不调用密码登录接口。
6. 确认绑定后若 `uni.login` 暂不可用，前端提示用户重试，不提交空 code，也不降级为未绑定的密码登录。后端继续保持 `optional` 模式，仅用于兼容尚未升级的生产前端和紧急回滚。
7. 加载页自动登录全程静默，不显示绑定提示、模态框或失败 Toast；只有自动登录无法建立对应会话时才进入密码登录页兜底。
8. 前端不接收、不保存、不展示 OpenID、session key 或小程序 app secret。

# 5. 公开接口

## 5.1 获取登录验证码

### 基本信息

```http
GET /api/v1/auth/captcha
```

| 项目 | 内容 |
|---|---|
| 权限 | 公开 |
| 说明 | 生成登录页字母验证码 |
| 涉及表 | `auth_captchas` |

### Query 参数

无。

### 响应字段

| 字段 | 类型 | 说明 |
|---|---|---|
| captcha_id | string | 验证码 ID，登录时提交 |
| captcha_image | string | base64 图片，可以是 SVG 或 PNG |
| expires_in | int | 有效期，单位秒 |

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "captcha_id": "b5f7a6d8-2a3e-4f91-98e5-4c22f47a1f31",
    "captcha_image": "data:image/svg+xml;base64,PHN2ZyB4bWxucz0iLi4u",
    "expires_in": 300
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 业务规则

1. 验证码长度建议 4 位。
2. 只保存验证码 hash，不保存明文。
3. 默认有效期 5 分钟。
4. 同一个验证码登录成功后标记 `used_at`。
5. 前端点击“换一张”时重新请求本接口。

## 5.2 成员登录

### 基本信息

```http
POST /api/v1/auth/login
```

| 项目 | 内容 |
|---|---|
| 权限 | 公开 |
| 说明 | 使用喵喵号、密码、验证码登录；1.1.1 起可在首次验证通过后绑定当前微信 OpenID |
| 涉及表 | `users`、`auth_captchas`、`user_profiles` |

### 请求体

```json
{
  "student_no": "20252160A1010",
  "password": "123456",
  "captcha_id": "b5f7a6d8-2a3e-4f91-98e5-4c22f47a1f31",
  "captcha_code": "A7KD",
  "agree_terms": true,
  "wechat_code": "0f3abc...",
  "agree_wechat_bind": true
}
```

### 请求字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| meow_no | string | 是 | 喵喵号，格式如 `trmx0001` |
| student_no | string | 否 | 兼容旧字段，含义同喵喵号 |
| password | string | 是 | 密码 |
| captcha_id | string | 是 | 验证码 ID |
| captcha_code | string | 是 | 用户输入的验证码 |
| agree_terms | boolean | 是 | 是否勾选协议；当账号 `must_change_password = true` 时必须为 `true`，已完成首次改密的账号可为 `false` |
| wechat_code | string | 否 | 微信小程序 `wx.login` / `uni.login` 返回的临时 code；后端用它换取 OpenID |
| agree_wechat_bind | boolean | 条件必填 | 当本次登录会创建微信绑定时必须为 `true` |

### 响应字段

| 字段 | 类型 | 说明 |
|---|---|---|
| access_token | string | JWT |
| token_type | string | 固定为 Bearer |
| expires_in | int | access token 有效期，单位秒 |
| must_change_password | boolean | 是否必须修改初始密码 |
| user | object | 当前用户信息 |

### 响应示例

```json
{
  "code": 0,
  "message": "login success",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.xxx",
    "token_type": "Bearer",
    "expires_in": 604800,
    "must_change_password": true,
    "user": {
      "id": "0d65df4f-10c0-45e4-a48d-b04e0af7d3d2",
      "student_no": "trmx0001",
      "meow_no": "trmx0001",
      "nickname": "金炜然",
      "avatar_url": null,
      "role": "member",
      "status": "active",
      "profile_completed": false
    },
    "next_action": "change_password"
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 后端处理流程

```text
1. 校验验证码是否存在、未过期、未使用
2. 校验验证码内容
3. 根据 student_no / meow_no 查询 users
4. 校验用户状态 status 是否为 active
5. 校验 locked_until 是否仍在锁定期
6. 校验密码
7. 失败则增加 login_failed_count，必要时锁定账号
8. 如果用户 `must_change_password = true`，校验 `agree_terms` 必须为 true
9. 如果请求包含 `wechat_code` 且微信鉴权模式不是 `off`，后端调用微信 `code2Session` 换取 OpenID
10. 如果用户尚未绑定 OpenID，则校验 `agree_wechat_bind = true`，并确认该 OpenID 未绑定其他账号后写入绑定
11. 如果用户已绑定 OpenID，则校验本次 OpenID 与库内绑定一致
12. 成功则清空 login_failed_count 和 locked_until
13. 更新 last_login_at；如本次使用微信 code，则同步更新 last_wechat_login_at
14. 标记验证码 used_at
15. 签发 JWT，payload 写入 token_version
16. 返回 token 和用户信息
```

### 错误码

| code | HTTP | 说明 |
|---:|---:|---|
| 40002 | 400 | 缺少必要参数 |
| 40003 | 400 | 验证码错误 |
| 40004 | 400 | 验证码已过期 |
| 40005 | 400 | 未勾选协议 |
| 40103 | 401 | 学号或密码错误 |
| 40303 | 403 | 账号已被禁用 |
| 40304 | 403 | 微信绑定不匹配或未确认绑定 |
| 40903 | 409 | 当前微信已绑定其他账号 |
| 42301 | 423 | 登录失败次数过多，账号临时锁定 |

## 5.3 微信 OpenID 自动登录

### 基本信息

```http
POST /api/v1/auth/wechat/login
```

| 项目 | 内容 |
|---|---|
| 权限 | 公开 |
| 说明 | 使用微信临时 code 换取 OpenID，并为已绑定成员签发 JWT |
| 涉及表 | `users`、`user_profiles` |

### 请求体

```json
{
  "code": "0f3abc..."
}
```

### 请求字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| code | string | 是 | 微信小程序 `wx.login` / `uni.login` 返回的临时 code |

### 响应字段

与 `POST /api/v1/auth/login` 成功响应一致。

### 后端处理流程

```text
1. 检查 CATMAP_WECHAT_AUTH_MODE 是否为 optional 或 enforced
2. 使用后端配置的小程序 appid/secret 调微信 code2Session
3. 从微信响应取得 OpenID；不向前端返回 OpenID
4. 根据 users.wechat_openid 查询未删除用户
5. 校验用户状态为 active
6. 更新 last_login_at 和 last_wechat_login_at
7. 签发 JWT，payload 写入 token_version
8. 返回 token、用户信息和 next_action
```

### 错误码

| code | HTTP | 说明 |
|---:|---:|---|
| 40001 | 400 | 参数错误或微信 code 无效 |
| 40104 | 401 | 当前微信尚未绑定喵喵号 |
| 40303 | 403 | 账号已被禁用 |
| 50001 | 500 | 微信服务暂不可用或服务端配置缺失 |

# 6. 登录后接口

## 6.1 获取当前登录用户

### 基本信息

```http
GET /api/v1/auth/me
Authorization: Bearer <access_token>
```

| 项目 | 内容 |
|---|---|
| 权限 | 登录用户 |
| 说明 | 前端刷新页面后恢复登录态，也用于判断管理员入口 |
| 涉及表 | `users`、`user_profiles` |

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": "0d65df4f-10c0-45e4-a48d-b04e0af7d3d2",
    "student_no": "20252160A1010",
    "role": "member",
    "status": "active",
    "must_change_password": true,
    "profile": {
      "nickname": "金炜然",
      "avatar_url": null,
      "real_name": "金炜然",
      "department": "计算机学院",
      "grade": "2025"
    }
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 业务规则

1. 不返回 `password_hash`。
2. 如果 token 过期或无效，返回 `40102`。
3. 如果 token 中的 `token_version` 与数据库不一致，返回 `40102`。
4. 如果 `role = admin`，前端可以在个人中心显示管理员入口。

## 6.2 续签 Access Token

### 基本信息

```http
POST /api/v1/auth/renew
Authorization: Bearer <access_token>
```

| 项目 | 内容 |
|---|---|
| 权限 | 已完成首次改密的登录用户 |
| 说明 | 用仍有效的 access token 换取新的 access token，新的有效期从续签时间重新计算 |
| 涉及表 | `users` |

### 请求体

无。

### 响应字段

| 字段 | 类型 | 说明 |
|---|---|---|
| access_token | string | 新签发的 access token |
| token_type | string | 固定为 Bearer |
| expires_in | int | 新 token 有效期，单位秒，当前为 604800 |

### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.renewed",
    "token_type": "Bearer",
    "expires_in": 604800
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 业务规则

1. 只接受尚未过期、签名合法、`token_version` 与数据库一致的 access token。
2. 续签不引入 `refresh_tokens` 表，不保存服务端会话。
3. 新 token 的 `iat` 和 `exp` 从续签请求时间重新计算。
4. `must_change_password = true` 时不允许续签，仍只能访问 `/auth/me`、`/auth/password`、`/auth/logout`。
5. 前端建议在 token 剩余有效期小于 1 天时调用本接口；如果已经返回 401，则清空登录态并跳转登录页。

### 错误码

| code | HTTP | 说明 |
|---|---:|---|
| 40101 | 401 | 未登录 |
| 40102 | 401 | Token 无效或已过期 |
| 40301 | 403 | 请先修改初始密码 |
| 40303 | 403 | 账号已被禁用 |

## 6.3 修改自己的密码

### 基本信息

```http
PATCH /api/v1/auth/password
Authorization: Bearer <access_token>
```

| 项目 | 内容 |
|---|---|
| 权限 | 登录用户 |
| 说明 | 用户修改自己的密码，首次登录默认密码时必须调用 |
| 涉及表 | `users` |

### 请求体

```json
{
  "old_password": "123456",
  "new_password": "N1etzsche@2026",
  "confirm_password": "N1etzsche@2026"
}
```

### 请求字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| old_password | string | 是 | 旧密码 |
| new_password | string | 是 | 新密码 |
| confirm_password | string | 是 | 确认新密码 |

### 密码规则建议

| 规则 | 建议 |
|---|---|
| 最小长度 | 8 位 |
| 最大长度 | 20 位 |
| 字符要求 | 至少包含字母和数字，仅允许大小写字母、数字和 `@_!` |
| 与旧密码关系 | 不能与旧密码相同 |
| 空白字符 | 不允许全空白 |

### 响应示例

```json
{
  "code": 0,
  "message": "密码修改成功，请重新登录",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.new",
    "token_type": "Bearer",
    "expires_in": 604800,
    "must_change_password": false,
    "profile_completed": false,
    "token_invalidated": true,
    "next_action": "complete_profile"
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 后端处理流程

```text
1. 校验旧密码是否正确
2. 校验 new_password 和 confirm_password 是否一致
3. 校验新密码强度
4. 校验新密码不能与旧密码相同
5. 更新 users.password_hash
6. 更新 users.password_updated_at
7. 设置 users.must_change_password = false
8. users.token_version + 1
9. 清空 login_failed_count 和 locked_until
10. 返回新 access token，前端替换旧 token 并进入资料初始化页
```

### 错误码

| code | HTTP | 说明 |
|---:|---:|---|
| 40001 | 400 | 参数错误 |
| 40103 | 401 | 旧密码错误 |
| 40902 | 409 | 新密码不能与旧密码相同 |

## 6.4 解绑自己的微信

### 基本信息

```http
DELETE /api/v1/auth/wechat-binding
Authorization: Bearer <access_token>
```

| 项目 | 内容 |
|---|---|
| 权限 | 登录用户，包括管理员 |
| 说明 | 解绑当前 Token 所属用户自己的微信，不接收目标用户 ID |
| 涉及表 | `users` |

### 请求体

无。

### 响应示例

```json
{
  "code": 0,
  "message": "微信绑定已清除",
  "data": {
    "user_id": "0d65df4f-10c0-45e4-a48d-b04e0af7d3d2",
    "wechat_bound": false,
    "token_version": 3,
    "token_invalidated": true
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 业务规则

1. 用户身份只能取自当前有效 JWT，不允许在路径或请求体指定其他用户。
2. 清空 `wechat_openid`、`wechat_bound_at` 和 `last_wechat_login_at`。
3. `token_version + 1`，本次请求使用的 Token 及同账号旧 Token 立即失效。
4. 前端调用前必须二次确认并告知“解绑后将立即退出登录”；成功后先清除本地会话，再跳到登录页。
5. 下次进入小程序时需用喵喵号和密码登录并重新绑定当前微信。
6. 当前账号尚未绑定微信时返回 `40001`，不修改 `token_version`，当前会话保持有效。

### 错误码

| code | HTTP | 说明 |
|---:|---:|---|
| 40001 | 400 | 当前账号尚未绑定微信 |
| 40101 | 401 | 未登录 |
| 40102 | 401 | Token 无效或已过期 |

## 7.2 创建成员账号

### 基本信息

```http

```

| 项目 | 内容 |
|---|---|

| 涉及表 | `users`、`user_profiles`、`admin_operation_logs` |

```json
{
```

### 请求字段

| 字段 | 类型 | 必填 | 说明 |

| initial_password | string | 是 | 初始密码 |

| profile.nickname | string | 是 | 昵称 |
| profile.real_name | string | 否 | 真实姓名 |
| profile.department | string | 否 | 学院或专业 |
| profile.grade | string | 否 | 年级 |

```json
{

    "must_change_password": true
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 业务规则

1. `student_no` 必须唯一。
2. `initial_password` 必须哈希后写入 `password_hash`。

4. 默认 `must_change_password = true`。
5. 默认 `token_version = 1`。
6. 创建成功后写入管理员操作日志。
7. 接口永远不返回初始密码和密码哈希。

## 7.4 管理员重置成员密码

### 基本信息

```http
PATCH /api/v1/admin/users/{user_id}/password
Authorization: Bearer <admin_token>
```

| 项目 | 内容 |
|---|---|
| 权限 | 管理员 |
| 说明 | 管理员为成员重置密码 |
| 涉及表 | `users`、`admin_operation_logs` |

### Path 参数

| 参数 | 类型 | 说明 |
|---|---|---|
| user_id | string | 目标用户 ID |

### 请求体

```json
{
  "new_password": "123456",
  "must_change_password": true
}
```

### 响应示例

```json
{
  "code": 0,
  "message": "密码已重置",
  "data": {
    "user_id": "0d65df4f-10c0-45e4-a48d-b04e0af7d3d2",
    "must_change_password": true
  },
  "trace_id": "018f3c2e9b7e4f1a9a7e6c8d9"
}
```

### 后端处理流程

```text
1. 校验当前用户是 admin
2. 查询目标用户
3. 哈希 new_password
4. 更新 password_hash
5. must_change_password = true
6. password_updated_at = now()
7. login_failed_count = 0
8. locked_until = null
9. token_version + 1
10. 写入 admin_operation_logs
```

### 业务规则

1. 重置密码后，目标用户旧 token 必须失效。
2. 重置密码后，目标用户下次登录必须改密。
3. 不返回新密码和密码哈希。
4. 如果目标用户是 super_admin，普通 admin 不应允许重置。


# 10. MVP 落地顺序

建议按下面顺序实现：

```text
1. users / user_profiles / auth_captchas 表
2. 密码哈希工具
3. JWT 签发和解析工具
4. GET /auth/captcha
5. POST /auth/login
6. GET /auth/me
7. PATCH /auth/password
8. 管理员创建成员账号
9. 管理员重置密码
10. 管理员禁用 / 恢复账号
```

这样可以先保证成员能够登录系统，再继续开发地图、猫咪库、任务模块。
