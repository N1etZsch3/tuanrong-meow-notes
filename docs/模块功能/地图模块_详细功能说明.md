# 校园猫协地图任务系统｜地图模块详细功能说明

> 文档版本：V1.0  
> 模块名称：地图模块  
> 适用阶段：MVP / 第一版功能设计  
> 设计定位：地图模块是系统首页，也是成员查找点位、搜索校园猫协相关位置、查看任务/猫咪/物资点、导航，以及管理员在地图上进行点位操作的核心入口。

---

# 1. 模块定位

地图模块是「校园猫协地图任务系统」的首页和空间工作台。

它不是通用地图产品，而是围绕校园猫协内部协作设计的专用地图页面，主要承担以下职责：

1. 展示校园范围内的猫协业务点位。
2. 支持成员搜索任务点、猫咪点、物资点、地标和地址。
3. 支持成员点击点位后查看简略信息。
4. 支持成员从点位卡片发起导航。
5. 支持地图缩放时不同层级的点位预览。
6. 支持底部动态内容区域根据用户状态和搜索状态切换内容。
7. 为管理员在地图上长按插点、编辑点位、拖拽修改点位位置等能力预留功能入口。

地图模块第一版重点是：

```text
成员端：看点、搜点、选点、查看简略信息、导航。
管理员端：预留地图上直接操作标点的能力。
```

# 3. 顶部标题区

## 3.1 默认展示内容

顶部标题区默认展示：

```text
猫协地图 + 猫爪 logo
探索校园 · 守护猫咪
```

该区域用于建立页面识别，不承载搜索和扫一扫功能。

## 3.2 去除内容

原型图右上角的以下按钮不保留：

```text
搜索
扫一扫
```

原因：

1. 搜索功能统一放入地图下方的可拖拽搜索栏。
2. 第一版内部成员系统不需要扫一扫入口。
3. 顶部区域应尽量干净，避免压缩地图可视面积。

## 3.3 滚动 / 拖拽联动

当用户向下拖动搜索栏，进入地图浏览模式时：

```text
顶部标题和 logo 逐渐淡出
地图区域向上扩展
地图区域向下扩展
可视地图面积增大
```

当用户向上拖动搜索栏或点击搜索栏，进入搜索模式时：

```text
顶部标题区域可以隐藏或弱化
搜索栏和底部搜索结果区成为页面主视觉
地图区域被压缩或作为背景保留
```

# 4. 地图展示区

## 4.1 地图服务范围

地图只服务于本校校园范围。

地图默认进入后应定位到校园中心，并限制或弱化校外区域。

第一版需要支持：

1. 默认中心点为校园中心。
2. 默认缩放级别适合校园浏览。
3. 限制地图可拖动范围。
4. 只加载校园范围内的业务点位。
5. 点位按照校园区域分类。

## 4.2 地图基础能力

地图底层由高德地图提供，第一版需要使用高德地图完成：

1. 底图展示。
2. 地图缩放。
3. 地图拖动。
4. Marker 标点展示。
5. 用户当前位置显示。
6. 步行导航能力。
7. 管理员地图点击或长按选点。
8. 地图显示范围限制。

系统自己负责：

1. 任务点数据。
2. 猫咪点数据。
3. 物资点数据。
4. 地标点数据。
5. 路线文字说明。
6. 现场照片。
7. 点位搜索。
8. 点位简略信息展示。
9. 点位业务状态展示。

# 5. 地图点位类型

地图上主要展示四类点位：

| 点位类型 | 说明 | 示例 |
|---|---|---|
| 猫咪点 | 常驻猫咪、高频出现点、临时观察点 | 小橘常驻点、体育馆后草坪猫咪点 |
| 物资点 | 猫粮、诱捕笼、航空箱、药品等存放点 | 猫协物资点 #1 |
| 任务点 | 日常任务、喂食任务、巡查任务、清洁任务等 | 食堂后方投喂清洁 |
| 紧急任务点 | 紧急救助、受伤猫咪、需要快速响应的任务 | 北门草丛紧急救助任务 |

后期可以扩展：

| 点位类型 | 说明 |
|---|---|
| 地标点 | 校门、教学楼、食堂、宿舍区、体育馆等 |
| 固定喂食点 | 长期稳定投喂位置 |
| 临时观察点 | 某次观察产生的临时位置 |
| 路线提示点 | 用于校园内部路径辅助说明 |

# 6. Marker 展示规则

## 6.1 Marker 图标

不同点位需要使用不同 Marker 图标。

建议图标风格统一，颜色区分明显：

| 点位类型 | 图标建议 | 颜色建议 |
|---|---|---|
| 常驻猫咪 | 猫爪 / 猫头 | 紫色 / 绿色 |
| 物资点 | 箱子 / 背包 / 物资包 | 橙色 |
| 日常任务 | 任务单 / 剪贴板 | 蓝色 |
| 紧急任务 | 感叹号 / 红色定位点 | 红色 |
| 地标点 | 建筑 / 旗帜 / 定位点 | 灰色 / 绿色 |

## 6.2 Marker 缩放预览规则

地图上的点位预览与地图缩放等级有关。

### 6.2.1 地图缩小时

当地图缩放等级较低、点位较密集时：

```text
只显示 Marker 图标
不显示点位标题
不显示图片小卡片
必要时可以做 Marker 聚合
```

目的：

1. 减少地图视觉拥挤。
2. 保持地图可读性。
3. 避免点位卡片互相遮挡。

### 6.2.2 地图放大到一定程度时

当地图缩放等级达到设定阈值后，部分点位可以展示紧贴 Marker 的小预览卡片。

小预览卡片内容：

```text
点位标题
点位第一张图片 / 封面图
```

示例：

```text
[猫咪照片] 小橘常驻点
[物资照片] 猫协物资点 #1
[任务照片] 北门紧急任务
```

预览卡片要求：

1. 紧贴对应 Marker。
2. 不遮挡主要地图操作。
3. 优先展示高优先级点位，例如紧急任务。
4. 点位过多时可以只展示部分高优先级点位。
5. 点击预览卡片等同于点击 Marker。

### 6.2.3 预览控制字段建议

地图点位数据中建议包含：

```json
{
  "cover_photo_url": "https://example.com/cover.jpg",
  "preview_enabled": true,
  "preview_min_zoom": 17,
  "display_level": 10
}
```

说明：

| 字段 | 作用 |
|---|---|
| cover_photo_url | 地图贴点预览卡片使用的封面图 |
| preview_enabled | 是否允许该点位显示地图小预览卡片 |
| preview_min_zoom | 地图缩放到多少级后展示预览 |
| display_level | 点位展示优先级，紧急任务应更高 |

# 7. 可拖拽搜索栏

## 7.1 搜索栏位置

搜索栏替代原型图中「常驻猫咪、物资点、日常任务、紧急任务」那一排图例区域。

搜索栏位于地图下方、动态内容区上方。

默认文案：

```text
搜索点位、地标、地址
```

也可以使用：

```text
搜索猫咪、任务、物资点、地标
```

## 7.2 搜索范围

搜索栏主要支持搜索：

1. 任务点。
2. 猫咪点。
3. 物资点。
4. 地标。
5. 校园区域。
6. 地址或位置描述。
7. 猫咪名称或别名。
8. 任务标题。
9. 物资点名称。

示例搜索词：

```text
北门
体育馆
小橘
猫粮点
紧急任务
教学楼B
宿舍区
食堂后方
诱捕笼
```

## 7.3 搜索栏拖拽能力

搜索栏既是搜索入口，也是地图页的抽屉拖拽把手。

它有三种主要状态：

| 状态 | 触发方式 | 页面表现 |
|---|---|---|
| 默认状态 | 进入地图页 | 地图正常高度，搜索栏在地图下方 |
| 地图浏览状态 | 向下拖动搜索栏 | 地图上下扩展，顶部标题消失 |
| 搜索模式 | 点击搜索栏或向上拖动 | 搜索栏和底部内容区占据主要页面 |

# 8. 地图浏览状态

## 8.1 进入方式

用户向下拖动搜索栏时进入地图浏览状态。

## 8.2 页面变化

进入地图浏览状态后：

```text
地图向上扩展
地图向下扩展
顶部标题区逐渐消失
底部动态内容区压缩
搜索栏保留为较小状态
用户可以更大范围查看地图
```

## 8.3 适用场景

地图浏览状态适合：

1. 用户查看校园点位整体分布。
2. 用户拖动地图找附近点位。
3. 用户点击 Marker 查看底部简略信息。
4. 用户配合定位按钮查看自己和点位的距离。

# 9. 搜索模式

## 9.1 进入方式

用户点击搜索栏，或向上拖动搜索栏时进入搜索模式。

## 9.2 页面变化

进入搜索模式后：

```text
搜索栏进入输入状态
键盘弹出
地图区域被压缩或弱化
底部动态内容区扩展为搜索结果区
搜索栏 + 结果列表占据主要页面
```

## 9.3 搜索结果实时更新

用户输入关键词时，系统动态匹配相关内容。

匹配对象包括：

1. 任务点。
2. 猫咪点。
3. 物资点。
4. 地标点。
5. 校园区域。
6. 地址文本。
7. 路线说明中的关键词。
8. 位置说明中的关键词。

## 9.4 搜索结果展示内容

每条搜索结果建议展示：

```text
图标
标题
类型标签
简要说明
距离
状态
```

示例：

```text
[红色紧急图标] 北门草丛紧急救助任务
紧急任务 · 进行中 · 距离 120m

[紫色猫爪图标] 小橘常驻点
常驻猫咪 · 健康 · 教学楼B附近

[橙色物资图标] 猫协物资点 #1
物资点 · 猫粮 / 诱捕笼 · 体育馆旁

[灰色地标图标] 北门
地标 · 校园入口
```

## 9.5 搜索无结果

当搜索没有匹配结果时，展示：

```text
没有找到相关点位
可以换个关键词试试，例如“北门”“小橘”“猫粮”
```

管理员用户可以额外提示：

```text
长按地图可以添加新的点位
```

# 10. 底部动态内容区

底部动态内容区替代原型图中的「最新任务」固定区域。

它不是固定展示最新任务，而是根据当前状态动态变化。

## 10.1 状态 A：搜索栏为空，成员没有已接取任务

展示「最新任务」。

内容包括：

1. 最新发布任务。
2. 紧急任务优先。
3. 可接取任务优先。
4. 展示距离、状态、类型、参与人数。

示例：

```text
最新任务

北门草丛紧急救助任务
紧急任务 · 进行中 · 距离 120m

食堂后方投喂清洁
日常任务 · 可接取 · 距离 250m
```

## 10.2 状态 B：搜索栏为空，成员存在已接取任务

优先展示「我的进行中任务」。

内容包括：

1. 我已接取的任务。
2. 任务状态。
3. 截止时间。
4. 距离。
5. 快捷操作。

示例：

```text
我的任务

北门草丛紧急救助任务
紧急任务 · 进行中 · 距离 120m
按钮：导航 / 查看详情 / 完成打卡
```

如果有多个已接任务，可以按紧急程度和截止时间排序。

## 10.3 状态 C：搜索栏有文字

展示搜索结果列表。

搜索结果动态匹配：

1. 任务点。
2. 地标。
3. 猫咪点。
4. 物资点。
5. 地址。
6. 校园区域。

用户可以点击任意结果。

## 10.4 状态 D：用户点击搜索结果

点击搜索结果后：

```text
地图定位到该点位
该点位 Marker 高亮
底部动态内容区切换为该点位简略信息
显示导航按钮
```

## 10.5 状态 E：用户点击地图 Marker

点击 Marker 后：

```text
Marker 高亮
地图视角移动到该点位附近
底部动态内容区展示该点位简略信息
```

点击地图空白区域后：

```text
取消选中 Marker
底部内容区回到默认状态
```

5. 位置名称。
6. 距离。
7. 路线说明摘要。
8. 导航按钮。

```text
任务标题

任务状态
优先级
```

# 12. 导航功能

## 12.1 导航入口

以下位置可以出现导航按钮：

1. 选中点位后的底部简略信息卡片。
2. 搜索结果列表中的快捷操作。
3. 我的进行中任务卡片。
4. 任务详情页。
5. 猫咪详情页。
6. 物资点详情页。

地图模块重点处理前 3 类入口。

## 12.2 导航内容

导航需要包含：

1. 目标点位名称。
2. 目标经纬度。
3. 高德导航参数。
4. 路线说明。
5. 入口提示。
6. 附近地标。
7. 现场照片。

## 12.3 校园内部非标准地点说明

校园猫协很多点位不是标准 POI，因此导航不能只依赖高德地图。

例如：

```text
导航终点：体育馆西门
路线说明：到达体育馆西门后，沿右侧小路向后走约 50 米，看到草坪后往左侧树丛方向走，小橘常在长椅附近。
```

因此点位应支持：

| 信息 | 作用 |
|---|---|
| route_instruction | 路线说明 |
| landmark_hint | 附近地标提示 |
| entrance_hint | 入口说明 |
| map_point_photos | 现场图、入口图、路线图 |

# 13. 用户当前位置

地图页面需要支持用户当前位置展示。

## 13.1 展示内容

用户当前位置在地图上展示为蓝色定位点。

可以显示：

1. 当前定位点。
2. 定位精度范围。
3. 朝向箭头，可选。

## 13.2 定位按钮

地图上应提供「定位」按钮。

点击后：

```text
请求定位权限
获取用户当前位置
地图移动到当前位置
刷新附近点位距离
```

## 13.3 定位失败

定位失败时提示：

```text
无法获取当前位置，请检查定位权限
```

并允许用户继续手动查看地图和搜索点位。

# 14. 管理员地图操作能力预留

管理员地图操作属于地图模块、管理员模块、任务模块、猫咪库模块、物资管理模块之间的交叉能力。

地图模块第一版需要在功能说明和接口设计中预留，不必一次性完成所有复杂业务闭环。

## 14.1 管理员长按地图快速插点

### 14.1.1 触发方式

管理员在地图上长按某个位置。

### 14.1.2 页面反馈

长按后出现快速插点菜单。

菜单选项可以包括：

```text
添加任务点
添加常驻猫咪点
添加物资点
添加地标点
添加临时观察点
```

### 14.1.3 操作流程

```text
管理员长按地图
  ↓
系统获取长按位置的 lng / lat
  ↓
弹出快速插点菜单
  ↓
管理员选择点位类型
  ↓
进入对应点位编辑表单
  ↓
填写标题、说明、路线、图片等信息
  ↓
保存
  ↓
创建 map_points 记录
  ↓
根据点位类型跳转或创建对应业务记录
```

### 14.1.4 与业务模块关系

| 快速插点类型 | 地图模块职责 | 后续业务模块职责 |
|---|---|---|
| 添加任务点 | 创建空间点位、保存坐标 | 任务模块发布任务 |
| 添加常驻猫咪点 | 创建猫咪相关点位 | 猫咪库维护猫咪档案 |
| 添加物资点 | 创建物资空间点位 | 物资模块维护物资详情 |
| 添加地标点 | 创建地标点位 | 地图模块可独立处理 |
| 添加临时观察点 | 创建临时点位 | 猫咪观察记录模块处理 |

## 14.2 管理员点位编辑状态

管理员不能在普通浏览状态下直接拖动点位，避免误操作。

必须先将点位设置为编辑状态。

### 14.2.1 进入编辑状态

流程：

```text
管理员点击 Marker
  ↓
底部动态内容区展示点位简略信息
  ↓
点击「编辑点位」
  ↓
点位进入编辑状态
  ↓
Marker 外观发生变化，例如高亮、出现拖拽手柄
```

### 14.2.2 编辑状态下可操作内容

编辑状态下，管理员可以：

1. 拖拽 Marker 修改位置。
2. 修改点位标题。
3. 修改点位类型。
4. 修改位置名称。
5. 修改详细位置描述。
6. 修改路线说明。
7. 修改入口提示。
8. 修改附近地标提示。
9. 修改封面图和点位照片。
10. 修改显隐状态。
11. 修改 Marker 图标。
12. 修改点位展示优先级。
13. 修改是否允许预览卡片。
14. 修改预览卡片展示缩放阈值。

### 14.2.3 拖拽修改点位位置

流程：

```text
点位进入编辑状态
  ↓
Marker 变为可拖拽
  ↓
管理员拖动 Marker 到新位置
  ↓
前端实时显示新经纬度
  ↓
管理员点击保存
  ↓
后端更新 map_points.lng
  ↓
后端更新 map_points.lat
  ↓
后端更新 map_points.geom
  ↓
地图刷新 Marker 位置
```

### 14.2.4 取消编辑

管理员可以取消编辑。

取消后：

```text
Marker 回到原位置
表单修改不保存
退出编辑状态
```

### 14.2.5 保存编辑

保存成功后：

```text
更新点位信息
退出编辑状态
刷新地图点位
底部动态内容区展示更新后的点位简略信息
记录管理员操作日志，后期功能
```

# 15. 点位表单信息

管理员编辑点位时，地图模块涉及的基础表单字段包括：

| 字段 | 说明 |
|---|---|
| point_type | 点位类型 |
| point_scope | 长期点位或临时点位 |
| name | 点位名称 |
| subtitle | 副标题 |
| description | 点位说明 |
| location_name | 位置名称 |
| location_detail | 详细位置描述 |
| lng | 经度 |
| lat | 纬度 |
| route_instruction | 路线说明 |
| landmark_hint | 附近地标提示 |
| entrance_hint | 入口说明 |
| icon_key | Marker 图标 key |
| display_level | 展示优先级 |
| visibility | 可见状态 |
| status | 点位状态 |
| cover_photo_url | 封面图 |
| preview_enabled | 是否展示贴点预览 |
| preview_min_zoom | 贴点预览最低缩放等级 |

业务模块扩展字段不在地图点位基础表单中强行承担。

例如：

| 业务字段 | 所属模块 |
|---|---|
| 任务最大参与人数 | 任务模块 |
| 任务截止时间 | 任务模块 |
| 猫咪健康状态 | 猫咪库模块 |
| 猫咪绝育状态 | 猫咪库模块 |
| 物资库存数量 | 物资管理模块 |

# 16. 地图模块状态模型

地图页可以抽象为以下状态：

| 状态编号 | 状态名称 | 触发方式 | 页面表现 |
|---|---|---|---|
| S1 | 默认状态 | 进入地图页 | 地图 + 搜索栏 + 默认动态内容 |
| S2 | 地图浏览状态 | 向下拖动搜索栏 | 地图扩大，顶部标题消失 |
| S3 | 搜索输入状态 | 点击搜索栏 / 向上拖动 | 搜索栏和结果区占据主要页面 |
| S4 | 搜索结果状态 | 输入关键词 | 动态展示搜索结果 |
| S5 | 点位选中状态 | 点击 Marker / 搜索结果 | 地图高亮点位，底部显示点位卡片 |
| S6 | 管理员长按插点状态 | 管理员长按地图 | 出现快速插点菜单 |
| S7 | 管理员点位编辑状态 | 管理员点击编辑点位 | Marker 可拖拽，表单可编辑 |

# 17. 核心交互流程

## 17.1 用户进入地图页

```text
用户进入地图页
  ↓
前端请求地图初始化数据
  ↓
加载校园中心点、边界、缩放等级
  ↓
加载地图 Marker 配置
  ↓
加载默认点位
  ↓
查询用户是否有进行中任务
  ↓
底部动态内容区展示“我的任务”或“最新任务”
```

## 17.2 用户搜索点位

```text
用户点击搜索栏
  ↓
进入搜索模式
  ↓
输入关键词
  ↓
前端请求搜索接口
  ↓
后端返回任务点、猫咪点、物资点、地标等结果
  ↓
底部动态内容区展示搜索结果
```

## 17.3 用户点击搜索结果

```text
用户点击搜索结果
  ↓
地图移动到目标点位
  ↓
目标 Marker 高亮
  ↓
请求点位简略信息
  ↓
底部动态内容区展示点位卡片
  ↓
用户可以查看详情或导航
```

## 17.4 用户点击地图 Marker

```text
用户点击 Marker
  ↓
目标 Marker 高亮
  ↓
请求点位简略信息
  ↓
底部动态内容区展示点位卡片
  ↓
用户可以查看详情或导航
```

## 17.5 用户导航到点位

```text
用户点击导航按钮
  ↓
系统获取点位经纬度
  ↓
系统获取路线说明、入口提示、现场照片
  ↓
调用高德地图导航能力
  ↓
展示导航路线
  ↓
同时展示校园内部路线说明
```

## 17.6 管理员长按插点

```text
管理员长按地图
  ↓
获取长按位置经纬度
  ↓
弹出快速插点菜单
  ↓
选择点位类型
  ↓
进入点位编辑表单
  ↓
填写信息并保存
  ↓
创建地图点位
  ↓
根据类型进入任务/猫咪/物资后续流程
```

## 17.7 管理员拖拽修改点位

```text
管理员点击 Marker
  ↓
点击编辑点位
  ↓
点位进入编辑状态
  ↓
拖拽 Marker 到新位置
  ↓
前端显示新经纬度
  ↓
管理员点击保存
  ↓
后端更新点位坐标
  ↓
地图刷新点位位置
```

# 18. 权限规则

## 18.1 普通成员权限

普通成员可以：

1. 查看地图。
2. 查看公开点位。
3. 搜索点位。
4. 点击 Marker。
5. 查看点位简略信息。
6. 发起导航。
7. 查看任务点、猫咪点、物资点相关详情。
8. 从猫咪点添加观察记录。
9. 从任务点接取任务、完成打卡、放弃任务，具体能力由任务模块控制。

普通成员不能：

1. 长按地图插点。
2. 编辑点位。
3. 拖拽移动点位。
4. 隐藏点位。
5. 删除点位。
6. 修改点位表单信息。

## 18.2 管理员权限

管理员可以：

1. 使用普通成员全部能力。
2. 长按地图快速插点。
3. 编辑已有点位。
4. 进入点位编辑状态。
5. 拖拽修改点位位置。
6. 修改点位表单信息。
7. 上传和删除点位照片。
8. 隐藏或恢复点位。
9. 归档点位。
10. 通过点位创建任务、猫咪点、物资点等业务对象，后续挂钩对应模块。

# 19. 数据依赖

地图模块主要依赖以下数据表：

| 表名 | 作用 |
|---|---|
| campuses | 校园地图默认配置、中心点、边界、缩放等级 |
| campus_areas | 校园内部区域，如北门、体育馆、宿舍区 |
| map_points | 地图点位主表，保存经纬度、类型、名称、位置说明等 |
| map_point_photos | 点位照片，包括封面、现场图、路线图、入口图 |
| map_marker_configs | Marker 图标、颜色、层级、默认显示配置 |
| tasks | 任务业务详情，用于任务点卡片 |
| cats | 猫咪业务详情，用于猫咪点卡片 |
| supply_points | 物资点业务详情，用于物资点卡片 |
| task_participants | 判断当前用户是否已接任务、任务人数状态 |

# 20. 接口能力预留

地图模块后续接口可按两组设计。

## 20.1 成员端地图接口

```text
GET /api/v1/map/init
GET /api/v1/map/points
GET /api/v1/map/search
GET /api/v1/map/points/{point_id}/summary
GET /api/v1/map/points/{point_id}/navigation
```

### 说明

| 接口 | 用途 |
|---|---|
| /map/init | 获取校园配置、区域、Marker 配置、默认状态 |
| /map/points | 获取地图点位列表，支持筛选和范围查询 |
| /map/search | 搜索任务点、猫咪点、物资点、地标、地址 |
| /map/points/{id}/summary | 获取点位底部卡片简略信息 |
| /map/points/{id}/navigation | 获取导航所需信息和路线说明 |

## 20.2 管理员地图操作接口

```text
POST   /api/v1/admin/map/points
PATCH  /api/v1/admin/map/points/{point_id}
PATCH  /api/v1/admin/map/points/{point_id}/location
PATCH  /api/v1/admin/map/points/{point_id}/visibility
POST   /api/v1/admin/map/points/{point_id}/photos
DELETE /api/v1/admin/map/points/{point_id}/photos/{photo_id}
```

### 说明

| 接口 | 用途 |
|---|---|
| POST /admin/map/points | 管理员创建地图点位 |
| PATCH /admin/map/points/{id} | 管理员编辑点位基础信息 |
| PATCH /admin/map/points/{id}/location | 管理员拖拽后更新点位经纬度 |
| PATCH /admin/map/points/{id}/visibility | 管理员隐藏、恢复点位 |
| POST /admin/map/points/{id}/photos | 上传点位照片 |
| DELETE /admin/map/points/{id}/photos/{photo_id} | 删除点位照片 |

```text
任务发布
任务接取
```


# 23. 最终功能定义

地图模块最终定义为：

```text
地图页是校园猫协任务系统的空间工作台。

普通成员通过地图查看、搜索、选择、导航到任务点、猫咪点、物资点和地标点。
搜索栏同时承担搜索入口和可拖拽抽屉的作用。
底部动态内容区根据搜索状态、用户任务状态、点位选中状态切换展示内容。
地图 Marker 根据缩放级别切换展示形态：缩小时只显示图标，放大后显示贴点预览卡片。
管理员可以在地图上长按快速插点，也可以将已有点位切换为编辑状态后拖拽修改位置，并编辑点位表单信息。
地图模块只负责点位空间信息、展示信息、搜索、预览、导航和点位基础编辑；任务发布、猫咪档案、物资详情继续交给对应业务模块承接。
```
