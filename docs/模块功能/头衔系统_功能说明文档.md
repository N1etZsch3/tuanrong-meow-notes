# 头衔系统功能说明

## 1. 模块定位

头衔是猫协成员的当前身份标识。本期只负责身份展示、唯一占用和会长交接，不把头衔扩展为任务、药品、地图等业务模块的权限来源。头衔与成员所属部门相互独立。

## 2. 头衔注册表

`none` 表示无头衔，数据库统一存为 `NULL`，前端不显示徽章。其余 12 个头衔全局唯一：

| key | 显示名称 | 盾牌 |
|---|---|---|
| `president` | 会长 | 金色 |
| `vice_president` | 副会长 | 紫色 |
| `survival_head` / `survival_deputy` | 生存保障部部长 / 副部长 | 绿色 |
| `activity_head` / `activity_deputy` | 活动部部长 / 副部长 | 绿色 |
| `publicity_head` / `publicity_deputy` | 宣传部部长 / 副部长 | 绿色 |
| `secretary_head` / `secretary_deputy` | 秘书部部长 / 副部长 | 绿色 |
| `care_head` / `care_deputy` | 养护部部长 / 副部长 | 绿色 |

前端为 12 个非空头衔分别配置不同的标签背景色，并复用项目盾牌 SVG 生成金、紫、绿三种本地素材。

## 3. 权限与生命周期

- 只有当前 `title=president` 的用户可以查询头衔占用目录、授予或剥夺其他成员的非会长头衔，以及转让会长。
- 普通管理员不自动获得头衔管理权。头衔操作使用独立授权通道，不受“管理员账号不可由普通管理员编辑”的资料护栏影响。
- 任何非会长头衔持有者都可主动退出；无头衔时退出接口幂等成功。
- 会长不可普通退出，必须原子转让，避免出现无会长状态。
- 会长转让给普通成员时，接任者自动成为 `admin`，并增加 `token_version` 使其旧令牌失效；接任者原头衔同时释放。转让给已有管理员时不额外改变令牌版本。
- 软删除普通成员会释放其头衔。恢复账号不会自动恢复旧头衔。
- 初始会长由开发者脚本设置；若已有其他会长，脚本拒绝覆盖。

## 4. 数据与并发保证

迁移 `20260718_0019_add_profile_title` 为 `user_profiles` 新增 nullable `title varchar(64)`，并创建 `title IS NOT NULL` 的部分唯一索引 `uq_user_profiles_title`。服务层先检查占用并锁定相关资料行，数据库唯一索引兜底处理并发竞争，冲突统一返回 409/63009。

会长转让在一个数据库事务内完成：锁定双方资料、释放旧会长、释放接任者旧头衔、写入新会长、按需提升角色并写审计。任何一步失败都会回滚。

## 5. 小程序交互

- `TitleBadge.vue` 统一渲染盾牌、标签颜色和文字；无头衔不占位。个人资料卡、人员列表和人员详情均复用该组件。
- 人员详情仅向会长显示头衔操作。选择器只列出空闲头衔和目标当前持有的头衔，不允许选择 `president`；会长交接使用独立按钮。
- 会长创建成员时可选择空闲初始头衔；其他管理员不显示该选择器，后端也会拒绝其非空提交。
- 非会长持有者可在设置页确认退出头衔。
- 授予、剥夺、退出和转让都使用微信原生确认框，只有确认分支发起写请求。

采用微信小程序原生 [`picker`](https://developers.weixin.qq.com/miniprogram/dev/component/picker.html) 的 selector 模式承载最多 12 个选项；不使用最多只能展示 6 项的 ActionSheet。确认操作使用 [`wx.showModal`](https://developers.weixin.qq.com/miniprogram/dev/api/ui/interaction/wx.showModal.html)，徽章按[自定义组件规范](https://developers.weixin.qq.com/miniprogram/dev/framework/custom-component/)复用。

## 6. 接口与错误码

- `GET /api/v1/admin/titles`：会长查询 12 个头衔的占用状态和持有者摘要。
- `PATCH /api/v1/admin/users/{user_id}/title`：会长授予或剥夺普通头衔。
- `POST /api/v1/admin/titles/transfer`：原子转让会长。
- `POST /api/v1/profile/me/title/resign`：当前用户退出非会长头衔。
- `POST /api/v1/admin/users`：会长可在 `profile.title` 中设置初始头衔。

错误码：63009 头衔已占用；63010 需要会长权限；63011 头衔非法或流转方式非法；63012 会长必须先转让。

初始会长命令（在 `backend` 目录执行）：

```powershell
py -3.11 -m scripts.grant_president --meow-no <现有成员喵喵号>
```

脚本只接受现有、active、未删除成员，不会覆盖已有其他会长。

## 7. 当前边界

- 未实现任期历史、多人共享同一头衔、一人多头衔或头衔申请审批。
- 未按头衔开放其他业务权限；后台管理能力仍使用现有角色体系。
- 数据库迁移、初始会长 seed 和微信开发者工具/真机交互需在目标环境单独验收后再部署。
