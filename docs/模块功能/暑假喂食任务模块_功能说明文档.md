# 校园猫协地图任务系统｜暑假喂食任务模块功能说明文档

> 版本：Summer Feeding V1  
> 适用阶段：暑假可用版 / 任务模块收敛版  
> 模块范围：仅支持「暑假喂食任务」一种任务  
> 设计目标：先完成暑假真实使用所需的最小闭环，同时保留后期扩展到完整任务模块的空间。

---

## 1. 模块定位

暑假喂食任务模块是任务模块的收敛版本。

本版本只解决一件事：

```text
管理员发布固定喂食点的暑假投喂任务，成员根据地图找到点位，在指定日期完成投喂，并在任务详情页形成完成动态记录。
```

本版本不追求完整任务协作体系，优先保证：

1. 任务可发布。
2. 点位可导航。
3. 执行日期清晰。
4. 成员可完成投喂。
5. 任务动态能沉淀。
6. 任务图片能展示。
7. 后期能扩展到完整任务模块。

### 当前维护补充（2026-07-10）

任务详情页不再单独展示「完成照片」区块。已提交的完成照片只在对应的任务动态详情中展示和预览，避免与任务动态重复；未提交的打卡表单仍可按原流程选择、移除和提交照片。

## 2. 本版本功能边界

### 2.1 本版本只做

| 功能 | 是否做 | 说明 |
|---|---:|---|
| 暑假喂食任务 | 做 | `task_type = feeding`，任务名称、说明、日期、点位、物资、图片、动态 |
| 地图点位 | 做 | 管理员发布时选择地图点或使用当前位置 |
| 日历选择执行日期 | 做 | 管理员从日历中选择具体执行日期 |
| 时间区间展示 | 做 | 由最早执行日期和最晚执行日期形成区间 |
| 任务详情页 | 做 | 展示任务图片、名称、日期、地址、说明、物资、动态 |
| 导航前往 | 做 | 复用地图模块导航能力 |
| 完成投喂 | 做 | 成员点击完成，生成执行日完成记录和任务动态 |
| 任务动态时间线 | 做 | 展示任务创建、每日完成、修改等记录 |
| 任务图片 | 做 | 发布任务时上传，详情页轮播展示，文件走统一 COS 上传模块 |

### 2.2 本版本暂不做

| 功能 | 处理方式 |
|---|---|
| 多任务类型 | 暂不做，只保留 `feeding` |
| 接取 / 放弃 / 满员 | 暂不做，后期完整任务模块再恢复 |
| 管理员审核完成 | 暂不做，成员完成后直接记为当日已完成 |
| 排班与指派 | 暂不做页面闭环，库表可保留后期能力 |
| 复杂物资库存 | 暂不做库存扣减，物资字段默认展示“猫粮、水” |
| 实时 WebSocket | 暂不做，详情页进入时拉取，完成后刷新 |
| 多人协同状态 | 暂不做，只记录“谁完成了哪一天的投喂” |

## 3. 角色与权限

### 3.1 普通成员

普通成员可以：

1. 在地图上查看暑假喂食任务点。
2. 进入任务详情页。
3. 查看任务图片轮播。
4. 查看任务名称、执行时间、任务点、任务说明、物资。
5. 查看任务动态时间线。
6. 导航到任务点。
7. 在当天或允许补录的日期完成投喂。
8. 上传完成照片，若本版本选择开启完成照片。

### 3.2 管理员

管理员可以：

1. 从「我的」页面进入管理员入口发布任务。
2. 在地图上长按某个位置，进入发布暑假喂食任务流程。
3. 发布、编辑、取消、归档暑假喂食任务。
4. 在发布任务时选择地图点或使用当前位置。
5. 从日历中选择任务执行日期。
6. 上传任务图片。
7. 查看任务完成动态。
8. 撤销某一天的错误完成记录，作为可选管理能力。

## 4. 任务信息结构

暑假喂食任务详情只展示以下内容：

| 信息 | 来源 | 说明 |
|---|---|---|
| 任务图片 | `task_photos` / 上传模块 | 详情页顶部轮播图 |
| 任务名称 | `tasks.title` | 如“学生宿舍区北侧喂食点” |
| 时间 | `task_execution_dates` | 展示为区间 + 已选择日期 |
| Marker 地图点 | `map_points` | 经纬度、地址、路线说明、导航 |
| 任务说明 | `tasks.description` | 如“暑假投喂” |
| 物资 | `tasks.required_items` | 默认“猫粮、水” |
| 任务动态 | `task_activity_logs` | 时间线渲染 |

## 5. 时间设计

### 5.1 任务不是一次性任务

暑假喂食任务不是“某一天完成后整个任务结束”的任务。

它更接近长期任务或日常任务：

```text
一个固定喂食点
  ↓
暑假期间若干个执行日期
  ↓
每天完成后只更新当天的执行状态
  ↓
整个任务仍继续存在，直到所有执行日期结束或管理员归档
```

### 5.2 管理员发布时选择具体执行日期

管理员在发布页中通过日历选择需要执行的日期。

示例：

```text
执行日期：
2026-07-02
2026-07-09
2026-07-16
2026-07-23
```

前端展示时可以显示：

```text
2026-07-02 至 2026-07-23
共 4 次投喂
```

任务详情页中可以展示最近一次或当前日期状态：

```text
本次任务日期：2026年7月2日
```

也可以在后续版本增加“全部日期日历视图”。

### 5.3 完成逻辑

成员点击「完成投喂」后，系统完成的是某一个执行日期，而不是整个任务。

处理规则：

1. 默认完成当天执行日期。
2. 如果当天没有执行计划，提示“今天不是该任务执行日”。
3. 如果允许补录，成员可以选择最近未完成日期。
4. 如果该执行日期已完成，按钮展示“已完成”，不允许重复完成。
5. 完成后新增任务动态。

## 6. 任务状态设计

### 6.1 父任务状态

父任务表示整个暑假喂食任务。

| 状态 | 说明 |
|---|---|
| `pending` | 已创建但未发布，当前版本可不展示 |
| `in_progress` | 正在执行中，暑假喂食任务主要状态 |
| `completed` | 整个暑假周期结束，管理员归档或系统自动完成 |
| `cancelled` | 任务取消 |
| `archived` | 后期扩展，历史归档 |

当前暑假版推荐：

```text
发布后直接进入 in_progress。
每个执行日期完成后，不修改父任务为 completed。
所有执行日期结束后，由管理员手动归档或系统自动置为 completed。
```

### 6.2 执行日期状态

执行日期表示某一天是否完成投喂。

| 状态 | 说明 |
|---|---|
| `pending` | 待完成 |
| `completed` | 已完成 |
| `cancelled` | 当日取消 |
| `skipped` | 当日跳过，不计为异常 |
| `missed` | 过期未完成，可作为查询态或后期定时任务写入 |

## 7. 任务动态设计

任务动态用于任务详情页时间线。

动态来源包括：

1. 管理员发布任务。
2. 管理员修改任务。
3. 成员完成某一天投喂。
4. 管理员取消某一天执行。
5. 管理员取消或归档整个任务。

### 7.1 时间线展示示例

```text
2026-07-02 09:30
系统生成本次投喂任务

第一次任务完成
Nietzsche 于 2026-07-02 18:24 完成投喂

第二次任务完成
小林 于 2026-07-09 18:11 完成投喂

第三次任务完成
阿杰 于 2026-07-16 17:56 完成投喂
```

### 7.2 动态生成规则

成员完成投喂后，后端应同时写入：

```text
task_execution_dates.status = completed
task_checkins
task_activity_logs
```

前端不需要自己拼接动态文案，只渲染后端返回的动态列表。

## 8. 任务图片设计

任务图片用于任务详情页顶部轮播。

### 8.1 上传场景

管理员发布或编辑任务时上传。

### 8.2 上传流程

```text
前端选择图片
  ↓
调用统一上传模块
  ↓
图片压缩 / 缩略图生成 / 上传 COS
  ↓
返回 file_id、file_url、thumbnail_url
  ↓
发布任务接口保存图片关联
```

### 8.3 展示规则

1. 第一张图片作为默认封面。
2. 详情页顶部使用轮播图展示。
3. 地图 Marker 预览卡片可以使用封面缩略图。
4. 原图只在详情预览或大图查看时加载。
5. 列表、地图卡片优先使用缩略图。

## 9. 地图点设计

### 9.1 发布任务时选择点位

管理员有两种方式选择任务点：

| 方式 | 说明 |
|---|---|
| 地图选点 | 在地图上点击或拖动 Marker 确认经纬度 |
| 使用当前位置 | 获取当前定位作为任务点 |

### 9.2 地图长按发布入口

管理员在地图上长按后，出现操作菜单：

```text
发布暑假喂食任务
```

点击后进入发布页，并自动带入长按位置经纬度。

### 9.3 点位保存规则

发布暑假喂食任务时同时创建或复用：

```text
map_points
tasks
task_execution_dates
task_photos
task_activity_logs
```

`map_points` 只保存位置和地图展示信息，任务业务信息仍放在 `tasks` 及任务相关表中。

## 10. 发布任务流程

### 10.1 管理员入口发布

```text
我的
  ↓
管理员入口
  ↓
发布暑假喂食任务
  ↓
填写任务信息
  ↓
地图选点 / 使用当前位置
  ↓
日历选择执行日期
  ↓
上传任务图片
  ↓
发布
```

### 10.2 地图长按发布

```text
地图页长按
  ↓
出现“发布暑假喂食任务”按钮
  ↓
进入发布页
  ↓
自动带入长按坐标
  ↓
填写任务名称、说明、日期、物资、图片
  ↓
发布
```

### 10.3 发布表单字段

| 字段 | 必填 | 默认值 | 说明 |
|---|---:|---|---|
| 任务名称 | 是 | 无 | 如“学生宿舍区北侧喂食点” |
| 地图点 | 是 | 无 | 地图选点或当前位置 |
| 执行日期 | 是 | 无 | 日历多选 |
| 任务说明 | 否 | 暑假投喂 | 简短说明 |
| 物资 | 否 | 猫粮、水 | 可编辑文本 |
| 任务图片 | 否 | 无 | 建议至少 1 张 |
| 路线说明 | 否 | 无 | 校园内部非标准地点建议填写 |

## 11. 成员完成投喂流程

```text
成员进入任务详情
  ↓
查看日期、物资、路线、任务动态
  ↓
点击导航前往
  ↓
完成投喂
  ↓
点击“完成投喂”
  ↓
系统确认当前执行日期
  ↓
提交完成记录
  ↓
执行日期状态变为 completed
  ↓
任务动态新增一条完成记录
  ↓
前端刷新详情页
```

完成记录建议字段：

| 字段 | 必填 | 说明 |
|---|---:|---|
| 执行日期 | 是 | 默认当天 |
| 是否完成 | 是 | 默认 true |
| 处理结果 | 否 | 如“已补充猫粮和水” |
| 备注 | 否 | 特殊情况说明 |
| 完成照片 | 否 | 如果开启拍照打卡，可上传 |
| 完成位置 | 否 | 可记录成员提交时定位 |

## 14. 与后期完整任务模块的关系

暑假版应当作为完整任务模块的子集落地。

### 14.1 本次保留的扩展点

| 扩展点 | 当前处理 | 后期扩展 |
|---|---|---|
| 任务类型 | 固定 `feeding` | 扩展 daily / emergency / patrol / rescue 等 |
| 时间模型 | 执行日期表 | 支持一次性任务、周期任务、截止时间任务 |
| 任务参与 | 暂不接取 | 恢复 `task_participants` |
| 指派排班 | 暂不做页面 | 恢复 `task_assignments`、`duty_schedules` |
| 审核 | 暂不做 | 恢复 `pending_review`、审核记录 |
| 物资 | 文本字段 | 扩展结构化物资与库存 |
| 通知 | 可后置 | 新任务、完成、异常通知 |

### 14.2 关键原则

暑假版不要为了完整任务模块提前做复杂页面，但库表设计要避免推倒重来。

最重要的设计点是：

```text
父任务保存任务本身。
执行日期表保存每一天是否需要做、是否完成。
完成记录表保存谁在什么时候完成了哪一天。
动态表保存前端时间线。
```
