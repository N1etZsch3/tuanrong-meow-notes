# 校园猫协地图任务系统｜猫咪库模块功能文档

> 文档版本：V1.0  
> 模块名称：猫咪库模块  
> 适用阶段：MVP / 第一版功能设计  
> 适用端：移动端微信小程序 / H5  
> 关联模块：地图模块、任务模块、个人中心模块、管理员模块、图片上传模块  

---

# 1. 模块定位

猫咪库模块是校园猫咪信息档案中心，负责沉淀猫协长期维护的猫咪资料、活动点位、观察记录、健康记录和相关任务记录。

它在系统中承担三类核心职责：

1. **查找猫咪**：成员可以通过猫咪库快速找到某只猫，确认它的照片、名字、花色、别名、常驻区域和最近出现时间。
2. **维护档案**：管理员可以创建、编辑、归档猫咪档案，维护猫咪照片、别名、健康状态、绝育状态、性格标签和注意事项。
3. **沉淀记录**：成员和管理员可以持续提交观察记录、健康记录，并将相关任务记录挂接到猫咪档案中。

猫咪库在底部主菜单中作为一级入口存在，与「地图」「任务」「我的」并列。

# 2. 模块边界

## 2.1 猫咪库负责

| 能力 | 说明 |
|---|---|
| 猫咪档案 | 猫咪名称、头像、花色、性别、绝育、健康、亲人程度、状态 |
| 猫咪别名 | 支持一只猫多个称呼，方便搜索和识别 |
| 猫咪照片墙 | 保存头像、档案照、近照、健康异常照、观察来源照片 |
| 活动点位 | 关联常驻点、高频出现点、喂食点、最后出现点 |
| 观察记录 | 记录成员每一次看到猫、没看到猫、发现异常的情况 |
| 健康记录 | 记录受伤、生病、送医、绝育、康复等健康事件时间线 |
| 相关任务记录 | 展示与该猫相关的喂食、巡查、救助、送医、抓捕等任务 |
| 收藏猫咪 | 支撑个人中心「收藏猫咪」入口，作为后续增强功能 |

## 2.2 猫咪库不负责

| 能力 | 所属模块 | 说明 |
|---|---|---|
| 经纬度、路线说明、Marker 展示 | 地图模块 | 猫咪库只关联点位，不直接维护地图渲染逻辑 |
| 任务发布、接取、放弃、完成打卡 | 任务模块 | 猫咪库只展示相关任务和跳转入口 |
| 成员资料、我的观察记录聚合 | 个人中心模块 | 个人中心从猫咪库观察记录表反查 |
| 成员账号、角色、权限 | 鉴权 / 管理员模块 | 猫咪库只根据角色显示不同操作 |
| 图片压缩、缩略图生成、COS 上传 | 文件上传模块 | 猫咪库只使用图片文件返回的 URL 或文件 ID |

# 3. 用户角色与权限

## 3.1 普通成员

普通成员可以：

1. 查看猫咪库列表。
2. 搜索猫咪名称、别名、花色、区域。
3. 按区域、健康状态、绝育状态、亲人程度、最近出现时间筛选猫咪。
4. 查看猫咪详情。
5. 查看猫咪照片墙。
6. 查看猫咪常驻点位和最近出现点位。
7. 从猫咪详情跳转地图并导航。
8. 添加猫咪观察记录。
9. 查看健康记录时间线。
10. 查看相关任务记录。
11. 收藏或取消收藏猫咪，后期增强。

普通成员不允许直接修改猫咪基础档案、删除照片、归档猫咪。

## 3.2 管理员

管理员拥有普通成员能力，并额外可以：

1. 新增猫咪档案。
2. 编辑猫咪基础信息。
3. 维护猫咪别名。
4. 上传、排序、删除猫咪照片。
5. 设置主头像。
6. 修改健康状态。
7. 修改绝育状态。
8. 修改亲人程度。
9. 设置常驻区域和常驻点位。
10. 维护活动点位关系。
11. 添加或编辑健康记录。
12. 管理观察记录。
13. 归档猫咪档案。
14. 从地图长按插点流程中创建常驻猫咪点。

# 5. 猫咪库列表页

## 5.1 页面目标

猫咪库列表页负责快速浏览和查找校园猫咪。页面重点是：

1. 快速识别猫咪。
2. 快速判断猫咪最近在哪里出现。
3. 快速发现需要关注的猫咪。
4. 快速进入详情页。

列表页只展示摘要信息，完整档案放在详情页。

## 5.2 页面布局

推荐结构：

```text
顶部标题区
  猫咪库
  校园猫咪档案 · 共 N 只

搜索框
  搜索猫名 / 别名 / 花色 / 区域

快捷筛选
  全部 / 需关注 / 未绝育 / 最近出现 / 久未出现

控制栏
  全部区域 / 更多筛选 / 最近出现排序

猫咪列表卡片
  猫咪照片 + 名称 + 标签 + 区域 + 最近出现时间

底部主菜单
  地图 / 猫咪库 / 任务 / 我的
```

## 5.3 搜索能力

搜索框支持以下字段：

| 搜索对象 | 来源字段 |
|---|---|
| 猫咪名称 | `cats.name` |
| 猫咪别名 | `cat_aliases.alias_name` |
| 花色 | `cats.coat_color` |
| 常驻区域 | `campus_areas.name` |
| 常驻点位 | `map_points.name`, `map_points.location_name` |
| 性格标签 | `cats.personality_tags` |

搜索结果按以下优先级排序：

1. 名称完全匹配。
2. 别名完全匹配。
3. 名称或别名模糊匹配。
4. 区域 / 点位匹配。
5. 花色 / 标签匹配。
6. 最近出现时间倒序。

## 5.4 快捷筛选

列表页顶部只保留高频筛选：

| 筛选项 | 后端条件建议 | 说明 |
|---|---|---|
| 全部 | 默认条件 | 展示正常可见猫咪 |
| 需关注 | `health_status in ('watching','abnormal','injured','sick','in_treatment')` 或 `need_follow_up = true` | 优先发现异常猫咪 |
| 未绝育 | `neuter_status = 'not_neutered'` | 猫协管理高频需求 |
| 最近出现 | `last_seen_at >= now() - interval '3 days'` | 查看近期活跃猫咪 |
| 久未出现 | `last_seen_at < now() - interval '7 days'` 或 `last_seen_at is null` | 关注可能失踪或低频出现猫咪 |

## 5.5 更多筛选

更多筛选放入底部弹窗或侧边筛选面板。

筛选项包括：

| 筛选项 | 可选值 |
|---|---|
| 区域 | 全部区域、宿舍区、教学楼、体育馆、食堂、北门、图书馆等 |
| 花色 | 橘猫、狸花、奶牛、黑猫、三花、白猫、玳瑁、自定义 |
| 健康状态 | 健康、需观察、异常、受伤、疑似生病、治疗中、已恢复、未知 |
| 绝育状态 | 未知、未绝育、已绝育、已预约、暂不适合 |
| 亲人程度 | 亲人、一般、怕人、有攻击性、未知 |
| 档案状态 | 正常在校、暂时失踪、已领养、已死亡、转移照护、归档 |
| 最近出现时间 | 今天、三天内、一周内、一周前、未知 |

## 5.6 排序规则

| 排序项 | 说明 |
|---|---|
| 最近出现 | `last_seen_at desc nulls last` |
| 需关注优先 | 健康异常、未绝育、久未出现优先 |
| 名称排序 | 按名称拼音或创建顺序 |
| 最近更新 | `updated_at desc` |
| 新建优先 | `created_at desc` |

## 5.7 猫咪列表卡片

卡片只展示关键摘要，避免重复展示字段名和值。

建议字段：

| 区域 | 内容 |
|---|---|
| 左侧图片 | 猫咪头像或最近照片缩略图 |
| 主标题 | 猫咪名称 |
| 标签 | 健康状态、绝育状态、亲人程度、久未出现等 |
| 副标题 | 花色、主要别名、特殊摘要 |
| 位置信息 | 主要活动区域或常驻点位 |
| 时间信息 | 最近出现时间 |
| 右侧箭头 | 进入详情页 |

卡片示例：

```text
[照片] 小橘           健康 已绝育 亲人
      橘猫 · 别名橘子
      学生宿舍区
      今天 09:20
```

异常卡片示例：

```text
[照片] 台阶狸花       待观察 已绝育
      狸花 · 左眼待观察
      教学楼A
      今天 08:45
```

## 5.8 空状态

| 场景 | 展示文案 |
|---|---|
| 无猫咪数据 | 暂无猫咪档案，管理员可以先添加猫咪 |
| 搜索无结果 | 没有找到相关猫咪，换个关键词试试 |
| 筛选无结果 | 当前筛选下没有猫咪，可以调整筛选条件 |
| 网络失败 | 加载失败，请稍后重试 |

管理员在空状态下可以展示「新增猫咪」按钮。

# 6. 猫咪详情页

## 6.1 页面目标

猫咪详情页负责展示某只猫的完整档案和历史记录。成员进入详情页后，应能回答：

1. 这只猫是谁。
2. 长什么样。
3. 常在哪里。
4. 最近有没有出现。
5. 当前健康状态如何。
6. 是否已绝育。
7. 能不能靠近。
8. 喂食、抓捕、送医有什么注意事项。
9. 最近有哪些观察和任务记录。

## 6.2 页面结构

建议结构：

```text
顶部图片区
  猫咪主图 / 照片轮播
  名称、别名、状态标签

基础档案区
  花色、性别、绝育状态、健康状态、亲人程度、档案状态

位置区
  主要活动区域、常驻点位、最近出现点位、导航入口

性格与注意事项区
  性格标签、性格描述、喂食注意、抓捕注意、医疗注意

记录区
  观察记录、健康记录、相关任务

底部操作区
  添加观察记录、查看地图、导航、管理员编辑
```

## 6.3 顶部图片区

展示内容：

1. 主头像或封面图。
2. 猫咪名称。
3. 主要别名。
4. 健康状态标签。
5. 绝育状态标签。
6. 亲人程度标签。
7. 档案状态标签。

状态标签颜色建议：

| 状态 | 颜色建议 |
|---|---|
| 健康、已绝育 | 绿色 / 蓝色低饱和标签 |
| 需观察、久未出现 | 橙色标签 |
| 受伤、疑似生病、治疗中 | 红色或橙红色标签 |
| 已领养、归档 | 灰色标签 |

# 7. 添加观察记录

## 7.1 页面目标

观察记录用于记录成员某一次看到猫或没有看到猫的情况，是猫咪状态追踪的基础数据。

## 7.2 入口

1. 猫咪详情页点击「添加观察记录」。
2. 地图猫咪点卡片点击「添加观察记录」。
3. 任务完成打卡后自动生成观察记录。
4. 管理员补录。
5. 健康异常反馈流程。

## 7.3 表单字段

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| cat_id | uuid | 是 | 关联猫咪 |
| observed_at | datetime | 是 | 观察时间，默认当前时间 |
| map_point_id | uuid | 否 | 观察点位 |
| area_id | uuid | 否 | 观察区域 |
| lng / lat | number | 否 | 临时观察坐标 |
| is_seen | boolean | 是 | 是否见到猫 |
| health_status | enum | 否 | 观察到的健康状态 |
| cat_condition | enum | 否 | 猫咪当时状态 |
| behavior_status | string | 否 | 行为状态 |
| appetite_status | enum | 否 | 进食情况 |
| is_injured | boolean | 否 | 是否受伤 |
| injury_description | text | 否 | 受伤描述 |
| need_follow_up | boolean | 否 | 是否需要后续观察 |
| suggest_task | boolean | 否 | 是否建议发布任务 |
| description | text | 否 | 行为描述 |
| remark | text | 否 | 备注 |
| photos | file[] | 否 | 现场照片 |

## 7.4 表单交互建议

1. 先选择「见到了 / 没见到」。
2. 如果选择「没见到」，只保留时间、地点、备注、照片。
3. 如果选择「见到了」，展示健康、行为、食欲、受伤、后续观察等字段。
4. 如果健康状态为受伤、疑似生病、异常，自动勾选「需要后续观察」。
5. 如果勾选「建议发布任务」，提交后通知管理员或进入任务创建入口。
6. 照片上传使用统一文件上传模块，前端展示缩略图。

## 7.5 提交后的联动

提交观察记录后，后端可以同步更新：

1. `cats.last_seen_at`。
2. `cats.last_seen_map_point_id`。
3. `cats.health_status`，当观察记录确认异常时。
4. `cat_map_points.last_seen_at`。
5. 个人中心「我的观察记录」。
6. 通知管理员，后期增强。

# 8. 健康记录

## 8.1 页面目标

健康记录用于形成某只猫的健康事件时间线，适合记录受伤、生病、绝育、送医、用药和康复等事件。

## 8.2 来源

健康记录可以来自：

1. 管理员主动添加。
2. 观察记录触发。
3. 任务完成打卡触发。
4. 送医任务结束后补充。
5. 绝育任务完成后补充。

## 8.3 健康记录类型

| 类型 | 说明 |
|---|---|
| injury | 受伤 |
| sickness | 生病 |
| treatment | 治疗 |
| medicine | 用药 |
| neuter | 绝育 |
| recovery | 康复 |
| death | 死亡 |
| abnormal | 异常观察 |
| checkup | 检查 |
| other | 其他 |

## 8.4 时间线展示

健康记录页按 `occurred_at desc` 展示。

示例：

```text
2026-06-25  恢复观察
小橘左眼分泌物减少，精神正常，继续观察两天。

2026-06-22  异常观察
左眼有分泌物，建议后续观察。
```

# 9. 猫咪照片墙

## 9.1 照片类型

| 类型 | 说明 |
|---|---|
| avatar | 主头像 |
| profile | 档案照 |
| recent | 近照 |
| health | 健康异常照 |
| neuter | 绝育相关照 |
| observation | 观察记录照 |
| other | 其他 |

## 9.2 展示规则

1. 列表页优先使用 `cats.avatar_url`，没有头像时取最新近照或档案照。
2. 详情页顶部展示头像或照片轮播。
3. 照片墙按类型和时间展示。
4. 观察记录和健康记录中的照片可以进入猫咪照片墙。
5. 列表和卡片中优先加载 `thumbnail_url`，详情预览再加载原图。

## 9.3 管理员操作

管理员可以：

1. 上传照片。
2. 设置头像。
3. 修改说明。
4. 调整排序。
5. 删除照片记录。
6. 将照片关联到观察记录或健康记录。

# 10. 地图联动

## 10.1 猫咪库到地图

猫咪详情页可以跳转地图页，带上：

```text
cat_id
map_point_id
lng
lat
```

地图页接收参数后：

1. 定位到猫咪常驻点或最近出现点。
2. 高亮对应 Marker。
3. 展示猫咪点位底部卡片。
4. 支持导航。

## 10.2 地图到猫咪库

地图页点击猫咪点后，底部卡片提供：

1. 查看猫咪档案。
2. 添加观察记录。
3. 导航。

# 11. 任务联动

## 11.1 猫咪详情展示相关任务

猫咪详情页展示与该猫相关的任务，包括：

1. 喂食任务。
2. 巡查任务。
3. 救助任务。
4. 送医任务。
5. 抓捕任务。
6. 绝育任务。
7. 健康观察任务。

## 11.2 任务与观察记录联动

任务完成打卡可以生成猫咪观察记录。

建议流程：

```text
成员完成任务打卡
  ↓
填写是否见到猫、猫咪状态、健康情况、照片
  ↓
写入 task_checkins
  ↓
如果关联猫咪，则写入 cat_observation_records
  ↓
必要时更新 cats.last_seen_at / health_status
```

# 12. 管理员添加猫咪流程

## 12.1 从管理端添加

```text
管理员进入「我的」
  ↓
进入管理员入口
  ↓
选择猫咪管理
  ↓
点击新增猫咪
  ↓
填写猫咪名称、花色、性别、绝育、健康、亲人程度
  ↓
上传照片
  ↓
填写性格标签、注意事项
  ↓
选择常驻区域
  ↓
选择或新建常驻点位
  ↓
保存
```

## 12.2 从地图长按添加

```text
管理员在地图长按某个位置
  ↓
选择「添加常驻猫咪点」
  ↓
填写点位名称、位置说明、路线说明
  ↓
进入猫咪档案表单
  ↓
填写猫咪基础信息
  ↓
保存 map_points + cats + cat_map_points
```

## 12.3 创建成功后的结果

1. 猫咪库列表出现新猫咪。
2. 地图页出现猫咪点 Marker。
3. 猫咪详情页可查看常驻点位。
4. 后续成员可以添加观察记录。

# 14. 数据更新规则

## 14.1 最近出现时间

当新增观察记录且 `is_seen = true` 时：

1. 如果 `observed_at > cats.last_seen_at`，更新 `cats.last_seen_at`。
2. 如果存在 `map_point_id`，更新 `cats.last_seen_map_point_id`。
3. 如果该点位已存在于 `cat_map_points`，更新 `cat_map_points.last_seen_at`。

## 14.2 健康状态

当观察记录或健康记录确认状态变化时：

1. 普通健康观察可以只写观察记录。
2. 明确异常、受伤、疑似生病时，可以更新 `cats.health_status`。
3. 健康事件形成后，应写入 `cat_health_records`。
4. 恢复正常时，应写入康复健康记录，并更新 `cats.health_status = recovered` 或 `healthy`。

## 14.3 主头像

1. 管理员设置主头像时更新 `cats.avatar_url`。
2. 猫咪照片表保留原始照片记录。
3. 删除主头像照片时，应清空 `cats.avatar_url` 或自动选择下一张照片。

## 14.4 常驻点位

1. 一只猫允许多个点位。
2. `cats.primary_map_point_id` 指向主要常驻点。
3. `cat_map_points.is_primary = true` 的记录应与 `cats.primary_map_point_id` 保持一致。
4. 同一只猫建议只允许一个主要常驻点。

# 16. 前端实现建议

## 16.1 状态管理

Pinia 中建议维护：

```text
cat_list_filters
cat_list_sort
cat_list_cache
current_cat_detail
cat_observation_draft
```

## 16.2 列表性能

1. 分页加载。
2. 图片使用缩略图。
3. 搜索输入防抖。
4. 筛选条件变更后重置分页。
5. 从详情返回列表时保留滚动位置。

## 16.3 页面视觉

1. 列表页减少字段堆叠，突出猫名、图片、区域、最近出现和异常状态。
2. 详情页信息分组展示，避免长表单式阅读。
3. 异常状态使用更醒目的标签。
4. 普通状态使用低饱和标签。
5. 操作按钮固定在底部或分组底部，方便移动端点击。

# 17. 后端实现建议

建议服务分层：

```text
CatService
CatQueryService
CatPhotoService
CatObservationService
CatHealthService
CatMapPointService
AdminCatService
```

关键事务：

1. 管理员新增猫咪，同时写入 `cats`、`cat_aliases`、`cat_photos`、`cat_map_points`、`map_points`。
2. 新增观察记录，同时可能更新 `cats.last_seen_at`、`cats.health_status`、`cat_map_points.last_seen_at`。
3. 设置主常驻点，同时更新 `cats.primary_map_point_id` 和 `cat_map_points.is_primary`。
4. 设置主头像，同时更新 `cats.avatar_url` 和 `cat_photos`。
5. 归档猫咪时保留历史记录，不物理删除。
